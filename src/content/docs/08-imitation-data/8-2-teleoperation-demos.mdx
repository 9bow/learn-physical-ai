## 원격조작의 역할

모방학습에서 데이터는 모든 것입니다. 아무리 정교한 알고리즘도 질 낮은 시연 데이터로는 좋은 정책을 만들 수 없습니다. **원격조작(Teleoperation)** 은 인간 전문가가 로봇의 행동을 직접 제어하며 시연 데이터를 생성하는 방법입니다.

원격조작 시스템의 핵심 목표는 두 가지입니다: **직관적인 인터페이스**로 전문가가 쉽게 고품질 행동을 시연할 수 있어야 하고, **정확한 기록**으로 로봇의 실제 관절 상태와 환경 센서 데이터를 동기화하여 저장해야 합니다.

## 원격조작 방식 비교

### 방식 1: VR/모션 캡처 기반

VR 헤드셋과 컨트롤러 또는 모션 캡처 장갑을 이용하여 인간 손/팔의 동작을 로봇 관절 명령으로 변환합니다.

```text
VR 기반 원격조작 데이터 흐름:

사람 손 동작
    │ (VR 컨트롤러 / 글러브)
    ▼
손목 위치·방향, 손가락 굴곡
    │ (역운동학, IK)
    ▼
로봇 관절 각도 명령
    │
    ▼
실제 로봇 동작 + 센서 데이터 기록
```

**장점**: 자연스러운 손 동작을 활용할 수 있어 복잡한 조작 시연이 가능합니다. VR 시각 피드백으로 몰입감 있는 조작 환경을 제공합니다.

**단점**: VR 하드웨어 비용이 높습니다. 역운동학(IK) 매핑이 완벽하지 않아 로봇 손과 인간 손의 기구학 차이로 인해 부자연스러운 동작이 생길 수 있습니다. 학습 곡선이 존재합니다.

### 방식 2: 마스터-슬레이브 시스템

조작자가 로봇과 물리적으로 유사한 구조의 **마스터(master)** 장치를 조작하면, 그 명령이 실제 로봇(**슬레이브, slave**)에 그대로 전달됩니다.

**장점**: 마스터와 슬레이브의 기구학이 동일하면 IK 계산 없이 직접 관절 명령 전달이 가능합니다. 힘 피드백(force feedback)을 포함하면 촉각적 감각도 제공할 수 있습니다.

**단점**: 전용 마스터 장치 개발이 필요합니다. 원격 조작 시 통신 지연이 조작 품질에 영향을 줍니다.

### 방식 3: ALOHA류 양팔 마스터-슬레이브

**ALOHA(A Low-cost Open-source Hardware System for Bimanual Teleoperation)** 는 Stanford 연구팀이 개발한 저비용 양팔 원격조작 시스템입니다. 두 쌍의 동일한 로봇 팔을 사용하여, 한 쌍이 마스터(조작자 손으로 직접 구동), 다른 한 쌍이 슬레이브(마스터 명령 추종)로 작동합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              ALOHA 양팔 원격조작 구조                        │
│                                                              │
│  [조작자]                                                    │
│     │                                                        │
│  양손으로 마스터 로봇 팔 직접 조종                          │
│     │                                                        │
│  ┌──┴──────────────────┐                                    │
│  │  마스터 로봇 팔 (좌) │── 관절 상태 ──▶ 슬레이브 팔 (좌) │
│  └──────────────────────┘                                    │
│  ┌──────────────────────┐                                    │
│  │  마스터 로봇 팔 (우) │── 관절 상태 ──▶ 슬레이브 팔 (우) │
│  └──────────────────────┘                                    │
│                                                              │
│  카메라(1~3대) + 관절 인코더 → 데이터 기록                 │
└──────────────────────────────────────────────────────────────┘
```

ALOHA의 핵심 기여는 **저비용으로 높은 품질의 양팔 조작 시연**을 가능하게 한 것입니다. 상업용 마스터-슬레이브 시스템 대비 크게 낮은 비용으로 구축 가능하며, HuggingFace의 SO-100/SO-101 로봇이 이 설계를 계승합니다.

### 방식 4: 물리적 시연 (Kinesthetic Teaching)

조작자가 로봇 팔을 직접 손으로 잡고 원하는 동작을 수행합니다. 이때 로봇은 중력 보상 모드로 동작하여 관절 저항을 최소화합니다.

**장점**: 가장 직관적인 방법으로 별도 장치 없이 구현 가능합니다.

**단점**: 조작자가 로봇 팔의 무게를 직접 지지해야 하므로 피로가 빠릅니다. 미세하고 빠른 조작에는 부적합합니다.

## 시연 품질의 중요성

### 품질 vs 양의 트레이드오프

시연 데이터의 품질과 양 모두 중요하지만, 트레이드오프가 있습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│            시연 품질과 학습 성능의 관계                      │
│                                                              │
│  정책 성능 ▲                                                 │
│            │                    ● 고품질 소량               │
│            │               ●                                 │
│            │          ● 저품질 다량                         │
│            │     ●                                           │
│            └──────────────────────────────▶ 시연 수          │
│                                                              │
│  [결론] 고품질 100개 > 저품질 1,000개 (과제에 따라 다름)   │
└──────────────────────────────────────────────────────────────┘
```

**시연 품질에 영향을 주는 요소:**

- **일관성**: 동일 과제를 비슷한 방식으로 수행하는가?
- **완성도**: 과제를 끝까지 성공적으로 완료하는가?
- **스타일 다양성**: 하나의 해결책에 갇히지 않고 다양한 경로를 보여주는가?
- **조작자 전문성**: 인간도 처음 하는 과제는 품질이 낮습니다.

실용적으로는 **과제별 연습 단계**를 두어 조작자가 시스템에 익숙해진 후 기록을 시작하는 것이 효과적입니다.

## 데이터 포맷과 저장 구조

### 시연 데이터의 구성 요소

로봇 시연 데이터는 타임스텝마다 다음 정보를 포함합니다:

```text
타임스텝 t의 데이터:
├── observation (관측)
│   ├── images: {camera_0: [H,W,C], camera_1: [H,W,C], ...}
│   ├── joint_positions: [num_joints,]  # 관절 위치 (라디안)
│   ├── joint_velocities: [num_joints,]
│   └── gripper_state: [1,]  # 그리퍼 열림/닫힘
│
├── action (행동)
│   ├── joint_positions_target: [num_joints,]
│   └── gripper_command: [1,]
│
└── metadata
    ├── timestamp: float
    ├── episode_id: str
    └── task_description: str
```

### LeRobot 데이터셋 포맷

HuggingFace LeRobot은 표준화된 데이터셋 포맷을 제공합니다. 파케이(Parquet) 형식으로 고차원 데이터(이미지)와 저차원 데이터(관절 상태)를 효율적으로 저장합니다. 이 포맷은 8-3 챕터에서 더 자세히 다룹니다.

## 수집 비용과 효율화 전략

### 비용 구조

시연 데이터 수집 비용의 주요 구성:

| 비용 요소 | 내용 |
|-----------|------|
| 하드웨어 | 로봇, 원격조작 장치, 카메라, 컴퓨터 |
| 인력 | 조작자 시간, 데이터 검토 인력 |
| 공간 | 실험 공간, 물체/장면 준비 |
| 시간 | 시연당 수분 × 수백~수천 에피소드 |

### 효율화 전략

**1. 데이터 증강(Data Augmentation):**  
수집된 이미지에 색상 지터링, 랜덤 크롭, 좌우 반전 등을 적용하여 적은 데이터로 더 많은 학습 샘플을 확보합니다. 단, 물리적으로 불가능한 증강(중력 방향 뒤집기 등)은 피해야 합니다.

**2. 실패 에피소드 필터링:**  
자동화된 성공 감지(물체 위치 확인, 힘 센서)로 실패한 시연을 제거합니다. 실패 데이터를 포함하면 정책이 실패 패턴도 학습합니다.

**3. 크라우드소싱:**  
표준화된 원격조작 플랫폼을 통해 여러 조작자에게 분산하여 데이터를 수집합니다. 다양한 조작 스타일이 정책 일반화에 도움이 됩니다.

**4. 하이브리드 수집:**  
어려운 핵심 조작 부분만 인간이 시연하고, 나머지 이동이나 단순 구간은 자동화된 스크립트로 처리합니다.

## 시연 데이터 수집 파이프라인 예시

```text
┌──────────────────────────────────────────────────────────────┐
│          시연 데이터 수집 전체 파이프라인                    │
│                                                              │
│  1. 환경 초기화                                             │
│     └─ 물체 위치 무작위화, 카메라 확인                      │
│                  │                                           │
│                  ▼                                           │
│  2. 원격조작 시연 (인간 조작자)                             │
│     └─ 성공할 때까지 시도 (실패 시 재시작)                  │
│                  │                                           │
│                  ▼                                           │
│  3. 에피소드 검토                                           │
│     └─ 자동 성공 판별 + 선택적 수동 검토                    │
│                  │                                           │
│                  ▼                                           │
│  4. 데이터 저장                                             │
│     └─ LeRobot 포맷, 타임스탬프 동기화                      │
│                  │                                           │
│                  ▼                                           │
│  5. 데이터 검증                                             │
│     └─ 분포 확인, 이상 에피소드 제거                        │
└──────────────────────────────────────────────────────────────┘
```

## 정리

원격조작은 모방학습을 위한 시연 데이터를 수집하는 핵심 수단입니다. VR 기반, 마스터-슬레이브, 물리적 시연 등 방식마다 적합한 과제와 비용 구조가 다릅니다. ALOHA와 같은 저비용 양팔 시스템은 연구 접근성을 크게 높였습니다. 데이터 품질은 양 못지않게 중요하며, 체계적인 수집 파이프라인과 필터링이 필수입니다. 다음 챕터에서는 LeRobot 라이브러리를 통해 이 데이터를 어떻게 표준 포맷으로 관리하고 학습에 사용하는지 알아봅니다.
