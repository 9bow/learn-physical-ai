## 왜 대규모 데이터인가

자연어처리(NLP)에서 GPT 계열 모델이 보여준 것처럼, **데이터 규모와 다양성**은 모델의 일반화 능력에 결정적입니다. 수억 개의 문장으로 학습한 언어 모델은 학습에서 본 적 없는 새로운 질문에도 합리적으로 답할 수 있습니다.

로보틱스에서도 같은 질문이 대두됩니다: 다양한 로봇으로 다양한 과제를 수행한 대규모 데이터셋으로 학습하면, 새로운 로봇이나 새로운 과제에도 일반화되는 정책을 만들 수 있을까?

이 질문에 답하려는 협력 프로젝트가 **Open X-Embodiment**입니다.

## Open X-Embodiment 프로젝트

### 배경과 목표

Open X-Embodiment(개방형 다체화 데이터)는 Google DeepMind를 포함한 수십 개 연구 기관이 협력하여, 각자의 로봇 시연 데이터를 통합한 대규모 공개 데이터셋 프로젝트입니다. 2023년 공개된 초기 버전에는 다수의 기관이 수집한 데이터가 포함되었습니다.

**핵심 목표:**
- 다양한 로봇 플랫폼과 과제를 포괄하는 대규모 데이터셋 구축
- Cross-embodiment 학습 가능성 검증
- 커뮤니티가 함께 키워나가는 개방형 데이터 생태계 형성

```text
┌──────────────────────────────────────────────────────────────┐
│              Open X-Embodiment 데이터 구성                   │
│                                                              │

│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ Google   │  │ Stanford │  │  UC      │  │ Carnegie │   │
│  │ DeepMind │  │  HAI     │  │ Berkeley │  │  Mellon  │   │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘   │

│       ▼              ▼              ▼              ▼          │
```

### 통합 데이터 포맷: RLDS

Open X-Embodiment는 **RLDS(Reinforcement Learning Datasets)** 포맷을 사용합니다. RLDS는 에피소드 단위의 구조로, 각 에피소드는 스텝들의 시퀀스이며 각 스텝은 관측, 행동, 보상, 메타데이터를 포함합니다.

```python
# 개념 이해용 의사코드 - Open X-Embodiment 데이터 구조

episode = {
    "steps": [
        {
            "observation": {
                "image": <이미지 배열>,
                "wrist_image": <손목 카메라 이미지>,
                "state": <로봇 상태 벡터>,
            },
            "action": <행동 벡터>,
            "reward": <float>,
            "is_last": <bool>,
            "language_instruction": "pick up the red cup",
        },
        # ... 다음 스텝들
    ]
}
```

언어 지시(language instruction)가 각 에피소드에 포함된다는 점이 주목할 만합니다. 이는 언어 조건부(language-conditioned) 정책 학습을 가능하게 합니다.

## Cross-Embodiment 학습

### 개념

**Cross-embodiment 학습(다체화 교차 학습)** 은 서로 다른 로봇 형태(embodiment)의 데이터를 함께 학습하여, 특정 로봇에 종속되지 않는 일반 정책을 만들려는 시도입니다.

이 접근의 핵심 가정은: 다양한 로봇이 유사한 과제(물체 집기, 이동 등)를 수행할 때, 그 기저의 **과제 이해**(어떤 물체를 어디로 이동해야 하는가)는 공유될 수 있다는 것입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│           Cross-Embodiment 학습의 직관                       │
│                                                              │
│  로봇 A (6DOF 단팔)  ─▶ "빨간 컵을 집어서 왼쪽 선반에 놓아"│
│                              │                               │
│  로봇 B (이동 + 팔) ─▶       │ 공통 과제 표현              │
│                              │ (언어 + 시각 이해)           │
│  로봇 C (양팔)      ─▶       │                              │
│                              ▼                               │
│                         공유 특성 표현 학습                  │
│                              │                               │
│                              ▼                               │
│                    새 로봇에 적용 시 일반화?                 │
└──────────────────────────────────────────────────────────────┘
```

### 도전 과제

Cross-embodiment 학습은 여러 어려운 문제를 동반합니다:

**행동 공간의 이질성**: 로봇마다 관절 수와 종류가 다릅니다. 6DOF 팔과 7DOF 팔의 행동 벡터를 어떻게 통합할까요?

**관측 공간의 이질성**: 어떤 로봇은 손목 카메라가 있고, 어떤 로봇은 외부 카메라만 있습니다. 이미지 해상도와 위치도 다릅니다.

**데이터 품질 불균형**: 기관마다 수집 방법, 시연 품질, 데이터 양이 다릅니다.

이런 이질성을 처리하기 위해 RT-X 모델은 특별한 설계를 채택했습니다.

## RT-X 모델

**RT-X(Robotics Transformer X)**는 Open X-Embodiment 데이터로 학습된 대규모 범용 로봇 정책 모델입니다. "X"는 cross-embodiment를 의미합니다.

### RT-X의 핵심 설계

**로봇 유형 토큰**: 입력에 현재 로봇의 유형 정보를 토큰으로 포함시켜, 같은 모델이 다른 로봇들의 데이터를 구분하여 처리할 수 있게 합니다.

**언어 조건부 정책**: 자연어 지시를 조건으로 받아, 같은 시각 관측에서도 "집어"와 "밀어" 같이 다른 지시에 다른 행동을 생성합니다.

**행동 토큰화**: 연속적인 행동 벡터를 이산 토큰으로 변환하여 Transformer의 언어 토큰과 동일한 형식으로 처리합니다.

```text
RT-X 입력/출력 구조 (개념):

입력:
  [로봇 유형 토큰] + [언어 지시 토큰] + [이미지 토큰] + [상태 토큰]
                          │
                          ▼
              Transformer 기반 정책 모델
                          │
                          ▼
출력:
  [행동 토큰 시퀀스] → 디토큰화 → 관절 명령
```

### RT-X가 보여준 것

Open X-Embodiment 논문에서 보고된 주요 발견:

1. **데이터 혼합이 도움이 된다**: 여러 로봇의 데이터를 혼합하여 학습하면, 단일 로봇 데이터만으로 학습하는 것보다 해당 로봇에서도 더 좋은 성능을 보이는 경향이 있었습니다.

2. **새로운 로봇으로의 이전**: 학습 데이터에 없던 새로운 로봇 유형에 소량의 데이터로 파인튜닝했을 때 빠른 적응이 가능했습니다.

3. **언어 일반화**: 학습 중 보지 않은 언어 지시에도 합리적인 행동을 생성하는 경향이 나타났습니다.

단, 이런 결과들은 특정 실험 조건에서의 관찰이며, 일반적인 cross-embodiment 학습의 완전한 성공을 의미하지는 않습니다. 연구는 여전히 활발히 진행 중입니다.

## 데이터 규모와 다양성의 중요성

### 데이터 스케일링 법칙 (로보틱스에서)

NLP에서 확립된 스케일링 법칙(더 많은 데이터 + 더 큰 모델 = 더 좋은 성능)이 로보틱스에서도 성립하는지는 활발히 연구 중입니다. 로보틱스는 몇 가지 고유한 도전이 있습니다:

```text
┌──────────────────────────────────────────────────────────────┐
│         NLP vs 로보틱스 데이터 스케일링 비교                 │
├──────────────────────┬──────────────────────────────────────┤
│ 항목                 │ NLP                │ 로보틱스         │
├──────────────────────┼────────────────────┼──────────────────┤
│ 데이터 수집 비용     │ 저렴 (텍스트 크롤) │ 높음 (로봇 운용) │
│ 데이터 다양성        │ 자연어로 통일      │ 로봇마다 이질적  │
│ 실패 데이터 처리     │ 오답도 텍스트      │ 실패 = 위험/비용 │
│ 시뮬레이션 활용      │ 불필요             │ 보완 수단으로 활용│
└──────────────────────┴────────────────────┴──────────────────┘
```

이런 차이 때문에 로보틱스에서는 순수 데이터 스케일링보다 **데이터 품질, 다양성, 시뮬레이션 활용**의 조합이 더 중요하다는 시각이 있습니다.

### 데이터 다양성의 역할

Open X-Embodiment 실험들은 단순히 같은 과제의 데이터를 더 많이 수집하는 것보다, **다양한 과제와 환경**의 데이터를 포함하는 것이 일반화에 더 도움이 된다는 경향을 보였습니다. 이는 언어 모델에서 다양한 도메인의 텍스트가 일반 능력을 키우는 것과 유사합니다.

## 오픈 데이터 생태계의 현황

Open X-Embodiment는 하나의 시작점입니다. 이후 다양한 데이터셋 공유 이니셔티브가 등장했습니다:

- **LeRobot Hub**: HuggingFace에서 커뮤니티가 기여하는 소규모 데이터셋
- **DROID**: 다양한 환경에서 수집된 대규모 단팔 조작 데이터셋
- **BridgeData**: 다양한 주방 환경에서의 조작 데이터

이 생태계는 계속 성장 중이며, 로봇 학습 연구의 방향을 "개별 연구실의 데이터"에서 "커뮤니티 공유 데이터"로 전환하는 중입니다.

## 정리

Open X-Embodiment는 수십 개 기관의 다양한 로봇 데이터를 통합하여 cross-embodiment 학습의 가능성을 탐구했습니다. RT-X 모델은 이 다양한 데이터로 학습하여 데이터 혼합의 이점과 새로운 로봇으로의 빠른 적응 가능성을 보여주었습니다. 데이터 스케일과 다양성은 로봇 정책의 일반화 능력에 중요하지만, 수집 비용과 이질성 때문에 NLP만큼의 빠른 스케일링은 아직 도전적입니다. 오픈 데이터 생태계의 성장은 이 도전을 커뮤니티 힘으로 극복하려는 시도입니다.
