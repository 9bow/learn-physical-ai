## 왜 Diffusion Policy인가

Behavior Cloning에서 가장 다루기 어려운 문제 중 하나는 **다중 모드 행동 분포(multimodal action distribution)** 입니다. 동일한 상황에서 전문가가 여러 다른 방식으로 행동할 수 있을 때, 단순 평균을 예측하는 정책은 어느 모드도 제대로 수행하지 못합니다.

예를 들어, 물체를 왼쪽 또는 오른쪽으로 돌아가 집을 수 있는 상황에서 시연 데이터의 절반은 왼쪽, 절반은 오른쪽 경로를 포함한다면, 평균을 예측하는 BC 정책은 두 경로의 중간—물체를 향해 직진하는 불가능한 경로—를 생성할 수 있습니다.

**Diffusion Policy(확산 정책)** 는 이 문제를 해결하기 위해 확산 모델(diffusion model)의 강력한 생성 능력을 로봇 정책 학습에 적용합니다.

## 확산 과정의 직관

확산 모델은 이미지 생성(DALL-E, Stable Diffusion)에서 이미 그 능력을 입증했습니다. Diffusion Policy는 같은 원리를 **행동 시퀀스 생성**에 적용합니다.

### 핵심 아이디어: Denoising으로 행동 생성

확산 모델의 작동 원리는 두 단계입니다:

**학습(Forward process - 노이즈 추가):**  
깨끗한 행동 시퀀스에 단계적으로 가우시안 노이즈를 추가하여, 최종적으로 완전한 노이즈 상태로 변환하는 과정을 학습합니다.

**추론(Reverse process - 노이즈 제거):**  
순수 노이즈에서 시작하여, 관측 조건(현재 이미지 + 로봇 상태)을 바탕으로 단계적으로 노이즈를 제거(denoising)하여 깨끗한 행동 시퀀스를 생성합니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│              Diffusion Policy 생성 과정                          │
│                                                                  │
│  학습 시 (Forward Process):                                      │
│  깨끗한 행동 ──노이즈 추가──▶ 노이즈 행동 (타임스텝 T개)       │
│                                                                  │
│  추론 시 (Reverse Process):                                      │
│  순수 노이즈                                                     │
│      │  (관측: 이미지 + 로봇 상태 조건 적용)                    │
│      ▼                                                           │
│  노이즈 조금 제거 (스텝 T)                                       │
│      │                                                           │
│      ▼                                                           │
│  노이즈 조금 더 제거 (스텝 T-1)                                  │
│      │                                                           │
│      ▼  ... (반복, 보통 10~100 스텝)                             │
│      │                                                           │
│      ▼                                                           │
│  깨끗한 행동 시퀀스 (미래 N 스텝)                               │
└──────────────────────────────────────────────────────────────────┘
```

학습하는 신경망(**Noise Prediction Network 또는 Score Network**)은 노이즈가 얼마나 추가되었는지를 예측하거나, 현재 상태에서 노이즈를 제거하는 방향을 예측합니다. 이 네트워크가 관측(이미지, 로봇 상태)을 조건으로 받아, 관측에 맞는 적절한 행동 방향으로 denoising을 안내합니다.

## Multimodal 행동 분포 처리

Diffusion Policy가 Behavior Cloning(단순 MSE)에 비해 탁월한 이유는 **확률 분포 표현 능력** 때문입니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│         Multimodal 분포 처리 비교                                │
│                                                                  │
│  상황: 물체를 집을 때 왼쪽 또는 오른쪽 두 경로 존재             │
│                                                                  │
│  시연 데이터 분포:                                               │
│         왼쪽 경로           오른쪽 경로                         │
│    ████████████               ████████████                      │
│        (모드 1)                   (모드 2)                       │
│                                                                  │
│  일반 BC (MSE 예측):                                             │
│         ● (두 모드의 평균 → 두 모드 어느 쪽도 아님)             │
│                                                                  │
│  Diffusion Policy:                                               │
│    ●(왼쪽)  또는  ●(오른쪽)  (두 모드 중 하나를 선택적 생성)   │
└──────────────────────────────────────────────────────────────────┘
```

확산 모델은 복잡한 다중 모드 분포를 학습하는 데 이론적으로 강합니다. 노이즈 제거 과정에서 무작위성이 개입하므로, 같은 관측에서도 다른 실행마다 다른 모드의 행동을 생성할 수 있습니다.

## 아키텍처: U-Net과 Transformer 기반

Diffusion Policy의 노이즈 예측 네트워크는 두 가지 아키텍처로 구현됩니다:

### CNN-based (U-Net)

1D U-Net을 행동 시퀀스의 시간 차원에 적용합니다. 이미지 관측은 별도의 인코더로 처리한 후 조건으로 주입됩니다.

```python
# 개념 이해용 의사코드 - Diffusion Policy 추론

def generate_action(policy, observation, num_diffusion_steps=100):
    """
    observation: {
        "image": 이미지 텐서,
        "agent_pos": 로봇 상태
    }
    """
    # 관측 인코딩
    obs_cond = policy.encode_observation(observation)
    
    # 순수 노이즈에서 시작
    action = torch.randn(action_horizon, action_dim)
    
    # 단계적 denoising
    for t in reversed(range(num_diffusion_steps)):
        # 현재 노이즈 상태와 관측을 보고 노이즈 예측
        noise_pred = policy.noise_pred_net(
            noisy_action=action,
            timestep=t,
            obs_cond=obs_cond
        )
        
        # DDPM/DDIM 스케줄러로 노이즈 제거
        action = scheduler.step(noise_pred, t, action).prev_sample
    
    return action  # 생성된 행동 시퀀스
```

### Transformer-based

U-Net 대신 Transformer를 노이즈 예측 네트워크로 사용합니다. 더 긴 시간 범위의 의존성을 포착하는 데 유리합니다.

## ACT와 Diffusion Policy 비교

두 알고리즘 모두 현대 로봇 모방학습의 대표 방법이지만, 설계 철학과 적합한 상황이 다릅니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│               ACT vs Diffusion Policy 비교                       │
├──────────────────────┬────────────────────┬──────────────────────┤
│ 항목                 │ ACT                │ Diffusion Policy     │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 핵심 아이디어        │ 행동 청크 예측     │ Denoising으로 생성   │
│                      │ (VAE 기반)         │ (확산 과정)          │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 행동 분포 표현       │ 제한적(단일 모드)  │ 강함(다중 모드)      │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 추론 속도            │ 빠름 (단일 패스)   │ 느림 (반복 denoising)│
├──────────────────────┼────────────────────┼──────────────────────┤
│ 학습 복잡도          │ 비교적 단순        │ 높음                 │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 고주파 제어 적합성   │ 높음               │ 낮음 (지연 이슈)     │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 다중 모드 과제       │ 어려움             │ 자연스러움           │
├──────────────────────┼────────────────────┼──────────────────────┤
│ 하이퍼파라미터       │ 비교적 적음        │ 확산 스텝, 스케줄러  │
│ 민감도               │                    │ 등 추가 파라미터     │
└──────────────────────┴────────────────────┴──────────────────────┘
```

### ACT의 장점

ACT는 VAE(Variational Autoencoder) 기반으로 행동 청크를 예측합니다. 단일 패스 추론이므로 추론이 빠르고, 고주파(200Hz+) 제어에도 적합합니다. 비교적 구현이 단순하고 하이퍼파라미터가 적습니다.

**적합한 과제**: 행동 다양성이 제한적이고 빠른 반응 속도가 중요한 정밀 조작.

### Diffusion Policy의 장점

확산 모델의 강력한 분포 표현 능력으로 복잡한 다중 모드 행동을 표현합니다. 이미지 생성에서 검증된 아키텍처와 학습 기법을 직접 활용할 수 있습니다.

**적합한 과제**: 행동 다양성이 크거나 여러 해결책이 존재하는 과제, 추론 속도 제약이 덜 엄격한 과제.

## 추론 속도 문제와 해결책

Diffusion Policy의 주요 단점은 추론 시간입니다. 100 denoising 스텝이 필요하다면 10ms 추론 시간이 100배 늘어납니다. 이는 고속 로봇 제어에서 문제가 됩니다.

### DDIM (Denoising Diffusion Implicit Models)

DDIM은 확산 스텝 수를 크게 줄여도(100 → 10 스텝) 품질 저하를 최소화하는 기법입니다. Diffusion Policy도 DDIM을 적용하면 추론 속도를 크게 높일 수 있습니다.

### Consistency Policy

최근 연구에서 확산 모델의 생성 품질을 유지하면서 단 1~2 스텝으로 추론하는 Consistency Policy 등 새로운 접근이 등장하고 있습니다. 이 분야는 빠르게 발전 중입니다.

## Diffusion Policy의 성능

원본 Diffusion Policy 논문(Chi et al., 2023)에서는 여러 조작 과제에서 ACT, LSTM-GMM 등 기존 모방학습 알고리즘 대비 성능 우위를 보였습니다. 특히 다중 모드 행동이 포함된 과제에서 차이가 두드러졌습니다.

LeRobot 라이브러리는 Diffusion Policy의 표준 구현을 포함하고 있어, 8-3 챕터에서 소개한 워크플로와 동일하게 사용할 수 있습니다.

## 정리

Diffusion Policy는 확산 모델의 강력한 확률 분포 표현 능력을 로봇 정책 학습에 적용하여, 특히 다중 모드 행동 분포를 자연스럽게 처리합니다. 순수 노이즈에서 시작하여 관측 조건으로 안내되는 반복적 denoising으로 행동 시퀀스를 생성합니다. ACT와 비교했을 때 다중 모드 표현 능력이 강하지만 추론 속도가 느린 트레이드오프가 있습니다. 두 방법 모두 LeRobot 생태계에서 바로 사용할 수 있으며, 과제 특성에 따라 적합한 방법을 선택하는 것이 중요합니다.
