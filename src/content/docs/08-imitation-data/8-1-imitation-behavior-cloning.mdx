## 모방학습이란

로봇에게 복잡한 행동을 가르치는 방법은 크게 두 가지입니다. 하나는 **강화학습(Reinforcement Learning, RL)** 처럼 보상 신호를 통해 시행착오로 배우는 것이고, 다른 하나는 **모방학습(Imitation Learning, IL)** 처럼 전문가의 시연을 보고 그 행동을 따라 배우는 것입니다.

모방학습의 기본 아이디어는 직관적입니다. 사람이 로봇 팔을 원격으로 조종하거나 직접 움직여 물체를 집는 과정을 보여주면, 로봇은 그 "어떻게 하는가"를 학습합니다. 이는 보상 함수를 설계할 필요가 없고, 학습 데이터가 물리적으로 의미 있는 행동으로 구성된다는 장점이 있습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              강화학습 vs 모방학습 비교                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ 강화학습:                                                    │
│   환경 ─▶ 보상 신호 ─▶ 정책 개선 (시행착오)               │
│                                                              │
│ 모방학습:                                                    │
│   전문가 시연 ─▶ (상태, 행동) 쌍 ─▶ 정책 학습 (지도)     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## Behavior Cloning: 지도학습으로서의 모방

**Behavior Cloning(행동 복제, BC)** 은 모방학습의 가장 단순한 형태입니다. 전문가의 시연 데이터 — (관측, 행동) 쌍의 집합 — 를 지도학습으로 학습하여, 주어진 관측에서 전문가와 동일한 행동을 예측하는 정책을 만듭니다.

학습 목표는 단순합니다: 전문가가 상태 s에서 행동 a를 취했다면, 정책도 상태 s에서 행동 a를 출력하도록 훈련합니다.

```python
# 개념 이해용 의사코드 - Behavior Cloning 학습 루프

def train_bc(policy, demonstrations, num_epochs):
    """
    demonstrations: [(obs_1, action_1), (obs_2, action_2), ...] 시연 데이터
    """
    optimizer = Adam(policy.parameters())
    
    for epoch in range(num_epochs):
        for obs_batch, action_batch in DataLoader(demonstrations):
            predicted_actions = policy(obs_batch)
            
            # 전문가 행동과의 차이를 최소화
            loss = mean_squared_error(predicted_actions, action_batch)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
    
    return policy
```

BC의 매력은 구현 단순성입니다. 표준 지도학습 파이프라인과 동일하므로, 기존 딥러닝 도구를 그대로 활용할 수 있습니다.

## BC의 핵심 문제: Covariate Shift

그러나 BC는 실전에서 심각한 한계를 보입니다. 그 핵심은 **Covariate Shift(공변량 이동)** 문제입니다.

### 직관적 설명

BC 학습 시 정책은 **전문가가 방문한 상태들**만 관측합니다. 전문가는 실수 없이 최적 경로를 따라 이동하므로, 학습 데이터에는 "실수 후의 상태"가 거의 없습니다.

그런데 실제 배포 시 정책이 조금이라도 실수하면, 이전에 학습 중 본 적 없는 새로운 상태에 도달합니다. 이런 낯선 상태에서의 행동은 학습 데이터에 포함되지 않았으므로, 정책은 무엇을 해야 할지 모르고 더 큰 실수를 저지릅니다.

```text
┌──────────────────────────────────────────────────────────────┐
│           Covariate Shift 시각화                             │
│                                                              │
│  목표 ◀══════════════════════════ 시작                      │
│    ▲                                                         │
│    │  전문가 경로: ──────────────────────────▶              │
│    │                                                         │
│    │  BC 정책:    ────────╲                                  │
│    │                       ╲ 작은 오차                       │
│    │                        ╲                               │
│    │                         ╲───────── 낯선 상태들          │
│    │                                    (학습 데이터 없음)    │
│    │                                    ↓ 더 큰 오차         │
│    │                                    ↓↓ 발산             │
└──────────────────────────────────────────────────────────────┘
```

### Compounding Error (누적 오차)

Covariate shift에서 직접 파생되는 문제가 **Compounding Error(누적 오차)** 입니다. 작은 오차가 다음 상태를 악화시키고, 그 악화된 상태에서의 오차가 또 다음 상태를 더 악화시키는 연쇄 반응입니다.

이론적으로 BC 정책의 오차는 최악의 경우 에피소드 길이 T에 대해 O(T²)까지 누적될 수 있음이 알려져 있습니다(Ross &amp; Bagnell). 이는 오픈 루프 제어에서의 오차 누적과 유사합니다. 짧은 과제(물체 집기 한 번)에서는 문제가 적지만, 긴 과제(다단계 조립)에서는 치명적입니다.

## DAgger: 데이터셋 집합의 대화식 확장

**DAgger(Dataset Aggregation)** 는 Ross et al. (2011)이 제안한 알고리즘으로, covariate shift 문제를 직접 해결합니다.

핵심 아이디어: **학습 중 정책이 방문하는 상태에서 전문가에게 레이블을 수집**합니다. 즉, 정책이 실수해서 이상한 상태에 가더라도, 그 상태에서 전문가가 어떻게 해야 하는지 물어보고 그걸 학습 데이터에 추가합니다.

```text
DAgger 알고리즘 (개념 이해용 의사코드):

데이터셋 D = 초기 전문가 시연으로 초기화
정책 π_0 = BC(D)

반복 (i = 1, 2, ...):
  1. 현재 정책 π_i-1 로 환경 탐색 → 방문 상태들 수집
  2. 방문 상태들에 대해 전문가에게 행동 레이블 수집
  3. D = D ∪ {새로 수집된 (상태, 전문가 행동) 쌍}
  4. π_i = BC(D)  # 전체 데이터셋으로 재학습
```

DAgger는 이론적으로 covariate shift를 해결하지만, 실용적 한계가 있습니다. 정책이 방문하는 모든 이상한 상태에서 **실시간으로 전문가 레이블**이 필요합니다. 실제 로봇에서는 전문가 피로, 레이블링 일관성, 위험 상황 노출 등의 문제가 생깁니다.

이 때문에 실용적 구현에서는 DAgger보다 더 많은 다양한 초기 시연 수집, 데이터 증강, 또는 하이브리드 접근법(BC + RL fine-tuning)이 선호됩니다.

## 모방학습 vs 강화학습 비교

| 항목 | Behavior Cloning | 강화학습 (RL) |
|------|-----------------|--------------|
| 보상 함수 필요 | 불필요 | 필수 |
| 데이터 요구 | 전문가 시연 데이터 | 환경 상호작용 |
| 학습 속도 | 빠름 (지도학습) | 느림 (탐색 필요) |
| Covariate shift | 심각한 문제 | 없음 (자체 방문) |
| 성능 상한 | 전문가 수준 | 이론적 최적 |
| 희소 보상 처리 | 문제없음 | 어려움 |
| 다중 모드 행동 | 처리 어려움 | 자연스럽게 처리 |

### 모방학습의 장점

**보상 함수 설계 불필요**: RL에서 가장 어려운 문제 중 하나가 좋은 보상 함수 설계입니다. 모방학습은 이 문제를 우회합니다.

**안전한 학습**: 실제 로봇에서 RL로 학습하면 수많은 실패 시도가 필요합니다. BC는 전문가 시연에서 배우므로 실제 로봇 실행이 훨씬 적습니다.

**빠른 수렴**: 시연 데이터가 충분하면 RL보다 훨씬 빠르게 유용한 정책을 얻을 수 있습니다.

### 모방학습의 한계

**전문가 의존**: 데이터 품질과 양이 전문가의 능력과 수고에 제한됩니다.

**성능 상한**: BC 정책은 전문가 수준을 넘기 어렵습니다.

**분포 외 처리**: 학습 중 본 적 없는 상태에서 취약합니다.

## 실용적 시사점

현대 로보틱스에서는 순수 BC와 순수 RL 중 하나를 선택하기보다, **BC로 초기화하고 RL로 개선**하는 하이브리드 접근이 효과적입니다. BC가 제공하는 "합리적인 시작점"은 RL의 탐색 효율을 크게 높입니다.

또한 Diffusion Policy, ACT(Action Chunking Transformer)와 같은 최신 방법들은 BC의 기본 틀을 유지하면서, **다중 모드 행동 분포**와 **시간적 일관성** 문제를 해결합니다. 이들은 8-5 챕터에서 상세히 다룹니다.
