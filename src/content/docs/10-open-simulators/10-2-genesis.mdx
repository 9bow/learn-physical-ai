## Genesis란

**Genesis**는 2024년 말 공개된 범용 물리 시뮬레이션 플랫폼입니다. Genesis-Embodied-AI 팀이 주도하여 개발했으며, **로봇공학, 체화 AI, 물리 AI** 연구를 위한 통합 시뮬레이션 환경을 목표로 합니다.

Genesis의 핵심 설계 철학은 **하나의 플랫폼에서 다양한 물리 현상을 통합**하는 것입니다. 기존 시뮬레이터들이 강체(rigid body), 유연체(soft body), 유체(fluid) 중 일부에 특화된 것과 달리, Genesis는 이들을 단일 프레임워크에서 처리하는 것을 목표로 합니다.

> **주의:** Genesis는 2024년에 공개된 신생 프로젝트입니다. 공식 GitHub 저장소와 문서를 기준으로 기술하며, 외부에서 인용된 성능 수치나 기능 주장은 검증이 어려울 수 있으므로 비판적으로 해석해야 합니다.

## 주요 특징

### 다중 물리 솔버

Genesis는 용도에 맞는 물리 솔버를 선택하여 사용할 수 있습니다.

- **강체 솔버(Rigid Body Solver)**: 로봇 팔, 이동 로봇 등 전통적 로봇공학 시뮬레이션
- **MPM(Material Point Method)**: 모래, 진흙, 눈 같은 입자-연속체 전환 물질
- **SPH(Smoothed Particle Hydrodynamics)**: 유체 시뮬레이션
- **FEM(Finite Element Method)**: 변형 가능한 고체
- **PBD(Position-Based Dynamics)**: 천, 로프 등 실시간 연성체

이 다양성은 단순한 로봇 팔 조작을 넘어, 요리(유체 혼합), 접기(천 조작), 건설(토양 파기) 같은 복잡한 과제 시뮬레이션에 잠재력을 부여합니다.

### Python-first API

```python
# 개념 이해용 의사코드 — Genesis API (공식 문서 기반, 변경될 수 있음)
import genesis as gs

# 씬 생성
gs.init(backend=gs.cuda)  # GPU 백엔드 선택
scene = gs.Scene(
    viewer_options=gs.options.ViewerOptions(
        camera_pos=(0, -3.5, 2.5),
        camera_lookat=(0.0, 0.0, 0.5),
    )
)

# 평면 추가
plane = scene.add_entity(gs.morphs.Plane())

# 로봇 로드 (URDF)
robot = scene.add_entity(
    gs.morphs.URDF(file="franka_panda/panda.urdf"),
)

# 씬 빌드 및 실행
scene.build()
for i in range(1000):
    scene.step()
```

### GPU 병렬 환경

Genesis는 GPU 위에서 수천 개의 환경을 동시에 실행하는 것을 핵심 기능으로 내세웁니다. 강화학습에서 경험 수집 속도가 학습 효율을 크게 좌우하기 때문에, 이는 실질적인 이점입니다.

## 시각화 기능

Genesis는 내장 렌더러를 통해 시뮬레이션 결과를 시각화합니다. Rasterization 기반과 레이트레이싱(ray tracing) 기반 렌더링을 선택할 수 있으며, 카메라 센서 출력(RGB, 깊이)을 직접 학습 데이터로 사용할 수 있습니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│            Genesis 렌더링 파이프라인                            │
│                                                                 │
│  물리 시뮬레이션                                                │
│  (강체 + 유연체 + 유체)                                         │
│       │                                                         │
│       ▼                                                         │
│  렌더러 선택                                                    │
│  ├─ Rasterization: 빠름, 낮은 사실감                           │
│  └─ Ray Tracing:   느림, 높은 사실감                           │
│       │                                                         │
│       ▼                                                         │
│  카메라 출력                                                    │
│  ├─ RGB 이미지                                                  │
│  ├─ 깊이 맵 (Depth Map)                                        │
│  └─ 세그멘테이션 마스크                                        │
└─────────────────────────────────────────────────────────────────┘
```

## 현재 상태와 한계

Genesis는 유망한 방향성을 가진 프로젝트이지만, 신생 프로젝트로서 다음 한계를 인식하는 것이 중요합니다.

**성숙도 문제:**
- MuJoCo나 Isaac Sim에 비해 커뮤니티가 작고 생태계가 덜 발달했습니다.
- 문서와 예제가 아직 완전하지 않을 수 있습니다.
- API가 빠르게 변화하여 이전 버전 코드가 깨질 수 있습니다.

**검증 부족:**
- 물리 정확성, 시뮬레이션 속도 등의 주장을 독립적으로 검증한 연구가 아직 제한적입니다.
- 상용 프로젝트나 중요 연구에 사용하기 전에 충분한 자체 검증이 필요합니다.

**생성형 기능 미성숙:**
- 텍스트에서 장면이나 로봇을 자동 생성하는 기능은 초기 단계입니다.

## 어떤 상황에 Genesis가 적합한가

현 시점에서 Genesis 선택이 합리적인 경우:

- 강체와 유연체·유체를 **함께** 다루는 과제 시뮬레이션이 필요할 때
- 새로운 물리 현상(천, 진흙, 모래)을 로봇 학습에 통합하려는 연구
- 최신 기술을 탐색하는 연구 프로토타이핑

반면 안정성과 생태계가 중요한 프로덕션 수준 프로젝트에서는 MuJoCo, Isaac Sim, SAPIEN 같은 더 성숙한 플랫폼을 우선 고려하는 것이 현실적입니다.

다음 챕터에서는 매니퓰레이션(manipulation) 학습에 특화된 SAPIEN과 ManiSkill을 살펴봅니다.
