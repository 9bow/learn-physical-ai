## 의사결정 트리

프로젝트 요구사항에 따른 시뮬레이터 선택 가이드입니다.

```text
시뮬레이터 선택 의사결정 트리
═══════════════════════════════

프로젝트 시작
    │
    ▼
[질문 1] ROS 2 통합 기반 시스템 테스트가 주목적인가?
    ├─ YES ──▶  Gazebo + ros_gz_bridge
    └─ NO  ──▶ [질문 2]

[질문 2] NVIDIA GPU가 있고 포토리얼리스틱 렌더링이 필요한가?
    ├─ YES ──▶  Isaac Sim / Isaac Lab
    └─ NO  ──▶ [질문 3]

[질문 3] 대규모 GPU 병렬 강화학습이 핵심인가?
    ├─ YES ──▶ [질문 4]
    └─ NO  ──▶ [질문 7]

[질문 4] 관절 물체(서랍, 문 등) 조작이 주 과제인가?
    ├─ YES ──▶  SAPIEN / ManiSkill
    └─ NO  ──▶ [질문 5]

[질문 5] 접촉 정확도가 매우 중요한가 (정밀 조작)?
    ├─ YES ──▶  MuJoCo + MJX (MuJoCo Playground)
    └─ NO  ──▶ [질문 6]

[질문 6] 유연체·유체 등 비강체 물리가 필요한가?
    ├─ YES ──▶  Genesis (신생, 검증 필요)
    └─ NO  ──▶  MuJoCo (가장 안정적)

[질문 7] 표준 모방학습 벤치마크 비교가 목적인가?
    ├─ YES ──▶  robosuite + RoboMimic
    └─ NO  ──▶  MuJoCo (연구 기본값)
```

## 병용 시나리오

현실 프로젝트에서는 단일 시뮬레이터만 사용하는 경우가 드뭅니다. 각 시뮬레이터의 강점을 조합하는 병용 전략이 일반적입니다.

### 시나리오 1: VLA 개발 파이프라인

```text
Isaac Sim         ──▶  포토리얼리스틱 시각 데이터 생성
    +                  도메인 무작위화
MuJoCo / MJX     ──▶  대규모 RL로 기본 정책 학습
    +
실제 로봇         ──▶  최종 검증·파인튜닝
```

### 시나리오 2: 매니퓰레이션 연구

```text
ManiSkill         ──▶  GPU 병렬 환경에서 RL 학습
    +
robosuite         ──▶  표준 벤치마크 비교
    +
Gazebo + ROS 2    ──▶  MoveIt 2 통합 테스트
```

### 시나리오 3: 이동 로봇 (Locomotion)

```text
MuJoCo Playground ──▶  사족 보행 RL 학습 (병렬)
    +
Isaac Lab         ──▶  지형 다양화, 시각 관찰 추가
    +
실제 로봇         ──▶  Sim-to-Real 전이
```

## 실용적 추천

학습 단계별로 다음 순서를 추천합니다.

**입문:** MuJoCo + Gymnasium 표준 환경(CartPole, HalfCheetah 등)으로 RL 기초 익히기.

**조작 학습 입문:** robosuite의 Lift 과제 + RoboMimic 행동 복제 파이프라인.

**대규모 RL 실험:** ManiSkill3의 GPU 병렬 환경으로 성능 스케일링.

**시각 기반 Sim-to-Real:** NVIDIA GPU가 있다면 Isaac Sim/Lab 도전.

**ROS 2 통합:** 최종 시스템을 Gazebo + ROS 2에서 통합 테스트.

## 정리

시뮬레이터는 도구입니다. 가장 유명하거나 가장 새로운 것이 최선이 아닙니다. 프로젝트의 **핵심 요구사항**(GPU 병렬화, 접촉 정확도, ROS 통합, 시각 품질)을 먼저 정의하고, 그에 맞는 도구를 선택하는 것이 현실적입니다.

그리고 대부분의 현실 프로젝트는 단일 시뮬레이터로 끝나지 않습니다. 각 단계마다 적합한 도구를 활용하는 **멀티 시뮬레이터 워크플로**가 Physical AI 개발의 실상에 가깝습니다.
