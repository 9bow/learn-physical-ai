## robosuite란

**robosuite**는 Stanford University와 UT Austin의 ARISE Initiative가 개발한 **MuJoCo 기반 로봇 매니퓰레이션 시뮬레이션 프레임워크**입니다. 강화학습과 모방학습 연구를 위한 표준 환경과 도구를 제공합니다.

robosuite의 핵심 설계 원칙은 **모듈성(modularity)** 입니다. 로봇, 컨트롤러, 과제, 물체, 카메라를 독립적으로 교체·조합할 수 있어 다양한 실험 설정을 유연하게 구성합니다.


### 표준 과제

robosuite는 매니퓰레이션 연구에서 반복적으로 사용되는 표준 과제 모음을 제공합니다.

- **Lift**: 빨간 큐브를 특정 높이까지 들어올리기 (가장 기본)
- **Stack**: 큐브 위에 다른 큐브 쌓기
- **NutAssembly**: 사각/원형 나사를 막대에 끼우기
- **PickPlaceCan**: 캔 집어 컨테이너에 놓기
- **Door**: 문 손잡이 잡아 열기
- **Wipe**: 테이블 표면 닦기 (힘 제어 필요)
- **TwoArmHandover**: 두 로봇 팔 간 물체 전달

### 컨트롤러 시스템

robosuite는 다양한 저수준 컨트롤러를 제공합니다.

```python
# 개념 이해용 의사코드 — robosuite API
import robosuite as suite
from robosuite.controllers import load_controller_config

# 엔드이펙터 위치 제어 설정
controller_config = load_controller_config(
    default_controller="OSC_POSE"  # Operational Space Control (robosuite 1.5+에서는 load_composite_controller_config 사용)
)

# 환경 생성
env = suite.make(
    env_name="Lift",
    robots="Panda",
    controller_configs=controller_config,
    has_offscreen_renderer=True,
    use_camera_obs=True,
    camera_names=["frontview", "agentview"],
)

obs = env.reset()
for _ in range(500):
    action = policy(obs)  # (7,): 엔드이펙터 6DoF + 그리퍼
    obs, reward, done, info = env.step(action)
    if done:
        break
```

지원하는 컨트롤러 유형:
- **OSC_POSE**: 작업 공간 위치·자세 제어 (가장 직관적)
- **OSC_POSITION**: 위치만 제어
- **IK_POSE**: 역운동학 기반
- **JOINT_POSITION / JOINT_VELOCITY / JOINT_TORQUE**: 관절 공간 제어

## RoboMimic: 모방학습 프레임워크

**RoboMimic**은 ARISE Initiative가 개발한 **로봇 모방학습 연구 프레임워크**입니다. robosuite와 함께 설계되어 데이터 수집부터 학습, 평가까지 일관된 파이프라인을 제공합니다.

### 제공 알고리즘

RoboMimic은 주요 모방학습 알고리즘을 직접 구현하여 제공합니다.

| 알고리즘 | 설명 |
|---------|------|
| BC (Behavior Cloning) | 지도학습 기반 행동 복제 |
| BC-RNN | 순환 신경망 기반 BC (시간 의존성 처리) |
| IQL (Implicit Q-Learning) | 오프라인 강화학습 |
| CQL (Conservative Q-Learning) | 보수적 Q-learning |
| HBC (Hierarchical BC) | 계층적 행동 복제 |
| IRIS | 모방학습 + 강화학습 결합 |

### 표준 데이터셋

RoboMimic은 다양한 품질의 시연 데이터셋을 공개합니다. 특히 **다양한 인간 조작자 품질**을 포함하는 데이터셋이 특징입니다.

- **Proficient-Human (PH)**: 숙련된 조작자 시연
- **Multi-Human (MH)**: 다양한 기술 수준의 조작자 혼합
- **Machine-Generated (MG)**: 강화학습 에이전트로 생성

이 설계는 **데이터 품질이 모방학습에 미치는 영향**을 연구하는 데 직접 활용할 수 있습니다.

### 학습 실행

```python
# 개념 이해용 의사코드 — RoboMimic 학습
from robomimic.config import config_factory
from robomimic.algo import algo_factory
from robomimic.utils.train_utils import run_epoch

# 설정 로드
config = config_factory(algo_name="bc")
config.train.data = "path/to/demonstration.hdf5"
config.train.batch_size = 100
config.train.num_epochs = 2000

# 알고리즘 초기화
model = algo_factory(
    algo_name=config.algo_name,
    config=config,
    obs_key_shapes=obs_shapes,
    ac_dim=action_dim,
)

# 학습 루프
for epoch in range(config.train.num_epochs):
    train_loss = run_epoch(model, train_loader)
    if epoch % 50 == 0:
        success_rate = evaluate_rollout(model, env, n_episodes=50)
```

## 표준 벤치마크로서의 위치

robosuite와 RoboMimic의 가장 큰 기여는 **연구 간 비교를 가능하게 하는 표준화**입니다.

논문 A와 논문 B가 서로 다른 환경, 다른 컨트롤러, 다른 데이터셋으로 실험하면 결과 비교가 불가능합니다. robosuite의 Lift, NutAssembly 같은 표준 과제와 RoboMimic의 공개 데이터셋을 사용하면, 새 알고리즘의 성능을 기존 연구와 직접 비교할 수 있습니다.

이 표준화 역할 덕분에 매니퓰레이션 학습 분야에서 robosuite/RoboMimic은 **사실상의 표준(de facto standard)** 지위를 획득했습니다.

## 한계

**GPU 병렬화 미흡**: 기본 robosuite는 MJX처럼 GPU 병렬 환경을 지원하지 않습니다. 대규모 강화학습에서는 ManiSkill 같은 플랫폼이 더 유리합니다.

**시각적 사실감**: 기본 렌더링은 연구 수준으로 실제 로봇 카메라와 시각적 격차가 있습니다.

**Sim-to-Real 검증**: robosuite 환경에서 학습한 정책을 실제 Franka Panda에서 검증하는 연구 사례가 있지만, 전이 갭이 존재합니다.
