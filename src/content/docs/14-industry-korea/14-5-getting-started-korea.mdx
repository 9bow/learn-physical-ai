## 시작하기 전에: 이 분야의 특성 이해

Physical AI는 소프트웨어만으로 완결되지 않습니다. 시뮬레이션 학습으로 많은 것을 배울 수 있지만, 실물 로봇과의 접촉에서만 얻을 수 있는 직관도 있습니다. 반대로 실물 하드웨어 없이 시뮬레이터만으로도 강화학습, VLA 모델, sim-to-real 연구의 많은 부분을 실습할 수 있습니다. 예산과 목표에 따라 출발점을 고르면 됩니다.

## 한국어 학습 자료

### 공개 강의와 교재

현재 Physical AI를 특정한 한국어 전문 강의는 많지 않습니다. 대신 구성 요소별로 접근하는 것이 현실적입니다.

**강화학습 기초**: 조성현 저 『파이썬과 케라스로 배우는 강화학습』(위키북스), Sung Kim(김성훈) 교수의 유튜브 강의 시리즈가 국내에서 잘 알려져 있습니다. 서울대학교, KAIST 등의 AI 강의 영상도 공개되어 있습니다.

**딥러닝 기초**: fastai 한국어 스터디, 모두의딥러닝(홍콩과기대 김성훈 교수)이 입문자에게 인기 있습니다.

**로봇 운영체제(ROS2)**: ROS 공식 문서의 한국어 번역이 일부 존재하며, 국내 로봇 동호회와 유튜브 채널에서 ROS2 입문 콘텐츠를 찾을 수 있습니다.

영어가 가능하다면 공식 문서와 논문을 직접 읽는 것이 가장 최신이고 정확한 정보를 얻는 방법입니다. 이 분야는 발전 속도가 빠르고 한국어 번역이 따라가기 어렵습니다.

### 주요 논문 자원

arXiv(arxiv.org)의 Robotics(cs.RO), Learning(cs.LG) 카테고리를 정기적으로 확인하는 습관이 중요합니다. Papers with Code(paperswithcode.com)는 논문과 공개 코드를 연결해주어 실습에 유용합니다.

## 커뮤니티

### PyTorch Korea

**PyTorch Korea(pytorch.kr)** 는 국내 PyTorch 사용자 커뮤니티로, 딥러닝과 AI 전반에 대한 한국어 자료와 스터디 그룹이 활성화되어 있습니다. 로보틱스 특화는 아니지만, RL이나 컴퓨터 비전 관련 질문을 나눌 수 있는 한국어 커뮤니티 중 규모가 큰 편입니다.

### 오픈소스 커뮤니티와 GitHub

국내 연구자들이 GitHub에서 활발하게 활동합니다. Isaac Lab, MuJoCo, ROS2 관련 이슈와 PR 추적, 오픈소스 기여는 실력을 키우고 네트워크를 넓히는 효과적인 방법입니다.

### 학회와 워크숍

**한국로봇학회(KROS)** 와 **한국인공지능학회(KSAI)** 의 학술 대회는 국내 연구 동향을 파악하고 연구자들을 만나는 기회입니다. 국제 학회(ICRA, IROS, NeurIPS, CoRL)의 워크숍은 최신 연구 트렌드를 직접 접하는 장입니다.

## 실습 환경 구축

### 시뮬레이션: 비용 없이 시작

하드웨어 없이 시뮬레이터만으로 많은 것을 실습할 수 있습니다.

```text
┌────────────────────────────────────────────────────────────────┐
│            시뮬레이터별 입문 난이도 비교                        │
├────────────────────┬───────────────┬───────────────────────────┤
│ 시뮬레이터         │ 난이도        │ 추천 용도                 │
├────────────────────┼───────────────┼───────────────────────────┤
│ MuJoCo             │ 쉬움          │ RL 기초 실험               │
│ PyBullet           │ 쉬움          │ 무료, 빠른 프로토타이핑   │
│ Brax(JAX)          │ 중간          │ GPU 병렬 RL               │
│ Isaac Sim/Lab      │ 어려움        │ 고품질 시뮬, NVIDIA GPU   │
│ Gazebo/ROS2        │ 중간          │ ROS 통합 개발             │
└────────────────────┴───────────────┴───────────────────────────┘
```

MuJoCo 또는 PyBullet으로 시작하여 Gymnasium(OpenAI Gym 후계) 환경에서 기본 RL을 실습하는 것이 가장 낮은 진입 장벽입니다. Isaac Lab은 NVIDIA GPU가 필요하지만, 클라우드를 활용하면 노트북만으로도 사용할 수 있습니다.

### 저가 하드웨어 플랫폼

실물 로봇 실습에 관심이 있다면 비용 효율적인 플랫폼들이 있습니다.

**Unitree Go2 / H1 (사족·이족 보행)**: Unitree는 연구용 저가 로봇으로 국내 대학 연구실에서도 사용 사례가 늘고 있습니다. H1 이족 보행 로봇은 개인 구입이 어려운 가격대이지만, Go2 사족 보행 로봇은 연구 커뮤니티에서 활용됩니다.

**Raspberry Pi + 소형 서보 팔**: 비용이 매우 저렴하고, ROS2와 연동하는 실습에 좋습니다. 연구 수준의 조작 성능은 아니지만 기본 개념 실습에 충분합니다.

**LeRobot (Hugging Face)**: Hugging Face에서 공개한 저가 로봇 팔 프레임워크로, 3D 프린팅과 저렴한 서보 모터로 조작 로봇을 만들고 모방 학습을 실습할 수 있습니다. 국내에서도 관심이 높아지고 있습니다.

## GPU 자원 활용

물리 AI 시뮬레이션 연구는 GPU가 필수적입니다.

**클라우드 GPU**: AWS EC2(G4dn, P3), Google Cloud(A100/T4), Azure 등을 단기 임대할 수 있습니다. Lambda Labs, Vast.ai 같은 GPU 클라우드 서비스는 더 저렴한 경우도 있습니다.

**NAVER Cloud, NHN Cloud**: 국내 클라우드도 GPU 인스턴스를 제공합니다. 데이터가 국내에 머물러야 하거나 원화 결제가 필요한 경우 고려할 수 있습니다.

**대학/연구기관 HPC**: 대학원생이나 연구원이라면 소속 기관의 HPC 클러스터를 활용하는 것이 가장 경제적입니다. KISTI가 국내 연구자에게 슈퍼컴퓨터 자원을 제공하는 프로그램도 있습니다.

**개인 GPU**: 지속적으로 실험할 계획이라면 RTX 3090/4090 급 개인 GPU 투자가 장기적으로 경제적일 수 있습니다. Isaac Lab은 단일 RTX 3090으로도 기본 실험이 가능합니다.

## 진로 방향

Physical AI 관련 진로는 크게 세 방향으로 나뉩니다.

**연구(Research)**: 대학원 진학(국내 KAIST·POSTECH·서울대 또는 해외 CMU·Stanford·ETH Zurich 등) 후 학술 연구. 기초 원리를 깊이 파고드는 사람에게 적합합니다.

**산업 연구개발(Industry R&D)**: 국내외 로봇 기업(현대차그룹/Boston Dynamics, 두산로보틱스, 네이버랩스 등) 또는 NVIDIA, Google DeepMind, OpenAI 같은 글로벌 기업의 연구 부서. 실용 시스템 구축에 집중합니다.

**스타트업/창업**: 물류 자동화, 의료 로봇, 농업 로봇, 서비스 로봇 분야의 창업. 기술 스택과 시장 이해를 동시에 갖춰야 합니다.

어느 방향이든 **코딩 능력(Python, C++), 수학(선형대수, 확률론, 최적화), RL/딥러닝 기초, 로봇 운동학/동역학 이해** 가 공통 기반입니다.

## 권장 학습 로드맵

Physical AI를 처음 시작하는 사람을 위한 단계별 접근을 제안합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              입문자 권장 학습 순서                            │
├──────────────────────────────────────────────────────────────┤
│ 1단계: 기초 다지기 (1~3개월)                                 │
│   Python 숙달 + 선형대수/확률론 복습                         │
│   딥러닝 기초 (PyTorch)                                      │
│                                                              │
│ 2단계: RL 실습 (2~3개월)                                     │
│   Gymnasium + Stable-Baselines3                              │
│   MuJoCo/PyBullet 환경에서 기본 로코모션 과제                │
│                                                              │
│ 3단계: 로봇 특화 (3~6개월)                                   │
│   ROS2 기초 + Isaac Lab 입문                                 │
│   모방 학습(ACT, Diffusion Policy) 실습                     │
│                                                              │
│ 4단계: 심화 (지속)                                           │
│   VLA 모델, Sim-to-Real 프로젝트                             │
│   관심 분야 논문 재현 및 기여                                │
└──────────────────────────────────────────────────────────────┘
```

가장 중요한 것은 **실제로 코드를 실행하고 실험하는 것**입니다. 교재를 읽는 것과 실제 시뮬레이터에서 로봇이 넘어지고 다시 일어서는 과정을 지켜보며 디버깅하는 경험은 완전히 다릅니다. 이 학습 사이트에서 배운 개념들을 코드로 구현해보는 것을 강력하게 권장합니다.
