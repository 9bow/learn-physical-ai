## 매니퓰레이션이란

로봇 **매니퓰레이션(manipulation)**은 로봇 팔과 그리퍼(손)를 이용하여 물체를 집고, 옮기고, 조작하는 행위를 뜻합니다. 제조 라인의 물체 이송, 창고 피킹(warehouse picking), 외과 보조 기구 조작, 가정용 로봇의 도구 사용 등이 모두 매니퓰레이션에 해당합니다.

보행과 달리 매니퓰레이션은 **접촉의 품질**이 핵심입니다. 물체를 잡을 때의 미세한 손 자세, 물체 표면의 마찰, 가해지는 힘의 정밀한 제어가 성공과 실패를 가릅니다.

## 주요 태스크 계층

매니퓰레이션 태스크는 복잡도에 따라 계층으로 분류할 수 있습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              매니퓰레이션 태스크 복잡도 계층                  │
│                                                              │
│  낮음  ▲                                                     │
│        │  Reach (도달)                                       │
│        │    엔드이펙터를 목표 위치로 이동                     │
│        │    관측: 팔 상태 + 목표 위치                         │
│        │                                                     │
│        │  Grasp / Pick (파지 / 집기)                         │
│        │    물체를 안정적으로 집기                            │
│        │    관측: 팔 상태 + 물체 포즈 + 접촉 정보             │
│        │                                                     │
│        │  Place (내려놓기)                                   │
│        │    집은 물체를 목표 위치에 내려놓기                  │
│        │    Pick + 위치 정밀도 추가                           │
│        │                                                     │
│        │  Pick-and-Place                                     │
│        │    Pick + Place 연속 수행                            │
│        │                                                     │
│        │  In-hand Manipulation (인핸드)                      │
│        │    손 안에서 물체 회전·재파지                        │
│        │    다손가락 그리퍼 필요                              │
│  높음  ▼                                                     │
└──────────────────────────────────────────────────────────────┘
```

## 관측 설계의 난점

매니퓰레이션에서 관측 설계는 보행보다 훨씬 까다롭습니다.

### 물체 포즈 추정 문제

물체를 집으려면 물체의 위치와 방향(6-DoF 포즈)을 알아야 합니다. 시뮬레이션에서는 완벽한 포즈 정보를 얻을 수 있지만, 실제 환경에서는 카메라와 포즈 추정 알고리즘이 필요합니다.

이로 인해 두 가지 접근법이 존재합니다.

- **상태 기반(state-based)**: 시뮬레이션 내 실제 포즈 사용 → 학습은 쉬우나 실제 적용 시 포즈 추정 파이프라인 필요
- **이미지 기반(image-based)**: 카메라 이미지를 직접 관측으로 사용 → Sim-to-Real 어려움, 학습 비용 높음

Isaac Lab은 두 방식 모두 지원하며, 많은 연구가 상태 기반으로 먼저 학습 후 이미지 기반으로 이전하는 2단계 전략을 사용합니다.

### 접촉 정보의 중요성

물체를 잡는 동안 손과 물체 사이의 접촉 상태를 관측에 포함하면 학습이 크게 향상됩니다.

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
class ManipulationObsCfg:
    class PolicyObs(ObservationGroupCfg):
        # 팔 상태
        joint_pos = ...           # 관절 위치
        joint_vel = ...           # 관절 속도
        eef_pose = ...            # 엔드이펙터 포즈 (위치 + 방향)

        # 물체 상태
        object_pos = ...          # 물체 위치
        object_quat = ...         # 물체 방향 (쿼터니언)
        object_to_eef_pos = ...   # 물체에서 엔드이펙터까지 상대 위치

        # 목표
        target_pos = ...          # 목표 위치

        # 접촉 (중요)
        gripper_contact = ...     # 그리퍼 접촉 여부
```

## 보상 설계의 난점

매니퓰레이션 보상 설계의 가장 큰 문제는 **희박한 보상(sparse reward)**입니다.

"물체를 집으면 보상 1, 아니면 0"이라는 단순 보상은, 초기 정책이 물체를 집는 행동을 우연히 발견하기 전까지 아무 학습 신호도 없습니다. 이는 보행 태스크보다 훨씬 심각한데, 보행에서는 최소한 "서있으면 생존 보상"이라는 연속적 신호가 있기 때문입니다.

**Dense Reward(밀집 보상)** 설계로 이를 완화합니다.

```text
Pick 태스크 보상 예시:
┌─────────────────────────────────────────────────────────────┐
│ 단계별 보상 구성                                             │
│                                                             │
│ 1. 접근 보상: eef-물체 거리에 반비례                        │
│    r_reach = exp(-||eef_pos - obj_pos||)                    │
│                                                             │
│ 2. 파지 보상: 물체를 집었을 때 (접촉 + 들어올림)            │
│    r_grasp = is_contacted and obj_height > threshold        │
│                                                             │
│ 3. 리프팅 보상: 물체를 들어올릴수록 높은 보상               │
│    r_lift = clip(obj_height - table_height, 0, max_lift)   │
│                                                             │
│ 4. 목표 도달 보상: Pick-and-Place에서 목표 위치 근접        │
│    r_place = exp(-||obj_pos - target_pos||)                 │
└─────────────────────────────────────────────────────────────┘
```

## Dexterous Manipulation (손재주 조작)

**Dexterous manipulation**은 다손가락 핸드(dexterous hand)를 이용하여 손 안에서 물체를 조작하는 고급 기술입니다. 대표적 예시는 손 안에서 큐브를 특정 방향으로 회전시키는 태스크입니다.

OpenAI의 Dactyl 프로젝트(2019)는 Shadow Hand를 이용하여 루빅스 큐브를 푸는 데 필요한 큐브 조작을 RL로 학습했으며, 대규모 도메인 랜덤화를 통해 실제 로봇으로 이전하는 데 성공했습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              Dexterous Manipulation의 도전 요소               │
│                                                              │
│  고차원 행동 공간                                            │
│    Shadow Hand: 약 20개의 능동 자유도(DoF)                   │
│    행동 공간이 클수록 학습이 어려움                           │
│                                                              │
│  불안정한 접촉 동역학                                        │
│    손가락 끝과 물체 표면 간의 미끄러짐, 롤링 접촉             │
│    마찰 계수의 작은 변화가 결과에 큰 영향                    │
│                                                              │
│  부분 관측 (occlusion)                                       │
│    손가락이 카메라 시야를 가림                               │
│    촉각(tactile) 센서 필요성                                  │
│                                                              │
│  Sim-to-Real 갭                                              │
│    접촉 물리는 시뮬레이션이 특히 부정확한 영역               │
│    대규모 도메인 랜덤화가 필수                               │
└──────────────────────────────────────────────────────────────┘
```

## Isaac Lab의 매니퓰레이션 태스크

Isaac Lab은 Franka Panda, UR5, Kuka IIWA 등 산업용 로봇 팔을 위한 다양한 매니퓰레이션 태스크를 제공합니다.

```bash
# Franka Lift 태스크 (큐브 들어올리기)
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Lift-Cube-Franka-v0 \
  --num_envs 4096 --headless

# Stack 태스크 (큐브 쌓기)
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Stack-Cube-Franka-v0 \
  --num_envs 4096 --headless
```

## 고정 vs 이동 매니퓰레이터

| 유형 | 특징 | 주요 도전 |
|---|---|---|
| 고정 팔 | 베이스 고정, 팔만 제어 | 접촉 동역학, 파지 품질 |
| 이동 매니퓰레이터 | 이동 플랫폼 + 팔 | 전신 협조, 균형, 내비게이션 |
| 다손가락 핸드 | 다수 손가락 | 고차원 행동 공간, 접촉 복잡성 |

이동 매니퓰레이터(mobile manipulator)는 보행 제어와 팔 제어를 동시에 처리해야 하므로, 최근 연구의 중요한 방향입니다. Isaac Lab의 Spot + Arm 조합 태스크가 이러한 복합 태스크의 예입니다.

## 소결

로봇 매니퓰레이션은 정밀한 접촉 제어, 물체 포즈 추정, 희박 보상 문제라는 세 가지 핵심 도전을 갖습니다. 밀집 보상 설계와 단계적 접근, 그리고 대규모 도메인 랜덤화가 이를 극복하는 주요 전략입니다. Dexterous manipulation은 그 중에서도 가장 도전적인 영역으로, 많은 현재 연구의 최전선에 있습니다. 다음 챕터에서는 모든 RL 태스크에 공통으로 적용되는 보상 설계 원칙과 커리큘럼 학습을 심층적으로 다룹니다.
