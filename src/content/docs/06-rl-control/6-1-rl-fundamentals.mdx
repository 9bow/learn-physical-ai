## 강화학습과 로보틱스의 만남

전통적인 로봇 제어는 물리 모델 기반의 제어 이론(PID, 모델 예측 제어 등)에 의존했습니다. 이 접근법은 수학적으로 명확하지만, 정확한 물리 모델이 필요하고 예측 불가능한 환경 변화나 복잡한 접촉 동역학에 취약합니다.

**강화학습(Reinforcement Learning, RL)**은 다른 관점을 취합니다. 로봇이 스스로 시뮬레이션 환경과 상호작용하며 시행착오를 통해 최적 행동을 학습합니다. 수학적 모델 없이도 복잡한 보행, 매니퓰레이션, 균형 유지를 학습할 수 있습니다.

## 마르코프 결정 과정 (MDP)

RL의 수학적 기반은 **마르코프 결정 과정(Markov Decision Process, MDP)**입니다. MDP는 다음 다섯 가지 요소로 정의됩니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                   MDP 구성 요소                               │
│                                                              │
│  S (상태 공간, State Space)                                   │
│    로봇의 현재 상태를 나타내는 모든 가능한 값의 집합           │
│    예: 관절 각도, 속도, 베이스 자세, 목표까지의 거리           │
│                                                              │
│  A (행동 공간, Action Space)                                  │
│    로봇이 취할 수 있는 모든 가능한 행동                        │
│    예: 관절 위치 목표값, 토크 명령                             │
│                                                              │
│  P (전이 확률, Transition Probability)                        │
│    P(s'|s, a): 상태 s에서 행동 a를 취했을 때 s'로 이동할 확률 │
│    (시뮬레이션에서는 물리 엔진이 이를 결정)                    │
│                                                              │
│  R (보상 함수, Reward Function)                               │
│    R(s, a, s'): 전이 후 받는 즉각적 피드백 신호               │
│                                                              │
│  γ (할인 인수, Discount Factor)                               │
│    0 < γ < 1: 미래 보상의 현재 가치 비율 (보통 0.99 ~ 0.999) │
└──────────────────────────────────────────────────────────────┘
```

**마르코프 속성(Markov property)**은 미래 상태가 현재 상태와 행동만으로 결정되며 과거 이력에 무관하다는 가정입니다. 실제 로봇에서는 관측만으로 완전한 상태를 알기 어렵지만(부분 관측 환경, POMDP), 실용적으로는 최근 몇 프레임의 관측을 쌓아서 이를 근사합니다.

## 정책과 가치 함수

**정책(Policy, π)**: 상태를 입력으로 받아 행동을 출력하는 함수입니다.

- 결정론적 정책: `a = π(s)` — 항상 같은 상태에서 같은 행동
- 확률론적 정책: `a ~ π(·|s)` — 상태에서 행동의 확률 분포를 출력

딥 RL에서 정책은 신경망(뉴럴 네트워크)으로 파라미터화됩니다: `π_θ(a|s)`.

**가치 함수(Value Function, V)**: 현재 상태에서 정책 π를 따를 때 기대되는 누적 할인 보상입니다.

```
V^π(s) = E_π[∑_{t=0}^∞ γ^t R(s_t, a_t, s_{t+1}) | s_0 = s]
```

**행동-가치 함수(Q-Function, Q)**: 상태 s에서 행동 a를 취한 후 정책 π를 따를 때의 기대 누적 보상입니다.

```
Q^π(s, a) = E_π[∑_{t=0}^∞ γ^t R(s_t, a_t, s_{t+1}) | s_0 = s, a_0 = a]
```

## 온-폴리시 vs 오프-폴리시

RL 알고리즘은 데이터를 수집하는 정책과 학습하는 정책이 같은지에 따라 분류됩니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│            온-폴리시 vs 오프-폴리시 비교                          │
├──────────────────────────┬───────────────────────────────────────┤
│ 온-폴리시 (On-Policy)    │ 오프-폴리시 (Off-Policy)              │
├──────────────────────────┼───────────────────────────────────────┤
│ 수집 정책 = 학습 정책    │ 수집 정책 ≠ 학습 정책                 │
│ 리플레이 버퍼 불필요     │ 리플레이 버퍼 사용                    │
│ 데이터 효율 낮음         │ 데이터 효율 높음                      │
│ 안정적 수렴              │ 상대적으로 불안정할 수 있음           │
│ 대표: PPO, A3C           │ 대표: SAC, TD3, DDPG                  │
│ 시뮬레이션 기반 로보틱스 │ 실제 로봇 학습 (샘플 부족)            │
│ 에 주로 사용             │ 에 유리                               │
└──────────────────────────┴───────────────────────────────────────┘
```

Isaac Lab처럼 대규모 병렬 시뮬레이션이 가능한 환경에서는 데이터를 충분히 생성할 수 있으므로 온-폴리시 알고리즘(특히 PPO)이 많이 사용됩니다.

## PPO (Proximal Policy Optimization)

PPO는 로보틱스 RL에서 가장 널리 사용되는 알고리즘입니다. 2017년 OpenAI가 발표했으며, 구현 단순성과 안정적 성능이 장점입니다.

**핵심 아이디어**: 정책 업데이트 시 기존 정책에서 너무 멀리 이탈하지 않도록 클리핑으로 제한합니다.

```text
PPO Objective (클리핑 버전):

L_CLIP(θ) = E[min(
  r_t(θ) * A_t,                    ← 표준 정책 경사
  clip(r_t(θ), 1-ε, 1+ε) * A_t   ← 클리핑 적용
)]

r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)  (확률 비율)
A_t = 어드밴티지 추정값 (GAE 방법으로 계산)
ε = 0.2 (일반적인 클리핑 범위)
```

PPO의 주요 구성:

1. **수집 단계**: 현재 정책으로 `num_steps_per_env × num_envs` 데이터 수집
2. **GAE 계산**: 일반화된 어드밴티지 추정(Generalized Advantage Estimation)
3. **미니배치 업데이트**: 수집 데이터를 여러 에폭, 미니배치로 반복 학습
4. **클리핑**: 극단적 업데이트 방지

## SAC (Soft Actor-Critic)

SAC는 오프-폴리시 알고리즘으로, 엔트로피 정규화(entropy regularization)를 통해 탐험을 장려합니다.

**핵심 아이디어**: 보상을 최대화하면서 동시에 정책의 엔트로피(불확실성)도 최대화합니다. 이는 더 다양한 행동을 탐험하고 지역 최적해(local optima)에 빠지지 않도록 돕습니다.

```
SAC Objective:
J(π) = E[∑_t R(s_t, a_t) + α H(π(·|s_t))]

H(π(·|s)) = -E[log π(a|s)] (엔트로피)
α = 온도 파라미터 (탐험-착취 균형)
```

로보틱스에서 SAC는 실제 로봇 학습(실세계 데이터가 귀한 상황)이나 복잡한 접촉 동역학이 있는 매니퓰레이션 태스크에 자주 사용됩니다.

## 로보틱스에서의 RL 적합성

RL이 로봇 제어에 적합한 이유와 도전 과제를 함께 이해해야 합니다.

**RL이 강한 영역:**
- 접촉이 많고 동역학이 복잡한 태스크 (보행, 인핸드 매니퓰레이션)
- 모델링이 어려운 비선형 시스템
- 다양한 지형이나 객체에 적응하는 범용 정책

**RL의 도전 과제:**
- **샘플 비효율성**: 학습에 수백만~수억 번의 스텝 필요 → 고속 시뮬레이션으로 해결
- **보상 설계 어려움**: 보상이 잘못되면 의도치 않은 행동 학습 (reward hacking)
- **Sim-to-Real 갭**: 시뮬레이션에서 학습한 정책이 실제 로봇에서 실패할 수 있음 → 도메인 랜덤화로 완화
- **안전**: 학습 중 로봇이 위험한 행동을 할 수 있음 → 시뮬레이션 학습의 필요성

## Actor-Critic 구조

대부분의 딥 RL 알고리즘은 Actor-Critic 구조를 사용합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                 Actor-Critic 구조                             │
│                                                              │
│  관측(obs)                                                    │
│     │                                                        │
│     ├──────────────▶ Actor (정책 네트워크 π_θ)               │
│     │                    │                                   │
│     │                    ▼                                   │
│     │               행동 분포 μ, σ                           │
│     │                    │                                   │
│     │               행동 샘플링 a                            │
│     │                                                        │
│     └──────────────▶ Critic (가치 네트워크 V_φ)              │
│                          │                                   │
│                          ▼                                   │
│                     가치 추정 V(s)                           │
│                                                              │
│  PPO: Actor와 Critic이 파라미터를 일부 공유하는 경우 多      │
│  SAC: Actor와 Critic이 완전히 분리                           │
└──────────────────────────────────────────────────────────────┘
```

Isaac Lab의 RSL-RL은 `ActorCritic` 클래스로 이 구조를 구현하며, MLP(다층 퍼셉트론)와 LSTM 기반 아키텍처를 지원합니다. LSTM은 이전 상태 정보를 기억할 수 있어 부분 관측(partial observation) 환경에서 유용합니다.

## 소결

로봇 강화학습의 핵심은 MDP 프레임워크 위에서 정책과 가치 함수를 신경망으로 학습하는 것입니다. PPO는 온-폴리시 알고리즘으로 안정적이고 대규모 병렬 시뮬레이션에 적합하며, SAC는 오프-폴리시로 샘플 효율이 높아 실제 로봇 학습에 유리합니다. 다음 챕터에서는 이 알고리즘들이 4족·휴머노이드 보행이라는 구체적인 문제에 어떻게 적용되는지 살펴봅니다.
