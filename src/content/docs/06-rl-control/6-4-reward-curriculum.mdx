## 보상은 의도를 가르친다

강화학습에서 보상 함수는 학습의 방향을 결정하는 핵심 요소입니다. 알고리즘은 보상을 최대화하는 방향으로만 학습하며, **"무엇을 원하는지"가 아니라 "무엇을 보상으로 정의했는지"**를 학습합니다.

이 차이가 중요합니다. "로봇이 앞으로 빠르게 달리기를 원한다"는 의도를 "앞으로 이동한 거리에 비례한 보상"으로 정의하면, 로봇은 몸을 앞으로 기울여 넘어지면서 x축 이동 거리를 최대화하는 비정상적인 행동을 학습할 수 있습니다.

## Sparse Reward vs Dense Reward

### Sparse Reward (희박 보상)

성공 여부에만 보상을 부여합니다.

```text
r(s, a) = { +1  if goal_achieved
           { 0   otherwise

예: 물체를 목표 위치에 놓으면 +1, 아니면 0
```

**장점**: 목표를 정확하게 표현합니다. 보상 해킹이 적습니다.

**단점**: 초기 학습이 매우 느립니다. 우연히 성공을 경험할 때까지 학습 신호가 전혀 없습니다. 고차원 행동 공간이나 긴 에피소드에서 실용적이지 않습니다.

### Dense Reward (밀집 보상)

목표 달성까지의 과정마다 연속적인 보상을 제공합니다.

```text
예: 매 스텝마다 목표까지의 거리 감소분에 비례한 보상
r(s, a) = -||pos_t+1 - goal|| + ||pos_t - goal||
```

**장점**: 빠른 학습. 방향성 있는 그라디언트 신호를 제공합니다.

**단점**: 보상 설계 오류 시 reward hacking 위험이 높습니다.

## Reward Shaping

**Reward shaping**은 원래 보상 함수에 추가적인 보조 보상 항을 더하여 학습을 가속화하는 기법입니다. 이론적으로, 포텐셜 기반(potential-based) shaping은 최적 정책을 바꾸지 않음이 증명되어 있습니다.

실용적인 보상 shaping 원칙들:

```text
┌──────────────────────────────────────────────────────────────┐
│              Reward Shaping 설계 원칙                        │
│                                                              │
│  1. 행동을 분해하라                                          │
│     복잡한 태스크를 하위 목표로 나누고 각각 보상 부여         │
│     X "목표 도달" → O "접근 + 파지 + 리프팅 + 배치"         │
│                                                              │
│  2. 지수 형태를 사용하라                                     │
│     선형 거리 보상보다 exp(-dist/std)이 더 안정적            │
│     목표 근처에서 강한 그라디언트 제공                        │
│                                                              │
│  3. 부정적 행동에 패널티를 주어라                            │
│     에너지 소비, 급격한 행동 변화, 안전 위반                  │
│     가중치는 핵심 보상의 5~10% 수준으로 시작                  │
│                                                              │
│  4. 보상 항을 독립적으로 기록하라                            │
│     TensorBoard에서 각 항의 기여도를 모니터링                 │
│     이상한 행동이 나타나면 어느 항이 문제인지 추적            │
│                                                              │
│  5. 단순하게 시작하라                                        │
│     처음에는 1~2개 항으로 시작, 학습이 안 되면 항 추가       │
└──────────────────────────────────────────────────────────────┘
```

## Reward Hacking

**Reward hacking(보상 해킹)**은 정책이 의도하지 않은 방법으로 보상을 극대화하는 현상입니다. 보상이 목표의 "프록시(proxy)"일 뿐이기 때문에 발생합니다.

**실제 사례들:**

```text
태스크: 앞으로 빨리 달리기
보상: x 방향 이동 거리
해킹: 몸을 앞으로 기울여 미끄러지거나,
      다리로 지면을 밀어 탄도 운동처럼 이동

태스크: 물체 집기
보상: 물체가 테이블 위에서 올라간 높이
해킹: 물체를 집지 않고 팔로 밀어 올리거나,
      테이블 가장자리에서 떨어뜨려 낙하 중 높이 측정

태스크: 에너지 효율적 보행
보상: 이동 거리 - 토크 × 이동 거리
해킹: 전혀 움직이지 않음 (토크가 0이면 패널티도 0)
```

**Reward hacking 방지 전략:**

1. **여러 보상 항 조합**: 단일 보상은 해킹에 취약합니다. 속도 추종 + 에너지 효율 + 안정성 등 여러 측면을 동시에 보상합니다.

2. **물리적 제약 패널티**: 의도하지 않은 물리 상태(넘어짐, 충돌, 관절 한계 초과)에 강한 패널티를 부여합니다.

3. **보상 클리핑**: 극단적으로 큰 보상 값을 방지합니다.

4. **사람 검토**: 학습 중 정책의 행동을 주기적으로 시각화하고 확인합니다.

## Curriculum Learning

**커리큘럼 학습(Curriculum Learning)**은 학습 초기에는 쉬운 문제를 제시하고, 학습이 진행될수록 어려운 문제로 점진적으로 이행하는 전략입니다. 인간의 교육 방식과 유사합니다.

RL에서 커리큘럼 없이 어려운 태스크를 처음부터 학습하려 하면, 성공 경험을 거의 얻지 못하고 학습이 수렴하지 않을 수 있습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              Curriculum Learning 흐름                        │
│                                                              │
│  레벨 1: 쉬운 태스크                                        │
│    예: 평탄한 지면, 큰 물체, 가까운 목표                     │
│    성공률이 임계값(예: 70%) 달성 시 → 다음 레벨              │
│                                                              │
│                    ▼                                         │
│                                                              │
│  레벨 2: 중간 난이도                                        │
│    예: 약간의 요철, 중간 물체, 중간 거리 목표               │
│    성공률 임계값 달성 시 → 다음 레벨                         │
│                                                              │
│                    ▼                                         │
│                                                              │
│  레벨 3: 어려운 태스크                                       │
│    예: 험한 지형, 작은 물체, 먼 목표, 장애물                 │
│    여기서 최종 정책 완성                                     │
└──────────────────────────────────────────────────────────────┘
```

### Isaac Lab의 CurriculumManager

Isaac Lab은 `CurriculumManager`를 통해 커리큘럼을 관리합니다.

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
from isaaclab.managers import CurriculumTermCfg

class CurriculumCfg:
    """커리큘럼 설정"""
    terrain_levels = CurriculumTermCfg(
        func=mdp.terrain_levels_vel,
        params={
            "asset_cfg": SceneEntityCfg("robot"),
            # 성공률이 임계값 이상이면 난이도 상향
            "success_threshold": 0.7,
        }
    )
```

커리큘럼의 핵심은 **언제 다음 단계로 이행하는가**의 기준입니다. 일반적인 기준:

- **성공률 기반**: 최근 N 에피소드 중 성공 비율이 임계값 초과
- **보상 기반**: 평균 에피소드 보상이 특정 임계값 초과
- **시간 기반**: 고정된 학습 이터레이션 이후 자동 이행

### 지형 커리큘럼

보행 학습에서 가장 자주 사용되는 커리큘럼입니다. Isaac Lab은 격자 형태로 지형을 배치하고, 각 행(row)이 난이도 레벨에 해당하도록 구성합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              지형 격자 커리큘럼 구조 (Bird's Eye View)        │
│                                                              │
│  Col 0      Col 1      Col 2      Col 3      (지형 유형)     │
│  ┌────────┬────────┬────────┬────────┐                      │
│  │평탄    │평탄    │평탄    │평탄    │ Row 0: 레벨 1 (쉬움) │
│  ├────────┼────────┼────────┼────────┤                      │
│  │낮은요철│낮은요철│낮은요철│낮은요철│ Row 1: 레벨 2        │
│  ├────────┼────────┼────────┼────────┤                      │
│  │중간요철│중간요철│낮은계단│낮은계단│ Row 2: 레벨 3        │
│  ├────────┼────────┼────────┼────────┤                      │
│  │높은요철│높은요철│중간계단│불연속  │ Row 3: 레벨 4 (어려움)│
│  └────────┴────────┴────────┴────────┘                      │
│                                                              │
│  각 환경은 성과에 따라 해당 레벨의 지형에 배치              │
└──────────────────────────────────────────────────────────────┘
```

성공적으로 현재 지형을 통과하면 해당 환경은 한 레벨 어려운 지형으로 이동합니다. 실패가 반복되면 한 레벨 낮은 지형으로 돌아갑니다. 각 환경 인스턴스가 독립적으로 자신의 레벨을 관리하므로, 에이전트는 항상 자신의 현재 능력에 맞는 도전을 받습니다.

## Automatic Domain Randomization과의 연계

커리큘럼은 지형뿐만 아니라 물리 파라미터에도 적용할 수 있습니다. 초기에는 현실적인 물리 파라미터 범위를 좁게 설정하고, 학습이 진행됨에 따라 랜덤화 범위를 점진적으로 넓히는 방식입니다.

| 단계 | 질량 랜덤화 | 마찰 랜덤화 | 지연 랜덤화 |
|---|---|---|---|
| 초기 | ±5% | ±10% | 없음 |
| 중기 | ±20% | ±30% | ±10ms |
| 후기 | ±50% | ±80% | ±20ms |

이 전략은 7장에서 다룰 Sim-to-Real 이전과 직결됩니다.

## 소결

보상 설계는 RL의 성패를 결정하는 핵심 기술입니다. 희박 보상의 탐험 어려움을 밀집 보상으로 완화하되, 보상 해킹에 주의해야 합니다. 커리큘럼 학습은 어려운 태스크를 점진적으로 학습하게 하여 처음부터 복잡한 환경에서 학습하는 것보다 훨씬 안정적이고 효율적인 결과를 만들어냅니다. 다음 챕터에서는 커리큘럼과 함께 Sim-to-Real 이전의 핵심 기법인 Domain Randomization을 깊이 있게 다룹니다.
