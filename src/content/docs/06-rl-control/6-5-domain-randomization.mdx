## Sim-to-Real 갭의 본질

시뮬레이션에서 학습한 정책이 실제 로봇에서 실패하는 현상은 로보틱스 RL의 가장 오래된 문제입니다. 이를 **Sim-to-Real 갭(Sim-to-Real Gap)**이라고 합니다.

갭의 원인은 시뮬레이션이 현실을 완벽하게 모델링하지 못하는 데 있습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              Sim-to-Real 갭의 주요 원인                       │
│                                                              │
│  물리 모델 오차                                              │
│    ├─ 관절 마찰, 백래시(backlash) 미모델링                   │
│    ├─ 접촉 동역학의 단순화 (딱딱한 접촉 가정)                │
│    ├─ 공기 저항, 와류 등 무시                                │
│    └─ 강체(rigid body) 가정 (실제 구조는 유연함)             │
│                                                              │
│  하드웨어 특성 오차                                          │
│    ├─ 실제 질량·관성 텐서 불확실성                           │
│    ├─ 모터 특성 비선형성                                     │
│    ├─ 센서 노이즈와 지연(latency)                            │
│    └─ 제어 루프 타이밍 지터(jitter)                          │
│                                                              │
│  시각/환경 오차                                              │
│    ├─ 카메라 이미지 텍스처, 조명 차이                        │
│    └─ 실제 환경의 예측 불가능한 변수 (바람, 진동)            │
└──────────────────────────────────────────────────────────────┘
```

## Domain Randomization이란

**Domain Randomization(도메인 랜덤화)**는 학습 중에 시뮬레이션의 물리 파라미터와 시각 요소를 의도적으로 무작위로 변화시키는 기법입니다. 핵심 아이디어는 다음과 같습니다.

"현실이 어떤 특정 파라미터 값을 가지는지 정확히 모른다면, 가능한 모든 파라미터 범위에서 잘 작동하도록 정책을 훈련하자."

결과적으로 학습된 정책은 다양한 환경에 견고(robust)하게 작동하며, 실제 현실 파라미터도 그 범위 안에 포함될 가능성이 높아집니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              Domain Randomization의 직관적 설명               │
│                                                              │
│  랜덤화 없이 학습:                                           │
│  정책은 시뮬레이션의 특정 파라미터(예: 마찰=0.5)에 과적합   │
│  실제 로봇(마찰=0.3~0.7 불확실)에서 실패                    │
│                                                              │
│  랜덤화 후 학습:                                             │
│  마찰=0.1~0.9 전 범위에서 학습                               │
│  실제 로봇의 마찰값이 이 범위에 속하면 → 성공               │
│                                                              │
│  핵심: 랜덤화 범위 ⊇ 현실 파라미터 분포                     │
└──────────────────────────────────────────────────────────────┘
```

## 물리 파라미터 랜덤화

가장 중요한 랜덤화 유형입니다. Isaac Lab에서 `EventManager`를 통해 구현합니다.

### 주요 물리 파라미터

| 파라미터 | 설명 | 전형적 랜덤화 범위 |
|---|---|---|
| 질량(mass) | 로봇 링크의 질량 | 원래 값의 ±30~50% |
| 관성 텐서 | 링크의 회전 관성 | 원래 값의 ±30% |
| 마찰 계수 | 발-지면 마찰 | 0.5 ~ 1.5 |
| 반발 계수 | 충격 후 반발력 | 0.0 ~ 0.5 |
| 관절 강성 | PD 컨트롤러 kp | 원래 값의 ±20% |
| 관절 감쇄 | PD 컨트롤러 kd | 원래 값의 ±20% |
| 외부 힘 | 랜덤 외력 인가 | 0~20N, 무작위 방향 |

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
from isaaclab.managers import EventTermCfg

class EventCfg:
    """도메인 랜덤화 이벤트 설정"""

    # 리셋 시마다 물리 파라미터 랜덤화
    physics_material = EventTermCfg(
        func=mdp.randomize_rigid_body_material,
        mode="reset",  # "reset": 에피소드 리셋 시 / "interval": 주기적
        params={
            "asset_cfg": SceneEntityCfg("robot"),
            "static_friction_range": (0.5, 1.5),
            "dynamic_friction_range": (0.4, 1.2),
            "restitution_range": (0.0, 0.5),
        }
    )

    # 로봇 질량 랜덤화
    base_mass = EventTermCfg(
        func=mdp.randomize_rigid_body_mass,
        mode="reset",
        params={
            "asset_cfg": SceneEntityCfg("robot", body_names="base"),
            "mass_distribution_params": (-5.0, 5.0),  # 기본 질량 대비 ±5kg
        }
    )

    # 주기적으로 외부 힘 인가 (강인성 테스트)
    push_robot = EventTermCfg(
        func=mdp.push_by_setting_velocity,
        mode="interval",
        interval_range_s=(6.0, 12.0),  # 6~12초마다 한 번
        params={
            "velocity_range": {
                "x": (-0.5, 0.5),
                "y": (-0.5, 0.5),
            }
        }
    )
```

## 센서 노이즈와 지연 랜덤화

실제 센서는 이상적이지 않습니다. 노이즈와 지연을 시뮬레이션에 추가하면 실제 적용 시 정책의 강인성이 높아집니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              센서 노이즈 랜덤화 항목                          │
│                                                              │
│  관측 노이즈 (Observation Noise)                             │
│    ├─ 관절 위치 측정 오차: 가우시안 노이즈 ±0.01 rad        │
│    ├─ 관절 속도 측정 오차: 가우시안 노이즈 ±0.05 rad/s      │
│    ├─ IMU 가속도 오차: ±0.1 m/s²                            │
│    └─ IMU 각속도 오차: ±0.05 rad/s                          │
│                                                              │
│  제어 지연 (Control Latency)                                 │
│    ├─ 통신 지연: 5~20ms                                      │
│    └─ 액추에이터 응답 지연: 20~40ms                          │
│                                                              │
│  행동 랜덤화 (Action Randomization)                          │
│    └─ 행동에 가우시안 노이즈 추가                            │
└──────────────────────────────────────────────────────────────┘
```

제어 지연(latency)은 특히 중요합니다. 시뮬레이션에서 0ms 지연으로 학습한 정책은 실제 로봇의 20~40ms 통신·계산 지연에 취약합니다. 학습 시 지연을 랜덤화하면 이 갭이 크게 줄어듭니다.

## 시각 랜덤화 (Visual Randomization)

카메라 기반 관측을 사용하는 경우, 시각적 외관의 랜덤화도 중요합니다.

```text
시각 랜덤화 항목:
├─ 텍스처: 물체와 배경의 재질/패턴 무작위 변경
├─ 조명: 광원 위치, 강도, 색상 변경
├─ 카메라: 위치/각도 미세 변화, 렌즈 왜곡
└─ 물체 색상: RGB 색상 무작위 변경
```

OpenAI Dactyl 프로젝트는 극단적인 시각 랜덤화를 적용하여 완전히 무작위 텍스처 환경에서 학습한 후 실제 카메라 이미지로 이전하는 데 성공했습니다.

## Automatic Domain Randomization (ADR)

**ADR(Automatic Domain Randomization)**은 랜덤화 파라미터의 범위를 수동으로 설정하지 않고, 정책의 성과에 따라 자동으로 범위를 조정하는 방법입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              ADR 동작 원리                                    │
│                                                              │
│  1. 좁은 랜덤화 범위로 시작 (쉬운 변동)                     │
│       │                                                      │
│       ▼                                                      │
│  2. 정책 성과 측정 (성공률 또는 평균 보상)                   │
│       │                                                      │
│       ├─ 성과가 높은 경우 (임계값 이상)                      │
│       │    └─▶ 랜덤화 범위 확대 (더 극단적인 변동)           │
│       │                                                      │
│       └─ 성과가 낮은 경우 (임계값 이하)                      │
│            └─▶ 랜덤화 범위 축소 (더 쉬운 변동으로 복귀)     │
│                                                              │
│  3. 2~3 반복 → 점진적으로 랜덤화 범위 최대화                │
│                                                              │
│  결과: 정책이 감당할 수 있는 최대 랜덤화 범위 자동 탐색     │
└──────────────────────────────────────────────────────────────┘
```

ADR의 핵심 장점은 랜덤화 범위를 수동으로 튜닝하지 않아도 된다는 점입니다. 정책이 성장함에 따라 더 어려운 변동에 자동으로 노출됩니다.

## Isaac Lab에서의 EventManager

Isaac Lab은 도메인 랜덤화를 `EventManager`로 관리합니다. 이벤트는 세 가지 모드로 실행됩니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              EventManager 모드                               │
│                                                              │
│  "startup"                                                   │
│    씬 생성 시 한 번 실행                                     │
│    용도: 고정 물리 속성 설정                                  │
│                                                              │
│  "reset"                                                     │
│    각 에피소드 리셋 시 실행                                  │
│    용도: 물리 파라미터, 초기 상태, 지형 재생성               │
│    ← 가장 많이 사용되는 모드                                 │
│                                                              │
│  "interval"                                                  │
│    지정된 시간 간격으로 주기적 실행                          │
│    용도: 외부 힘 인가, 페이로드 변경 시뮬레이션              │
└──────────────────────────────────────────────────────────────┘
```

## 랜덤화 범위 설정 가이드

너무 좁은 범위는 Sim-to-Real 이전 효과가 없고, 너무 넓은 범위는 학습 자체가 불가능해집니다.

일반적인 접근법:

1. **현실 측정**: 가능하다면 실제 로봇의 물리 파라미터를 직접 측정합니다.
2. **보수적 시작**: 측정값의 ±10~20%로 시작합니다.
3. **점진적 확장**: 학습이 수렴하면 범위를 넓힙니다.
4. **실제 테스트**: 실제 로봇에서 실패 패턴을 분석하여 누락된 랜덤화 항목을 추가합니다.

```text
실제 Sim-to-Real 이전 사이클:

  시뮬레이션 학습 → 실제 테스트 → 실패 분석 → 랜덤화 개선 → 반복
        ↑_____________________________________________________|
```

## 소결

Domain Randomization은 시뮬레이션과 현실의 갭을 "정확히 모델링"이 아닌 "다양한 가능성을 모두 훈련"으로 해결합니다. 물리 파라미터(질량, 마찰, 강성), 센서 노이즈, 제어 지연, 시각적 요소의 랜덤화가 핵심이며, ADR은 랜덤화 범위를 자동으로 최적화합니다. Isaac Lab의 EventManager는 이를 체계적으로 구현하는 도구입니다. 이 기법을 보상 설계, 커리큘럼 학습과 결합하면 시뮬레이션에서 학습한 정책을 실제 로봇에 성공적으로 이전하는 가장 검증된 경로가 됩니다.
