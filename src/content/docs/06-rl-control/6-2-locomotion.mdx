## RL 보행 학습의 패러다임 전환

2010년대 중반까지 4족 로봇의 보행은 주로 전통적인 MPC(Model Predictive Control)와 발 배치 계획(footstep planning)을 조합하여 구현되었습니다. 이 방법은 수학적으로 정교하지만, 예측하지 못한 지형이나 충격에 대응하기 위해 복잡한 예외 처리가 필요했습니다.

ETH 취리히 RSL(로보틱 시스템 랩)의 연구팀은 2021년 ANYmal 로봇을 이용한 보행 RL 연구에서, Isaac Lab의 전신인 NVIDIA Isaac Gym을 활용하여 수천 개의 병렬 시뮬레이션 환경에서 단 몇 분~몇 시간 만에 강인한 보행 정책을 학습하는 결과를 발표했습니다. 이 접근법은 이후 로봇 보행 학습의 표준 패러다임이 되었습니다.

## 4족 보행 로봇의 보행 학습 구조

```text
┌──────────────────────────────────────────────────────────────┐
│              4족 보행 RL 학습 구조                            │
│                                                              │
│  입력 (관측)                   출력 (행동)                   │
│  ┌─────────────────┐          ┌─────────────────┐            │
│  │ 관절 위치(12)   │          │ 관절 위치 목표  │            │
│  │ 관절 속도(12)   │          │ (12개 관절)     │            │
│  │ 베이스 각속도(3)│ ──▶NN──▶ └─────────────────┘            │
│  │ 중력 방향(3)    │                                          │
│  │ 속도 명령(3)    │   (PPO + Actor-Critic MLP)              │
│  │ 이전 행동(12)   │                                          │
│  └─────────────────┘                                          │
│                                                              │
│  총 관측 차원: ~48 (고유수용감각 위주)                        │
│  총 행동 차원: 12 (4족 × 3 관절/다리)                        │
└──────────────────────────────────────────────────────────────┘
```

### 고유수용감각 (Proprioception) 위주 관측

보행 학습에서 가장 중요한 설계 결정 중 하나는 **관측에 카메라나 LiDAR 같은 외부 센서를 포함하지 않는 것**입니다. 대신 로봇 자신의 몸 상태(관절 각도·속도, IMU 데이터)만으로 보행을 학습합니다. 이를 **맹목 보행(blind locomotion)**이라고 합니다.

고유수용감각 관측의 장점:

- **센서 단순성**: 카메라나 LiDAR 없이 저렴한 센서로 구현 가능
- **Sim-to-Real 이전 용이**: 카메라 이미지의 시뮬레이션-현실 차이 없음
- **강인성**: 시각 정보 없이도 지형 변화에 적응하는 정책 학습 가능

실제로 잘 훈련된 맹목 보행 정책은 울퉁불퉁한 지형, 계단, 경사면을 발 감각(접촉 피드백)과 신체 움직임만으로 성공적으로 통과할 수 있습니다.

## 보행 보상 설계

보행 학습에서 보상은 여러 항목으로 구성됩니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              보행 보상 항목 구성                              │
│                                                              │
│  핵심 보상 (긍정)                                            │
│  ├── 목표 선속도 추적    +1.0  (xy 방향 속도 명령 추종)      │
│  ├── 목표 각속도 추적    +0.5  (요(yaw) 방향 회전 명령 추종) │
│  └── 생존 보상           +0.1  (에피소드 지속 시 소량 보상)   │
│                                                              │
│  패널티 (부정)                                               │
│  ├── 관절 토크           -1e-5 (에너지 소비 억제)            │
│  ├── 관절 가속도         -1e-7 (부드러운 움직임 유도)        │
│  ├── 행동 변화율         -0.01 (급격한 행동 변화 억제)       │
│  ├── 베이스 높이         -1.0  (몸통이 너무 낮으면 패널티)   │
│  ├── 바람직하지 않은 접촉 -1.0 (무릎/허벅지 지면 접촉)      │
│  └── 미끄러짐            -0.1  (발이 지면에서 미끄러질 때)   │
│                                                              │
│  보행 형태 보상                                              │
│  └── 발 공중 시간        +0.125 (양 발 교대 사용 유도)      │
└──────────────────────────────────────────────────────────────┘
```

**속도 명령 추종(velocity command tracking)**은 핵심 보상입니다. 조이스틱이나 자율 내비게이션 시스템이 `(v_x, v_y, ω_z)` 형태의 속도 명령을 내리면, 정책은 이 명령을 추종하도록 학습합니다. 이는 "지점 A에서 지점 B로 이동"이 아닌 "속도를 따라가라"는 범용 인터페이스를 제공합니다.

## 걸음새 (Gait) 자연 발생

흥미롭게도, 보행 RL에서는 명시적으로 걸음새(gait)를 지정하지 않아도 학습 과정에서 자연스러운 걸음새가 나타납니다. 발 공중 시간(feet air time) 보상과 속도 명령 추종을 조합하면, 저속에서는 워크(walk), 중속에서는 트롯(trot), 고속에서는 바운드(bound)에 가까운 동작이 스스로 발현됩니다.

```text
속도에 따른 걸음새 발현 (4족 로봇):

느림                                                빠름
 ◀────────────────────────────────────────────────▶
  Walk         Trot          Pace/Canter    Bound/Gallop
  (한 발씩)   (대각선 쌍)   (같은 쪽 쌍)   (두 발씩)
```

## 지형 랜덤화 (Terrain Randomization)

단일 평탄한 지면에서 학습한 정책은 실제 환경의 요철이나 경사면에서 실패합니다. **지형 랜덤화**는 학습 중 다양한 지형에 정책을 노출시켜 강인성을 높입니다.

Isaac Lab은 다양한 지형 생성기를 내장합니다.

| 지형 유형 | 설명 | 난이도 |
|---|---|---|
| 평탄 지면 | 기준 지형 | 낮음 |
| 무작위 요철 | 불규칙한 높낮이 | 중간 |
| 피라미드 계단 | 오르막/내리막 계단 | 높음 |
| 불연속 지형 | 끊어진 플랫폼 | 매우 높음 |
| 경사면 | 일정 각도 경사 | 중간 |
| 피라미드 경사 | 오르내리는 경사 | 높음 |

**지형 커리큘럼**은 쉬운 지형에서 시작하여 학습이 진행됨에 따라 어려운 지형을 점진적으로 도입합니다. 에이전트의 성공률에 따라 자동으로 난이도를 조절하는 방식이 일반적입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              지형 커리큘럼 적용 예시                          │
│                                                              │
│  초기: 90% 평탄 지면, 10% 요철                               │
│         │                                                    │
│         ▼ (성공률 70% 이상 달성)                             │
│  중기: 60% 평탄, 30% 요철, 10% 계단                          │
│         │                                                    │
│         ▼ (성공률 70% 이상 달성)                             │
│  후기: 20% 평탄, 40% 요철, 30% 계단, 10% 불연속              │
│         │                                                    │
│         ▼ (성공률 70% 이상 달성)                             │
│  완성: 다양한 지형에서 강인한 보행 정책                       │
└──────────────────────────────────────────────────────────────┘
```

## 휴머노이드 보행의 추가 도전

4족 보행에 비해 **휴머노이드(2족) 보행**은 더 복잡한 도전을 제시합니다.

**안정성 문제**: 2족 보행은 본질적으로 불안정하며, 지지 다각형(support polygon)이 매우 좁습니다. 4족 로봇은 안정적인 트롯 보행 시 항상 2개 이상의 발이 지면에 닿지만, 2족은 싱글 서포트 페이즈에서 한 발만 지면에 닿습니다.

**자유도 증가**: 일반적인 휴머노이드는 28~44개의 자유도를 갖습니다. 행동 공간이 크면 학습이 더 어려워집니다.

**팔-다리 조정**: 보행 중 팔의 스윙 동작이 균형에 영향을 미치므로, 전신(whole-body) 협조 동작을 학습해야 합니다.

최근 휴머노이드 보행 연구에서는 강화학습이 전통 제어를 능가하는 결과들이 나오고 있으며, Isaac Lab은 Unitree H1, G1, Boston Dynamics Atlas 등 다양한 휴머노이드 플랫폼을 위한 태스크를 제공합니다.

## Isaac Lab 보행 예제

Isaac Lab의 공식 보행 태스크들:

```bash
# ANYmal C 평탄 지형 보행
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Flat-Anymal-C-v0 \
  --num_envs 4096 --headless

# ANYmal C 험지 보행
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Rough-Anymal-C-v0 \
  --num_envs 4096 --headless

# Unitree H1 휴머노이드 보행
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Flat-H1-v0 \
  --num_envs 4096 --headless
```

## 소결

RL 보행 학습은 고유수용감각 위주 관측, 속도 명령 추종 보상, 지형 커리큘럼의 조합으로 강인하고 적응력 있는 보행 정책을 만들어냅니다. 명시적인 걸음새 설계 없이도 자연스러운 보행이 발현되며, 맹목 보행 정책은 실제 로봇 적용에서 센서 의존성을 낮추는 장점이 있습니다. 다음 챕터에서는 보행보다 더 세밀한 제어가 필요한 매니퓰레이션 태스크를 다룹니다.
