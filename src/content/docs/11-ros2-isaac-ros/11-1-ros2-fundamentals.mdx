## ROS 2란 무엇인가

ROS 2(Robot Operating System 2)는 로봇 소프트웨어 개발을 위한 오픈소스 미들웨어 프레임워크입니다. "운영체제"라는 이름이 붙어 있지만 실제로는 리눅스 위에서 동작하는 라이브러리·툴 모음에 가깝습니다. 센서 데이터 처리, 경로 계획, 하드웨어 추상화, 시각화, 로깅 등 로봇 개발에 필요한 대부분의 인프라를 제공합니다.

ROS 1이 2007년 PR2 로봇을 위해 설계된 것과 달리, ROS 2는 2017년부터 실제 제품 환경(real-time, 다중 로봇, 임베디드 시스템)을 목표로 재설계되었습니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│                    ROS 2 소프트웨어 스택                         │
├─────────────────────────────────────────────────────────────────┤
│  응용 레이어: Nav2, MoveIt 2, Isaac ROS GEMs, 사용자 패키지     │
├─────────────────────────────────────────────────────────────────┤
│  ROS 2 클라이언트 라이브러리: rclcpp (C++) / rclpy (Python)     │
├─────────────────────────────────────────────────────────────────┤
│  rcl (ROS Client Library) — 공통 추상화                         │
├─────────────────────────────────────────────────────────────────┤
│  rmw (ROS Middleware Interface)                                  │
├───────────────────┬─────────────────────────────────────────────┤
│  Fast DDS         │  CycloneDDS  │  Connext DDS  │  (기타 구현) │
├───────────────────┴─────────────────────────────────────────────┤
│  운영체제: Linux / macOS / Windows / RTOS                       │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념 1: 노드(Node)

**노드(node)** 는 ROS 2의 기본 실행 단위입니다. 하나의 실행 가능한 프로세스로, 특정 역할(카메라 드라이버, 경로 플래너, 모터 제어기 등)을 담당합니다. 각 노드는 독립적으로 실행되며, 다른 노드와 토픽·서비스·액션을 통해 통신합니다.

노드는 단일 책임 원칙에 따라 작게 설계하는 것이 권장됩니다. 복잡한 시스템은 여러 노드가 협력하는 구조로 만듭니다.

```bash
# 노드 실행 (패키지명, 실행파일명 지정)
ros2 run turtlesim turtlesim_node

# 실행 중인 노드 목록 확인
ros2 node list

# 특정 노드 정보 조회
ros2 node info /turtlesim
```

## 핵심 개념 2: 토픽(Topic)

**토픽(topic)** 은 노드 간 비동기 단방향 데이터 스트림입니다. Publisher(발행자)가 메시지를 토픽에 게시하면, Subscriber(구독자)가 이를 수신합니다. 발행자와 구독자는 서로를 직접 알 필요 없이 토픽 이름으로만 연결됩니다.

이 패턴은 센서 데이터(카메라 이미지, LiDAR 포인트 클라우드, IMU 데이터 등) 처럼 지속적으로 흐르는 데이터에 적합합니다.

```bash
# 토픽 목록 확인
ros2 topic list

# 토픽 메시지 실시간 출력
ros2 topic echo /turtle1/pose

# 토픽 발행 주파수 확인
ros2 topic hz /camera/image_raw

# 직접 메시지 발행
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 1.0}, angular: {z: 0.5}}"
```

## 핵심 개념 3: 서비스(Service)

**서비스(service)** 는 요청(request)–응답(response) 방식의 동기 통신입니다. 클라이언트가 서버에 요청을 보내고, 서버가 처리 후 응답을 반환합니다. 카메라 보정 실행, 맵 저장, 파라미터 조회 등 한 번 요청하고 결과를 받아야 하는 작업에 적합합니다.

```bash
# 서비스 목록 확인
ros2 service list

# 서비스 타입 확인
ros2 service type /spawn

# 서비스 호출 (터틀 생성 예시)
ros2 service call /spawn turtlesim/srv/Spawn \
  "{x: 2.0, y: 2.0, theta: 0.0, name: 'turtle2'}"
```

## 핵심 개념 4: 액션(Action)

**액션(action)** 은 장시간 실행 작업을 위한 통신 패턴입니다. 서비스와 달리 실행 중에 **피드백(feedback)** 을 주기적으로 제공하며, 작업을 **취소(cancel)** 할 수도 있습니다. 로봇 팔 이동, 내비게이션 목표 이동, 물체 파지 등 완료까지 수 초 이상 걸리는 작업에 사용합니다.

```text
액션 통신 구조:

클라이언트                    서버
    │                           │
    │──── Goal 전송 ──────────▶│
    │◀─── Goal Accept/Reject ───│
    │                           │  (작업 실행 중)
    │◀─── Feedback (주기적) ────│
    │◀─── Feedback ─────────────│
    │                           │  (작업 완료)
    │◀─── Result ───────────────│
```

```bash
# 액션 목록 확인
ros2 action list

# 액션 서버에 골 전송 (개념 예시 — 실제 메시지 타입은 패키지별 확인)
ros2 action send_goal /navigate_to_pose \
  nav2_msgs/action/NavigateToPose \
  "{pose: {pose: {position: {x: 1.0, y: 1.0}}}}"
```

## 핵심 개념 5: DDS 미들웨어

ROS 2의 가장 중요한 아키텍처 변화는 **DDS(Data Distribution Service)** 를 통신 미들웨어로 채택한 것입니다. DDS는 OMG(Object Management Group)가 표준화한 실시간 발행-구독 미들웨어로, 산업용 시스템에서 수십 년간 검증된 기술입니다.

DDS 도입으로 ROS 2가 얻은 이점:

| 특성 | ROS 1 | ROS 2 (DDS) |
|------|-------|-------------|
| 통신 중재자 | rosmaster 필요 | 분산형, 단일 장애점 없음 |
| 실시간성 | 미지원 | QoS 정책으로 지원 |
| 다중 로봇 | 복잡한 설정 필요 | 도메인 ID로 격리 |
| 보안 | 없음 | DDS-Security 표준 |
| 언어 | Python/C++ 위주 | C, C++, Python, Java, etc. |

**QoS(Quality of Service)** 는 DDS가 제공하는 통신 품질 설정입니다. 신뢰성(Reliability: RELIABLE/BEST_EFFORT), 내구성(Durability: TRANSIENT_LOCAL/VOLATILE), 히스토리 깊이(History depth) 등을 토픽별로 설정할 수 있습니다.

## 패키지와 워크스페이스

**패키지(package)** 는 ROS 2 소프트웨어의 배포 단위입니다. 하나의 패키지는 관련 노드, 메시지 정의, 설정 파일, 런치 파일을 포함합니다.

**워크스페이스(workspace)** 는 여러 패키지를 묶어서 빌드하는 디렉토리입니다. 표준 구조는 다음과 같습니다.

```text
my_ws/
├── src/                  ← 소스 코드 패키지들
│   ├── my_robot_pkg/
│   │   ├── package.xml
│   │   ├── setup.py      (Python 패키지) 또는 CMakeLists.txt (C++)
│   │   └── my_robot_pkg/
│   │       └── my_node.py
│   └── my_msgs/
│       └── ...
├── build/                ← colcon 빌드 산출물
├── install/              ← 설치 파일 (source 하는 곳)
└── log/                  ← 빌드 로그
```

**colcon**은 ROS 2의 공식 빌드 도구입니다. catkin(ROS 1)을 대체하며, ament_cmake, ament_python 등 여러 빌드 시스템을 통합 지원합니다.

```bash
# 워크스페이스 루트에서 전체 빌드
colcon build

# 특정 패키지만 빌드
colcon build --packages-select my_robot_pkg

# 빌드 후 환경 설정 (항상 필요)
source install/setup.bash

# 패키지 생성 (Python)
ros2 pkg create --build-type ament_python my_robot_pkg

# 패키지 생성 (C++)
ros2 pkg create --build-type ament_cmake my_robot_pkg
```

## 런치 파일(Launch File)

여러 노드를 동시에 실행해야 하는 경우 **런치 파일(launch file)** 을 사용합니다. ROS 2의 런치 파일은 Python으로 작성하며, 노드 구성, 파라미터 설정, 네임스페이스 지정 등을 선언적으로 정의합니다.

```python
# my_robot.launch.py (개념 예시)
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_robot_pkg',
            executable='my_node',
            name='my_robot_node',
            parameters=[{'use_sim_time': True}]
        ),
        Node(
            package='rviz2',
            executable='rviz2',
            name='rviz2'
        ),
    ])
```

```bash
# 런치 파일 실행
ros2 launch my_robot_pkg my_robot.launch.py
```

## ROS 1과의 주요 차이

ROS 1(Noetic 등)에서 ROS 2로 전환하면서 반드시 알아야 할 변화들입니다.

```text
┌─────────────────────┬──────────────────────┬──────────────────────┐
│ 항목                │ ROS 1                │ ROS 2                │
├─────────────────────┼──────────────────────┼──────────────────────┤
│ 마스터 노드         │ roscore 필수          │ 불필요 (DDS 분산)    │
│ 빌드 시스템         │ catkin               │ colcon + ament       │
│ 통신 미들웨어       │ XMLRPC/TCP/UDP       │ DDS                  │
│ 실시간 지원         │ 없음                 │ QoS + RTOS 지원      │
│ 보안                │ 없음                 │ DDS-Security         │
│ 파이썬 버전         │ Python 2/3           │ Python 3 전용        │
│ 런치 파일 형식      │ XML (.launch)        │ Python (.launch.py)  │
│ 메시지 정의 위치    │ package/msg/         │ 동일 (구조 유지)     │
│ 콜백 실행           │ roslaunch spin       │ rclcpp::spin / executor│
└─────────────────────┴──────────────────────┴──────────────────────┘
```

ros_bridge 패키지를 사용하면 ROS 1과 ROS 2 노드가 공존하는 환경을 구성할 수 있습니다. 레거시 시스템과의 통합이 필요한 경우 활용합니다.

## 파라미터 시스템

ROS 2의 **파라미터(parameter)** 는 노드별로 관리됩니다. ROS 1의 전역 파라미터 서버와 달리, 각 노드가 자신의 파라미터를 소유하며 외부에서 조회·설정할 수 있습니다.

```bash
# 노드의 파라미터 목록 조회
ros2 param list /my_robot_node

# 파라미터 값 조회
ros2 param get /my_robot_node use_sim_time

# 파라미터 값 설정
ros2 param set /my_robot_node max_velocity 2.0

# YAML 파일로 파라미터 저장/로드
ros2 param dump /my_robot_node --output-dir .
```

ROS 2 기초를 이해했다면, 다음 챕터에서는 NVIDIA가 ROS 2를 하드웨어 가속으로 확장한 Isaac ROS와 GEMs(GPU-accelerated Extensions for ROS)를 살펴봅니다.
