## Isaac Sim과 ROS 2의 연결

Isaac Sim은 단독으로도 강력한 시뮬레이터이지만, 실제 로봇 소프트웨어 스택과 연결할 때 진정한 가치를 발휘합니다. **ROS 2 Bridge Extension**은 Isaac Sim의 시뮬레이션 세계와 ROS 2 생태계를 연결하는 다리 역할을 합니다.

이 연결을 통해 Isaac Sim에서 동작하는 가상 로봇은 실제 ROS 2 로봇처럼 동작합니다. Nav2, MoveIt 2, Isaac ROS GEMs 등 실제 배포 시 사용할 ROS 2 소프트웨어를 시뮬레이션 단계에서 그대로 테스트할 수 있습니다.

```text
┌──────────────────────────────────────────────────────────────────────┐
│                   Isaac Sim + ROS 2 통합 환경                        │
├─────────────────────────────┬────────────────────────────────────────┤
│      Isaac Sim              │        ROS 2 생태계                    │
│                             │                                        │
│  ┌─────────────────────┐    │    ┌──────────────────────────────┐   │
│  │  가상 로봇          │    │    │  Nav2 내비게이션 스택        │   │
│  │  - 관절 상태        │────────▶│  MoveIt 2 플래닝             │   │
│  │  - 카메라 이미지    │◀───────│  Isaac ROS GEMs (인식)       │   │
│  │  - LiDAR 포인트     │    │    │  사용자 정의 노드            │   │
│  │  - IMU 데이터       │    │    └──────────────────────────────┘   │
│  └─────────────────────┘    │                                        │
│           │                 │                                        │
│  ROS 2 Bridge Extension     │                                        │
│  (OmniGraph + Action Graph) │                                        │
│           │                 │                                        │
│  DDS (Fast DDS 기본값)──────┼──────────────────────────────         │
└─────────────────────────────┴────────────────────────────────────────┘
```

## ROS 2 Bridge Extension 활성화

Isaac Sim에서 ROS 2 Bridge를 사용하려면 먼저 Extension을 활성화해야 합니다.

```text
Isaac Sim GUI 기준 활성화 경로:
Window → Extensions → 검색창에 "ROS2 Bridge" 입력 → Enable 체크
```

또는 Python API로 프로그래밍 방식으로 활성화할 수 있습니다.

```python
# Isaac Sim Python 스크립트에서 ROS 2 Bridge 활성화 (개념 예시)
import omni.ext

manager = omni.kit.app.get_app().get_extension_manager()
manager.set_extension_enabled_immediate("omni.isaac.ros2_bridge", True)
```

Extension이 활성화되면 Isaac Sim과 동일 머신의 ROS 2 환경이 자동으로 연결됩니다. DDS 도메인 ID가 일치하는지 확인합니다(기본값: 0).

## OmniGraph: 데이터 흐름 구성

Isaac Sim에서 ROS 2 퍼블리시/구독 설정은 **OmniGraph(Action Graph)** 로 구성합니다. OmniGraph는 Isaac Sim의 비주얼 스크립팅 시스템으로, 노드를 연결해 데이터 처리 파이프라인을 정의합니다.

```text
OmniGraph 예시: 카메라 이미지 → ROS 2 토픽 퍼블리시

[On Simulation Step] ──▶ [Isaac Create Render Product]
                                    │
                                    ▼
                     [Isaac Camera Helper]
                     (camera_info, image_raw 토픽 설정)
                                    │
                                    ▼
                     [ROS2 Camera Info Publisher]
                     [ROS2 Image Publisher]
                     (토픽: /camera/camera_info, /camera/image_raw)
```

GUI에서 Action Graph 에디터를 열고 노드를 드래그·연결하는 방식으로 설정합니다. 복잡한 센서 설정도 JSON/USDA 파일로 저장하여 재사용할 수 있습니다.

## 센서 데이터 퍼블리시

### 카메라

Isaac Sim의 가상 카메라를 ROS 2 토픽으로 퍼블리시하면 표준 `sensor_msgs/Image`와 `sensor_msgs/CameraInfo` 메시지를 얻습니다.

```bash
# Isaac Sim에서 카메라 토픽이 퍼블리시되면 ROS 2 측에서 확인
ros2 topic list
# /camera/camera_info
# /camera/image_raw
# /camera/depth/image_rect_raw  (RGB-D 카메라의 경우)

# 이미지 데이터 확인
ros2 topic echo /camera/image_raw --field header
```

### LiDAR(포인트 클라우드)

Isaac Sim의 RTX LiDAR 센서는 `sensor_msgs/PointCloud2` 형식으로 퍼블리시됩니다. Nav2의 costmap 레이어가 이 데이터를 직접 소비합니다.

```bash
# LiDAR 포인트 클라우드 토픽 확인
ros2 topic hz /scan          # 2D LiDAR (LaserScan)
ros2 topic hz /point_cloud   # 3D LiDAR (PointCloud2)
```

### IMU

가상 IMU는 `sensor_msgs/Imu` 메시지를 퍼블리시합니다. 각속도(angular velocity)와 선가속도(linear acceleration)를 포함합니다.

### 관절 상태(Joint States)

로봇의 관절 위치, 속도, 토크 정보는 `sensor_msgs/JointState` 토픽으로 퍼블리시됩니다. MoveIt 2가 이 토픽을 구독하여 현재 로봇 상태를 파악합니다.

```bash
# 관절 상태 실시간 확인
ros2 topic echo /joint_states
```

## 관절 명령 수신

반대 방향의 통신도 필요합니다. ROS 2 컨트롤러 노드에서 Isaac Sim 가상 로봇의 관절로 명령을 전송합니다.

`ros2_control` 프레임워크와 Isaac Sim을 연동하면, 실제 로봇 하드웨어 인터페이스와 동일한 방식으로 시뮬레이션 관절을 제어할 수 있습니다.

```text
명령 흐름:

MoveIt 2 / Nav2
    │
    ▼ (ros2_control FollowJointTrajectory 액션)
Joint Trajectory Controller
    │
    ▼ (isaac_ros2_control 플러그인)
Isaac Sim 관절 액추에이터
    │
    ▼
가상 로봇 관절 이동
```

## SIL(Software-in-the-Loop) 테스트

**SIL(Software-in-the-Loop)** 테스트는 실제 하드웨어 없이 소프트웨어 전체 스택을 시뮬레이션 환경에서 검증하는 방법론입니다.

```text
SIL vs HIL vs 실제 로봇 비교:

┌───────────────┬──────────────────────────────┬────────────────────┐
│ 방법           │ 내용                          │ 단계               │
├───────────────┼──────────────────────────────┼────────────────────┤
│ SIL            │ 소프트웨어 + 시뮬레이터       │ 개발 초기          │
│ HIL            │ 소프트웨어 + 실제 하드웨어    │ 통합 테스트        │
│ 실제 로봇      │ 전체 시스템                  │ 최종 검증 및 배포  │
└───────────────┴──────────────────────────────┴────────────────────┘
```

Isaac Sim + ROS 2 Bridge로 구성한 SIL 환경의 이점:

1. **반복성(Reproducibility)**: 정확히 동일한 시나리오를 무한 반복 실행할 수 있습니다. 실제 로봇 실험에서는 환경 변수(조명, 마찰, 배터리 상태)가 달라집니다.
2. **안전성**: 위험한 시나리오(충돌, 낙하, 화재)를 안전하게 테스트할 수 있습니다.
3. **속도**: 실시간보다 빠르게 시뮬레이션을 가속할 수 있습니다.
4. **병렬화**: 수십 개의 시나리오를 동시에 실행하여 회귀 테스트를 자동화할 수 있습니다.

## 실용적인 SIL 워크플로

```bash
# 1. Isaac Sim 헤드리스 모드로 실행 (GUI 없이, 서버에서 실행 가능)
#    (실제 명령은 Isaac Sim 버전 및 설치 방식에 따라 다름 — 공식 문서 확인)

# 2. ROS 2 내비게이션 스택 실행
ros2 launch nav2_bringup navigation_launch.py use_sim_time:=true

# 3. RViz로 시각화 확인
ros2 launch nav2_bringup rviz_launch.py

# 4. 내비게이션 목표 전송
ros2 action send_goal /navigate_to_pose \
  nav2_msgs/action/NavigateToPose \
  "{pose: {header: {frame_id: map}, pose: {position: {x: 3.0, y: 1.0, z: 0.0}, orientation: {w: 1.0}}}}"
```

## use_sim_time 파라미터

Isaac Sim과 ROS 2를 연동할 때 반드시 설정해야 하는 것이 **use_sim_time** 파라미터입니다. 이 파라미터를 `true`로 설정하면 ROS 2 노드가 시스템 시계 대신 Isaac Sim이 퍼블리시하는 `/clock` 토픽 시간을 사용합니다.

가속 시뮬레이션(real-time factor > 1)에서 센서 데이터와 제어 루프의 타임스탬프가 올바르게 동기화되려면 use_sim_time 설정이 필수입니다.

```bash
# 시뮬레이션 시간 토픽 확인
ros2 topic echo /clock

# 노드 실행 시 파라미터 지정
ros2 run my_pkg my_node --ros-args -p use_sim_time:=true
```

Isaac Sim ROS 2 Bridge는 시뮬레이션과 실제 로봇 소프트웨어 스택 간의 경계를 허뭅니다. 다음 챕터에서는 이 통합 환경 위에서 동작하는 완전한 인식 파이프라인을 살펴봅니다.
