## Isaac ROS란 무엇인가

**Isaac ROS**는 NVIDIA가 개발한 하드웨어 가속 ROS 2 패키지 모음입니다. 일반 ROS 2 패키지와 인터페이스는 동일하지만, 내부적으로 NVIDIA GPU·VPI(Vision Programming Interface)·TensorRT·CUDA 등 NVIDIA 하드웨어 가속 기술을 활용해 처리 속도를 크게 향상시킵니다.

Isaac ROS는 특히 Jetson 플랫폼(Jetson Orin, AGX Orin 등)에서 자율 로봇의 실시간 인식·내비게이션·조작 파이프라인을 구동하는 데 최적화되어 있습니다. 데이터센터 GPU 서버뿐만 아니라 엣지 디바이스에서도 동일한 코드베이스로 동작합니다.

```text
┌─────────────────────────────────────────────────────────────────────┐
│                      Isaac ROS 생태계                                │
├─────────────────────────────────────────────────────────────────────┤
│  Isaac ROS GEMs (GPU-Accelerated Extensions)                        │
│  ┌──────────────┐ ┌───────────────┐ ┌──────────────┐ ┌──────────┐  │
│  │  인식 GEMs   │ │  VSLAM/위치추정│ │DNN Inference │ │조작/제어 │  │
│  │(Detection,   │ │(Visual Odom,  │ │(TAO Toolkit, │ │(cuMotion,│  │
│  │ Segmentation)│ │ SLAM)         │ │ TensorRT)    │ │ Isaac Manipulator)│
│  └──────────────┘ └───────────────┘ └──────────────┘ └──────────┘  │
├─────────────────────────────────────────────────────────────────────┤
│  NITROS (NVIDIA Isaac Transport for ROS) — zero-copy GPU 전송       │
├─────────────────────────────────────────────────────────────────────┤
│  ROS 2 rclcpp/rmw 레이어                                            │
├─────────────────────────────────────────────────────────────────────┤
│  NVIDIA GPU / CUDA / TensorRT / VPI / cuDNN                         │
├─────────────────────────────────────────────────────────────────────┤
│  Jetson Orin / 데이터센터 GPU                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## GEMs: GPU-Accelerated Extensions for ROS

**GEMs(GPU-Accelerated Extensions for ROS)** 는 Isaac ROS의 핵심 패키지 단위입니다. 각 GEM은 특정 기능(예: 객체 검출, 스테레오 깊이 추정, 포즈 추정)을 GPU에서 가속 실행하는 ROS 2 노드 또는 노드 조합으로 구성됩니다.

GEM을 사용하는 이유는 단순히 "빠르기 때문"만이 아닙니다. 로봇 인식 파이프라인에서 CPU 기반 처리는 복잡한 DNN 모델을 실시간 속도(30fps 이상)로 실행하기에 역부족인 경우가 많습니다. GEM은 CPU–GPU 데이터 전송을 최소화하면서 GPU 병렬 처리를 최대한 활용합니다.

## NITROS: Zero-Copy GPU 전송

일반적인 ROS 2 파이프라인에서 카메라 이미지가 GPU 처리 노드를 거칠 때마다 발생하는 문제가 있습니다. 이미지 데이터가 CPU 메모리(RAM)와 GPU 메모리 사이를 반복적으로 복사되는 것입니다. 고해상도 이미지나 포인트 클라우드에서는 이 복사 비용이 실제 처리 시간보다 크게 될 수 있습니다.

**NITROS(NVIDIA Isaac Transport for ROS)** 는 이 문제를 해결합니다. GPU 메모리에 올라간 데이터를 복사 없이(zero-copy) 다음 노드로 전달하는 전송 메커니즘입니다. NITROS를 지원하는 GEM들이 연속으로 연결된 파이프라인에서는 GPU 메모리가 처음부터 끝까지 유지됩니다.

```text
NITROS 없는 파이프라인 (일반 ROS 2):

[카메라 드라이버]──GPU→CPU copy──▶[토픽 직렬화]──CPU→GPU copy──▶[DNN 검출 노드]
                  ^^^^^^^^^^^^                     ^^^^^^^^^^^^
                  매 프레임마다 복사 비용 발생

NITROS 적용 파이프라인:

[카메라 드라이버]─────GPU 메모리 공유(zero-copy)─────▶[DNN 검출 노드]
                  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                  GPU 포인터만 전달, 실제 복사 없음
```

NITROS는 ROS 2 rmw 레이어 위에서 투명하게 동작합니다. 개발자가 NITROS API를 직접 호출하지 않아도, NITROS 지원 GEM들을 연결하면 자동으로 zero-copy 경로가 활성화됩니다.

## 대표 GEM: 인식(Perception)

### Isaac ROS Object Detection

NVIDIA TAO Toolkit으로 학습하거나 NVIDIA NGC에서 다운로드한 객체 검출 모델(DetectNet, YOLO, SSD 등)을 TensorRT로 최적화하여 실행합니다. 카메라 이미지를 입력받아 바운딩 박스와 클래스 레이블을 토픽으로 출력합니다.

### Isaac ROS Image Segmentation

픽셀 단위의 의미론적 분할(semantic segmentation)을 GPU에서 실행합니다. 장면의 각 픽셀을 "바닥", "장애물", "조작 가능 물체" 등으로 분류하여, 로봇이 환경을 이해하는 데 필요한 밀도 높은 인식 정보를 제공합니다.

### Isaac ROS Stereo Depth

스테레오 카메라의 좌우 이미지 쌍으로부터 깊이 맵(depth map)을 생성합니다. GPU에서 디스패리티(disparity) 계산을 수행하며, LiDAR 없이도 3D 공간 정보를 얻을 수 있습니다.

### isaac_ros_apriltag

GPU에서 AprilTag 마커를 검출하고 6-DoF 포즈를 추정합니다. 작업 공간 교정, 물체 위치 기준점 설정, 카메라-로봇 캘리브레이션 등에 활용됩니다.

## 대표 GEM: VSLAM과 위치 추정

**Isaac ROS Visual SLAM**은 카메라 이미지만으로 로봇의 위치와 자세를 추정하는 VSLAM(Visual Simultaneous Localization and Mapping) 기능을 제공합니다. NVIDIA cuVSLAM 라이브러리를 기반으로 하며, 스테레오 또는 RGB-D 카메라를 지원합니다.

GPS가 없는 실내 환경에서 로봇이 자신의 위치를 파악하고 지도를 동시에 구축하는 데 필요한 핵심 기능입니다.

## 대표 GEM: DNN 추론(Inference)

**Isaac ROS DNN Inference** GEM은 TensorRT 엔진 파일(.engine 또는 .trt)을 로드하여 ROS 2 노드로 실행하는 범용 추론 래퍼입니다. 입력 토픽으로 이미지나 텐서를 받아 추론 결과를 토픽으로 출력합니다.

Isaac Sim에서 합성 데이터로 학습한 모델을 Isaac ROS DNN Inference로 배포하면, 시뮬레이션에서 실제 로봇 소프트웨어 스택으로의 전환이 자연스럽게 이루어집니다.

## 대표 GEM: 조작(Manipulation)

**Isaac ROS cuMotion**은 모션 플래닝(motion planning)을 GPU에서 수행합니다. MoveIt 2와 통합하여 충돌 회피 경로를 밀리초 수준의 짧은 시간에 계산합니다. 전통적인 CPU 기반 플래너(OMPL 등)보다 빠른 응답 속도가 필요한 조작 작업에 유리합니다.

## Jetson 플랫폼과의 통합

Isaac ROS GEMs는 Jetson Orin 시리즈를 주요 타겟 하드웨어로 지원합니다. Jetson은 CPU, GPU, DLA(Deep Learning Accelerator), PVA(Programmable Vision Accelerator)를 단일 SoC에 통합한 엣지 AI 플랫폼입니다.

```text
┌──────────────────────────────────────────────────────┐
│              Jetson Orin SoC                         │
├──────────────┬───────────────┬───────────────────────┤
│  ARM Cortex  │  Ampere GPU   │  DLA (Deep Learning   │
│  CPU 코어    │  (2048 CUDA)  │  Accelerator) x2      │
├──────────────┴───────────────┴───────────────────────┤
│  PVA (이미지 처리 전용 프로세서)                     │
├──────────────────────────────────────────────────────┤
│  통합 메모리 (CPU/GPU 공유) — NITROS zero-copy 기반  │
└──────────────────────────────────────────────────────┘
```

Jetson의 **통합 메모리(unified memory)** 구조는 NITROS와 특히 궁합이 좋습니다. CPU와 GPU가 물리적으로 동일한 메모리를 공유하므로, zero-copy 전송이 하드웨어 레벨에서 자연스럽게 지원됩니다.

## Isaac ROS 시작하기

Isaac ROS는 공식 문서에서 Dev Container(Docker 기반) 환경 사용을 강력히 권장합니다. 의존성 관리가 복잡하기 때문입니다.

```bash
# Isaac ROS 공식 권장 방법: isaac_ros_common의 Dev Container 스크립트 사용
# 실제 상세 절차는 공식 문서 확인 필수
# https://nvidia-isaac-ros.github.io/getting_started/index.html

# Dev Container 진입 후 워크스페이스 빌드
colcon build --symlink-install

# 환경 소스
source install/setup.bash

# Isaac ROS Object Detection 예시 런치 (실제 설정은 공식 문서 참조)
ros2 launch isaac_ros_object_detection isaac_ros_detectnet.launch.py
```

Isaac ROS GEMs의 핵심 가치는 NITROS를 통한 zero-copy GPU 파이프라인과 TensorRT 최적화 추론의 결합에 있습니다. 다음 챕터에서는 Isaac ROS와 함께 자율 내비게이션을 담당하는 Nav2, 조작을 위한 MoveIt 2를 살펴봅니다.
