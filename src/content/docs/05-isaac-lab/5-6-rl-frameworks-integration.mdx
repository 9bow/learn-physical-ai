## Isaac Lab의 프레임워크 독립성

Isaac Lab 자체는 특정 RL 알고리즘을 내장하지 않습니다. 환경(environment)만 제공하고, 실제 정책 업데이트는 외부 RL 라이브러리에 위임합니다. 이 설계는 Isaac Lab이 "시뮬레이션 + 환경 관리" 레이어에 집중하고, 알고리즘 선택은 사용자에게 맡기는 분리(separation of concerns) 원칙을 따릅니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              Isaac Lab - RL 프레임워크 연동 구조              │
│                                                              │
│  ┌────────────────────────────────────────────┐             │
│  │            RL 라이브러리 레이어              │             │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──┐ │             │
│  │  │ RSL-RL  │ │rl_games │ │  SKRL   │ │SB3│ │             │
│  │  └────┬────┘ └────┬────┘ └────┬────┘ └─┬─┘ │             │
│  └───────┼───────────┼───────────┼─────────┼───┘             │
```

### RSL-RL

ETH 취리히 로보틱 시스템 랩(RSL)이 개발한 경량 PPO 구현체입니다. Isaac Lab 공식 보행 태스크(ANYmal, Unitree 등)에서 주로 사용됩니다. 빠른 실행 속도와 Isaac Lab과의 긴밀한 통합이 강점입니다.

RSL-RL은 `OnPolicyRunner` 클래스를 중심으로 설계됩니다.

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
from rsl_rl.runners import OnPolicyRunner

runner = OnPolicyRunner(
    env=wrapped_env,          # Isaac Lab 환경 (래퍼 적용)
    train_cfg=train_cfg,      # PPO 하이퍼파라미터
    log_dir="logs/my_exp",
    device="cuda:0"
)
runner.learn(num_learning_iterations=1500, init_at_random_ep_len=True)
```

설정은 Python 데이터클래스로 정의하며, 네트워크 아키텍처(MLP, LSTM), PPO 하이퍼파라미터(clip\_range, entropy\_coef, learning\_rate 등)를 지정합니다.

### rl_games

더 넓은 알고리즘 지원(PPO, SAC, A2C 등)과 YAML 기반 설정이 특징입니다. OmniIsaacGymEnvs 시절부터 NVIDIA 공식 파트너였으며, Isaac Lab에서도 지속적으로 지원됩니다.

```bash
# rl_games로 학습 실행
./isaaclab.sh -p scripts/reinforcement_learning/rl_games/train.py \
  --task Isaac-Cartpole-v0 \
  --num_envs 512 \
  --headless
```

### SKRL

모듈식 설계로 에이전트, 메모리, 정책 구성 요소를 독립적으로 교체할 수 있습니다. 연구 목적으로 다양한 알고리즘을 실험하거나 새로운 알고리즘을 구현할 때 유용합니다.

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
from skrl.agents.torch.ppo import PPO, PPO_DEFAULT_CONFIG
from skrl.envs.wrappers.torch import wrap_env

env = wrap_env(isaac_lab_env)  # Isaac Lab 환경 래핑

agent = PPO(
    models={"policy": policy_net, "value": value_net},
    memory=memory,
    cfg=PPO_DEFAULT_CONFIG,
    observation_space=env.observation_space,
    action_space=env.action_space,
)
trainer.train()
```

## Wrapper 구조

RL 라이브러리들은 일반적으로 Gymnasium(구 OpenAI Gym) 호환 인터페이스를 기대합니다. Isaac Lab 환경은 자체 인터페이스를 사용하므로, 각 RL 라이브러리마다 **래퍼(wrapper)**를 통해 인터페이스를 변환합니다.

```text
RL 라이브러리가 기대하는 인터페이스:
  obs, info = env.reset()
  obs, reward, terminated, truncated, info = env.step(action)

Isaac Lab 환경이 제공하는 인터페이스:
  obs_dict, extra = env.reset()
  obs_dict, reward, terminated, timed_out, extra = env.step(action)

래퍼(Wrapper)가 변환:
  dict 형태 obs → 단일 텐서
  timed_out → truncated
  Isaac Lab 타임스텝 → Gymnasium 표준
```

각 RL 라이브러리에 맞는 래퍼는 Isaac Lab 저장소의 `isaaclab_rl` 또는 각 라이브러리 연동 스크립트에 포함되어 있습니다.

## 하이퍼파라미터 설정

RSL-RL을 예로 들면, 주요 PPO 하이퍼파라미터는 다음과 같이 설정합니다.

```python
# 개념 이해용 의사코드 — 실제 API는 공식 문서 확인
@dataclass
class PPORunnerCfg:
    # 학습 파라미터
    num_steps_per_env: int = 24      # 환경당 수집 스텝 수
    max_iterations: int = 1500       # 총 PPO 업데이트 횟수
    save_interval: int = 50          # 체크포인트 저장 간격

    # PPO 알고리즘 파라미터
    clip_param: float = 0.2          # PPO 클리핑 계수
    entropy_coef: float = 0.01       # 엔트로피 보너스
    num_learning_epochs: int = 5     # 미니배치 업데이트 횟수
    num_mini_batches: int = 4        # 미니배치 수

    # 최적화 파라미터
    learning_rate: float = 1.0e-3
    max_grad_norm: float = 1.0

    # 네트워크 아키텍처
    policy_class_name: str = "ActorCritic"
    actor_hidden_dims: list = field(default_factory=lambda: [512, 256, 128])
    critic_hidden_dims: list = field(default_factory=lambda: [512, 256, 128])
```

## 학습 로깅

### TensorBoard

Isaac Lab은 기본적으로 TensorBoard 로깅을 지원합니다.

```bash
# 학습 실행 (자동으로 logs/ 에 저장)
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Flat-Anymal-C-v0 \
  --num_envs 1024 \
  --headless

# TensorBoard 시각화
tensorboard --logdir logs/rsl_rl/Isaac-Velocity-Flat-Anymal-C-v0/
```

기록되는 주요 지표:

- `Train/mean_reward`: 평균 에피소드 보상
- `Train/mean_episode_length`: 평균 에피소드 길이
- `Train/value_function_loss`: 가치 함수 손실
- `Train/surrogate_loss`: PPO 대리 손실
- 각 보상 항(term)별 개별 보상값

### Weights & Biases (W&B)

W&B를 사용하면 실험 추적, 하이퍼파라미터 비교, 팀 공유가 용이합니다.

```bash
# W&B 로깅 활성화
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Rough-Anymal-C-v0 \
  --num_envs 2048 \
  --headless \
  --logger wandb \
  --wandb_project my_project \
  --wandb_entity my_team
```

## 소결

Isaac Lab은 RSL-RL, rl\_games, SKRL, Stable-Baselines3 등 다양한 RL 라이브러리를 래퍼(wrapper)를 통해 연결하며, 특정 알고리즘에 종속되지 않는 구조를 갖습니다. 학습 실행은 태스크별 스크립트를 통해 이루어지며, TensorBoard와 W&B로 실험 결과를 추적할 수 있습니다. 보행 태스크 입문이라면 RSL-RL이, 다양한 알고리즘 실험이 목적이라면 SKRL이 적합합니다.
