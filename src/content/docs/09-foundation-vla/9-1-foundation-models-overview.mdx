## 왜 파운데이션 모델인가

2010년대 로봇 학습의 지배적 패러다임은 **과제별 전문 정책(task-specific policy)** 이었습니다. 컵을 집는 로봇, 문을 여는 로봇, 물건을 분류하는 로봇은 각각 별도로 설계되고 학습되었습니다. 이 방식은 특정 환경에서 뛰어난 성능을 낼 수 있지만, 치명적인 약점이 있습니다.

- 새로운 과제마다 수천 건의 시연 데이터와 재학습이 필요합니다.
- 물체 외관·배경 조명·카메라 위치가 조금만 달라져도 성능이 급감합니다.
- 두 과제를 동시에 처리하거나 순서를 바꾸면 대부분 실패합니다.

이런 한계를 돌파하기 위해 자연어 처리(NLP) 분야에서 나타난 패러다임인 **파운데이션 모델(foundation model)** 이 로봇공학에 도입되기 시작했습니다. NLP에서 대규모 언어 모델(Large Language Model, LLM)이 방대한 텍스트 데이터로 사전학습(pre-training)된 뒤 다양한 다운스트림 과제에 파인튜닝(fine-tuning)되는 것처럼, 로봇 파운데이션 모델은 **다수의 로봇, 다양한 환경, 수백 가지 과제**에 걸쳐 수집된 대규모 데이터로 사전학습됩니다.

## 파운데이션 모델의 정의

**로봇 파운데이션 모델(robot foundation model)** 은 다음 특성을 갖는 신경망 정책입니다.

1. **대규모 다중 과제 데이터로 사전학습**: 특정 로봇이나 환경에 한정되지 않고, 여러 종류의 조작(manipulation), 이동(locomotion), 항법(navigation) 데이터를 통합합니다.
2. **언어 조건부(language-conditioned)**: 자연어 명령("빨간 컵을 오른쪽 선반에 올려놔")으로 과제를 지정할 수 있습니다.
3. **전이 가능성(transferability)**: 소량의 새로운 데이터로 파인튜닝하면 처음 보는 환경·과제에서도 작동합니다.

```text
┌───────────────────────────────────────────────────────────────┐
│              로봇 파운데이션 모델 패러다임                     │
├──────────────────────┬────────────────────────────────────────┤
│   기존 접근 방식     │   파운데이션 모델 접근 방식            │
├──────────────────────┼────────────────────────────────────────┤
│ 과제별 정책 설계     │ 단일 대형 모델 → 다중 과제             │
│ 수천 건 시연 필요    │ 사전학습 후 소량(10~100건) 파인튜닝    │
│ 환경 변화에 취약     │ 일반화 능력 내재                       │
│ 언어 이해 별도 모듈  │ 언어·시각·행동 통합 처리              │
│ 재사용 어려움        │ 새 과제에 전이·적응                    │
└──────────────────────┴────────────────────────────────────────┘
```

## Generalist Policy란 무엇인가

**Generalist policy(범용 정책)** 는 단일 모델이 여러 과제, 여러 환경, 여러 로봇 형태에서 작동할 수 있는 정책입니다. 인간의 손재주를 생각해보면 이해가 쉽습니다. 사람은 컵을 집고, 열쇠를 돌리고, 피아노를 치는 동작을 각각 별도로 "프로그래밍"받지 않습니다. 어릴 때부터 수많은 물체 조작 경험을 축적하고, 새로운 도구를 만나도 비교적 빠르게 적응합니다. Generalist policy는 이 능력을 모델 안에 담으려 합니다.

핵심 개념인 **행동 토큰화(action tokenization)** 는 로봇의 연속적 모터 명령(관절 각도, 그리퍼 개폐 등)을 이산 토큰(discrete token)으로 변환합니다. 이렇게 하면 언어 모델의 토큰 예측 메커니즘을 그대로 활용하여 행동 시퀀스를 생성할 수 있습니다.

## LLM·VLM이 로봇에 들어오다

대형 언어 모델과 시각-언어 모델(Vision-Language Model, VLM)의 발전은 로봇 정책 설계에 새로운 가능성을 열었습니다.

**언어 모델의 기여:**
- 자연어 명령 이해 및 장기 계획(task planning) 능력
- 상식(common sense) 추론: "뜨거운 냄비를 집기 전에 장갑을 착용"
- 인간과의 자연스러운 소통 인터페이스

**시각-언어 모델의 기여:**
- 이미지와 언어를 공동 표현 공간에서 처리
- 새로운 물체를 언어 설명만으로 인식("초록색 직사각형 블록")
- 장면 이해(scene understanding)와 관계 추론

```text
┌─────────────────────────────────────────────────────────────────┐
│                VLM 기반 로봇 정책의 정보 흐름                    │
│                                                                 │
│  언어 명령  ──┐                                                 │
│              ├──▶  VLM Backbone  ──▶  행동 디코더  ──▶  모터   │
│  카메라 영상──┘   (공유 인코더)       (Action Head)    명령    │
│                                                                 │
│  사전학습: 인터넷 규모 텍스트·이미지                            │
│  파인튜닝:  로봇 시연 데이터(로봇 팔, 카메라, 조인트 각도)      │
└─────────────────────────────────────────────────────────────────┘
```

## 대규모 다중 로봇 데이터셋의 역할

파운데이션 모델의 성능은 학습 데이터의 규모와 다양성에 크게 의존합니다. 이를 위해 여러 연구 기관이 협력하여 구축한 대규모 데이터셋이 등장했습니다.

**Open X-Embodiment** 데이터셋은 수십 개 연구 기관의 로봇 시연 데이터를 통합한 대규모 다중 로봇 데이터셋으로, Google DeepMind가 주도했습니다. 다양한 로봇 형태(robot morphology)와 과제 종류를 포함하며, 이 데이터로 학습한 모델이 단일 환경 데이터로 학습한 모델보다 더 높은 일반화 성능을 보임을 보였습니다.

이 외에도 **Bridge Data**, **RoboSet**, **DROID** 등 다양한 데이터셋이 공개되어 있으며, 학습 데이터의 **다양성(diversity)** 이 데이터 크기만큼 중요하다는 것이 공통된 발견입니다.

## 파운데이션 모델의 현재 한계

파운데이션 모델이 유망하지만, 아직 해결해야 할 과제도 분명합니다.

- **추론 속도**: 대형 트랜스포머 모델은 실시간 제어(100Hz 이상)에 필요한 속도를 내기 어렵습니다. 경량화 또는 계층적 구조(고수준 계획 + 저수준 제어기 분리)가 연구 중입니다.
- **Sim-to-Real 갭**: 시뮬레이터에서 생성한 데이터로 학습하더라도 실물 로봇에서의 성능 차이가 남습니다.
- **접촉 풍부 조작**: 물체를 강하게 쥐거나 도구를 정밀하게 다루는 동작은 여전히 어렵습니다.
- **안전성**: 파운데이션 모델이 예상치 못한 행동을 출력할 때의 안전 보장이 미흡합니다.
