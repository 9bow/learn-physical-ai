## 왜 로봇 정책 평가는 어려운가

소프트웨어 함수를 평가하는 일은 비교적 명확합니다. 입력을 주고, 출력을 기대값과 비교하면 됩니다. 로봇 정책 평가는 근본적으로 다릅니다.

**물리 세계의 불확실성:**
- 물체의 마찰 계수, 미세한 위치 차이, 조명 변화 같은 요소가 결과를 바꿉니다.
- 같은 정책, 같은 초기 조건에서도 시도마다 결과가 다를 수 있습니다.
- 따라서 단 한 번의 시도로 정책의 품질을 판단하는 것은 신뢰할 수 없습니다.

**실기 평가와 시뮬레이션 평가:**

```text
┌──────────────────────────────────────────────────────────────────┐
│             실기 평가 vs 시뮬레이션 평가 비교                     │
├──────────────────────┬───────────────────────────────────────────┤
│   실기 평가           │   시뮬레이션 평가                         │
├──────────────────────┼───────────────────────────────────────────┤
│ 진정한 성능 반영      │ Sim-to-Real 갭 존재                       │
│ 재현성 낮음           │ 완벽한 재현성                             │
│ 시간·비용 큼          │ 빠르고 저렴                               │
│ 하드웨어 필요         │ 하드웨어 불필요                           │
│ 물리적 위험           │ 안전                                      │
│ 희귀 시나리오 어려움  │ 임의 시나리오 생성 가능                   │
└──────────────────────┴───────────────────────────────────────────┘
```

이 때문에 로봇 연구에서는 시뮬레이션 벤치마크에서 먼저 평가하고, 핵심 결과를 실제 하드웨어에서 검증하는 이중 평가가 일반적입니다.

## 성공률(Success Rate) 측정

로봇 정책의 주요 평가 지표는 **성공률(success rate)** 입니다. N번의 시도 중 과제를 성공적으로 완료한 비율입니다.

성공률 측정에서 주의해야 할 함정들:

**1. 시도 수 부족**: 성공률 70%인 정책을 10번 시도하면 결과가 크게 흔들립니다. 신뢰 구간을 보고할 때는 최소 50~100번 시도가 권장됩니다.

**2. 초기 조건 고정**: 물체 위치를 항상 같은 곳에 두면 정책이 그 위치를 "외웠을" 가능성이 있습니다. 초기 조건을 무작위화해야 진정한 일반화 능력을 평가합니다.

**3. 성공 기준 불명확**: "컵을 트레이에 올렸다"의 기준이 연구마다 다릅니다. 물체가 트레이 위에 있으면 되는가, 아니면 안정적으로 정지해야 하는가?

**4. 환경 리셋 방식**: 실패 후 환경을 리셋하는 방법이 다음 시도에 영향을 줍니다.

## 주요 벤치마크

### SimplerEnv

**SimplerEnv**는 실제 로봇 실험 결과와 상관성이 높은 시뮬레이션 환경을 제공하는 것을 목표로 합니다. Google DeepMind의 실제 로봇 데이터를 바탕으로 설계된 시뮬레이션 환경에서, VLA 모델의 성능을 빠르게 평가합니다.

주요 특징:
- 실제 로봇 실험과 시뮬레이션 결과 간 상관성을 명시적으로 분석
- 환경 무작위화 수준을 조절 가능
- OpenVLA 등 주요 VLA 모델의 공식 평가 플랫폼으로 활용

### LIBERO

**LIBERO(Lifelong Learning Benchmark for Robot Manipulation)** 는 평생 학습(lifelong learning)과 일반화를 중점적으로 평가합니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│                  LIBERO 벤치마크 구성                             │
│                                                                  │
│  LIBERO-Spatial: 같은 물체, 다른 위치 관계                       │
│  LIBERO-Object:  같은 과제, 다른 물체 종류                       │
│  LIBERO-Goal:    같은 물체, 다른 목표 상태                       │
│  LIBERO-Long:    다단계 장기 과제                                 │
│                                                                  │
│  각 분할: 50개 과제, 각 과제당 50건 시연 제공                    │
└──────────────────────────────────────────────────────────────────┘
```

LIBERO의 강점은 **일반화 유형을 명확히 분리**한다는 점입니다. 공간 일반화, 물체 일반화, 목표 일반화 중 어느 측면이 약한지 구체적으로 파악할 수 있습니다.

### CALVIN

**CALVIN(Composing Actions from Language and Vision)** 은 언어 명령의 조합성(compositionality)을 평가합니다.

핵심 아이디어: 단순 명령이 아닌 "A를 하고, B를 하고, C를 해"처럼 연속된 지시를 얼마나 잘 따를 수 있는지 측정합니다.

평가 지표:
- 1단계 성공률
- 2단계 연속 성공률
- 3단계, 4단계... (체인 길이별)

장기 과제 수행 능력을 평가하는 데 특히 유용합니다.

### Meta-World

**Meta-World**는 로봇 팔 조작 과제 50개를 표준화한 메타학습(meta-learning) 벤치마크입니다.

- 문 열기, 서랍 당기기, 못 박기 등 다양한 조작 과제
- 단일 과제 학습(ML1), 10개 과제 동시 학습(ML10), 50개 과제(ML50) 설정
- 학습 과제와 평가 과제 분리로 진정한 일반화 측정

Meta-World의 한계는 시각적 사실감이 낮아 실제 로봇과의 격차가 크다는 점입니다.

## 벤치마크 비교 요약

```text
┌────────────────┬──────────────┬──────────────┬──────────────────┐
│ 벤치마크       │ 초점         │ 물리 엔진    │ 시각적 사실감    │
├────────────────┼──────────────┼──────────────┼──────────────────┤
│ SimplerEnv     │ VLA 평가     │ MuJoCo       │ 높음(실물 대응)  │
│ LIBERO         │ 일반화 유형  │ MuJoCo+      │ 중간             │

│ Meta-World     │ 메타학습     │ MuJoCo       │ 낮음             │
└────────────────┴──────────────┴──────────────┴──────────────────┘
```

## 재현성(Reproducibility) 문제

로봇 학습 연구에서 **재현성**은 만성적인 문제입니다.

**재현성을 어렵게 만드는 요인:**

1. **랜덤 시드 의존성**: 다른 랜덤 시드로 학습하면 성능이 크게 달라질 수 있습니다.
2. **하이퍼파라미터 민감성**: 학습률, 배치 크기, 데이터 정규화 방식 등 세부 설정이 결과에 영향을 줍니다.
3. **데이터 전처리 불명확**: 논문에서 데이터 전처리 세부사항을 생략하면 재현이 어렵습니다.
4. **환경 버전 차이**: 시뮬레이터나 Python 라이브러리 버전이 달라지면 물리 동작이 미묘하게 달라집니다.

**재현성 개선 관행:**
- 학습 코드와 모델 가중치 공개
- 랜덤 시드를 여러 개 사용하여 평균과 표준편차 보고
- 정확한 환경 의존성 명시(requirements.txt, Docker 이미지)
- 전처리 코드 공개

## VLA 모델 평가 실제 사례

VLA 모델을 새로운 환경에 배포할 때 일반적인 평가 절차는 다음과 같습니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│           VLA 배포 평가 프로세스                                │
│                                                                 │
│  1. 오프라인 메트릭 확인                                        │
│     └─ 학습/검증 손실, 행동 예측 MSE                           │
│                           │                                     │
│  2. 시뮬레이션 롤아웃     ▼                                    │
│     └─ 50~100회 시도, 성공률 측정                              │
│        초기 조건 무작위화, 95% 신뢰 구간 계산                  │
│                           │                                     │
│  3. 실물 로봇 소규모 검증 ▼                                    │
│     └─ 20~30회 시도, 시뮬레이션과 격차 확인                   │
│                           │                                     │
│  4. 실물 로봇 전체 평가   ▼                                    │
│     └─ 표준 설정에서 50회 이상, 최종 성능 보고                │
└─────────────────────────────────────────────────────────────────┘
```

## 평가의 미래: 더 나은 방향

현재 벤치마크들은 여전히 실제 로봇의 복잡성을 완전히 반영하지 못합니다. 향후 중요해질 평가 요소들:

- **실패 분석(failure analysis)**: 성공률 외에 어떤 상황에서 실패하는지 분석
- **안전성 평가**: 과제 실패 시 주변 환경이나 사람에게 해를 끼치는지
- **분포 밖 일반화**: 학습 데이터와 크게 다른 상황에서의 성능
- **장기 성능**: 시간이 지남에 따른 정책 성능 저하(망각) 측정
