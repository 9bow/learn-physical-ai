## 학습 파이프라인 개요

VLA 모델 학습은 크게 두 단계로 구분됩니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│                  VLA 학습 파이프라인                              │
│                                                                  │
│  단계 1: 사전학습 (Pre-training)                                 │
│  ─────────────────────────────────                               │
│  대규모 비전-언어 데이터  ──▶  VLM Backbone 학습                │
│  (ImageNet, CC12M, ...)                                          │
│              +                                                   │
│  대규모 로봇 시연 데이터  ──▶  행동 예측 헤드 추가·학습          │
│  (Open X-Embodiment, ...)                                        │
│                                                                  │
│  단계 2: 파인튜닝 (Fine-tuning)                                  │
│  ─────────────────────────────                                   │
│  특정 환경 + 과제 데이터  ──▶  도메인 특화 적응                  │
│  (50~500건 시연)              (LoRA 또는 전체 파라미터)          │
│                                                                  │
│  배포 (Deployment)                                               │
│  ──────────────────                                              │
│  실제 로봇에서 실시간 추론 (폐루프 제어)                         │
└──────────────────────────────────────────────────────────────────┘
```

## 사전학습: 무엇을, 얼마나

VLA 사전학습은 두 종류의 데이터를 혼합하여 진행합니다.

**비전-언어 데이터 (VLM 사전학습):**
- 인터넷 규모의 이미지-텍스트 쌍 데이터
- 목적: 시각적 의미 이해, 언어 이해, 상식 추론 능력 확보
- 이 단계는 흔히 공개된 VLM(예: PaliGemma, Llama-Vision 등)을 베이스로 재활용합니다.

**로봇 시연 데이터 (Action 학습):**
- 다양한 로봇·환경·과제의 시연 궤적
- 각 궤적: 이미지 시퀀스 + 언어 명령 + 행동 시퀀스
- 대표 데이터셋: Open X-Embodiment, Bridge Data V2, DROID

사전학습은 수백~수천 GPU-시간의 계산이 필요하며, 일반 연구자가 처음부터 수행하기 어렵습니다. 실용적으로는 **공개된 사전학습 모델 가중치를 다운로드하여 파인튜닝**하는 방식을 사용합니다.

## 데이터 형식과 요구사항

VLA 학습 데이터의 기본 단위는 **에피소드(episode)** 입니다.

```text
에피소드 구조:
┌────────────────────────────────────────────────────────────────┐
│  에피소드: "빨간 컵을 트레이에 올려놓기"                        │
│                                                                │
│  타임스텝 t=0:                                                 │
│  ├─ 이미지: [H×W×3 RGB 배열]                                  │
│  ├─ 언어: "put the red cup on the tray"                        │
│  ├─ 로봇 상태: [관절1 각도, 관절2 각도, ..., 그리퍼 상태]     │
│  └─ 행동: [Δx=0.01, Δy=0.00, Δz=0.02, ..., gripper=0]       │
│                                                                │
│  타임스텝 t=1, 2, ..., T (과제 완료까지)                       │
│                                                                │
│  성공 여부: True/False                                         │
└────────────────────────────────────────────────────────────────┘
```

**Open X-Embodiment의 표준 형식 RLDS(Reinforcement Learning Dataset Specification)** 를 따르면 여러 데이터셋을 통합할 때 유리합니다.

## 행동 표현 방식 심화

VLA가 행동을 어떻게 표현하느냐는 성능에 직접 영향을 미칩니다.

### 이산 토큰화 (Discrete Tokenization)

연속값을 정해진 구간으로 분할하여 정수 인덱스로 변환합니다.

```python
# 개념 이해용 의사코드
import numpy as np

def tokenize_action(action: np.ndarray, n_bins: int = 256) -> list[int]:
    """연속 행동값을 이산 토큰으로 변환합니다."""
    # 각 차원을 [-1, 1] 범위로 정규화 후 빈 인덱스로 변환
    normalized = np.clip(action, -1.0, 1.0)
    tokens = ((normalized + 1.0) / 2.0 * (n_bins - 1)).astype(int)
    return tokens.tolist()

def detokenize_action(tokens: list[int], n_bins: int = 256) -> np.ndarray:
    """이산 토큰을 연속 행동값으로 복원합니다."""
    normalized = np.array(tokens) / (n_bins - 1) * 2.0 - 1.0
    return normalized
```

장점: LLM의 분류 손실(cross-entropy loss)을 그대로 활용 가능합니다.
단점: 양자화 오차가 발생하며, 빈 수를 늘리면 어휘 크기가 커집니다.

### 확산 기반 (Diffusion-based)

노이즈에서 시작하여 반복적 디노이징으로 행동을 생성합니다. 복잡한 다봉 분포를 자연스럽게 표현합니다. π0, Diffusion Policy 등이 이 방식을 사용합니다.

### 회귀 (Regression)

모델 출력을 평균 제곱 오차(MSE)로 직접 학습합니다. 간단하지만 행동 분포가 복잡할 때 "평균값 행동"으로 수렴하는 averaging problem이 발생합니다.

## 파인튜닝: 특정 환경 적응

사전학습된 VLA 모델을 특정 로봇·환경에 맞게 파인튜닝합니다.

### 전체 파인튜닝 (Full Fine-tuning)

모든 모델 파라미터를 업데이트합니다. 성능이 가장 좋을 수 있지만, 대형 모델에서는 GPU 메모리와 계산 비용이 매우 큽니다.

### LoRA 파인튜닝 (Parameter-Efficient Fine-tuning)

**LoRA(Low-Rank Adaptation)** 는 원본 가중치를 고정하고 소수의 저랭크 행렬만 학습합니다.

```python
# 개념 이해용 의사코드 — 실제 구현과 구조가 다를 수 있습니다
from peft import LoraConfig, get_peft_model

# 사전학습 VLA 모델 로드
model = load_pretrained_vla("openvla/openvla-7b")

# LoRA 설정: rank=16, 어텐션 레이어만 학습
lora_config = LoraConfig(
    r=16,                      # 저랭크 차원
    lora_alpha=32,             # 스케일 파라미터
    target_modules=["q_proj", "v_proj"],  # 적용 레이어
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
)

model = get_peft_model(model, lora_config)
# 전체 파라미터 중 약 1-2%만 학습 가능 상태로 전환
model.print_trainable_parameters()
# trainable params: 4,194,304 / total: 7,000,000,000 (약 0.06%)
```

LoRA를 사용하면 단일 A100 GPU에서도 대형 VLA 모델을 파인튜닝할 수 있습니다.

## 학습 루프와 손실 함수

```python
# 개념 이해용 의사코드
def training_step(batch):
    images, instructions, actions = batch

    # 행동 토큰화
    action_tokens = tokenize_action(actions)  # [B, T, A]

    # 모델 예측
    predicted_logits = model(
        images=images,
        instructions=instructions,
    )  # [B, T, A, n_bins]

    # 행동 예측 손실 (분류 문제로 처리)
    loss = cross_entropy(predicted_logits, action_tokens)

    return loss

# 학습 루프
for epoch in range(num_epochs):
    for batch in dataloader:
        optimizer.zero_grad()
        loss = training_step(batch)
        loss.backward()
        optimizer.step()
```

## 평가 루프

파인튜닝 중 정기적으로 시뮬레이터 또는 실제 로봇에서 평가를 수행합니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│                  평가 루프 구조                                   │
│                                                                  │
│  체크포인트 저장                                                  │
│       │                                                          │
│       ▼                                                          │
│  시뮬레이터 배포 (N번 롤아웃)                                    │
│       │                                                          │
│       ▼                                                          │
│  성공률 계산 (success rate)                                      │
│       │                                                          │
│  ┌────┴─────────────────────────┐                               │
│  │ 성공률 향상?                  │                               │
│  ├─ Yes: 최선 체크포인트 갱신   │                               │
│  └─ No:  계속 학습              │                               │
│                                 │                               │
│  최선 체크포인트 → 실물 로봇 검증│                               │
└──────────────────────────────────────────────────────────────────┘
```

## 데이터 효율성: 얼마나 필요한가

VLA 파인튜닝에 필요한 데이터 양은 과제 복잡도와 사전학습 모델 품질에 따라 달라집니다.

- **단순 집기 과제**: 50~100건 시연으로도 작동하는 경우가 보고됩니다.
- **복잡한 장기 과제**: 수백 건 이상 필요할 수 있습니다.
- **새로운 로봇 형태(새 embodiment)**: 관절 구조가 크게 다르면 더 많은 데이터가 필요합니다.

단, "N건으로 달성한다"는 주장은 환경·과제·평가 조건에 따라 크게 달라지므로, 특정 수치를 일반화하여 해석하지 않도록 주의해야 합니다.

다음 챕터에서는 학습된 정책을 평가하는 벤치마크와 방법론을 살펴봅니다.
