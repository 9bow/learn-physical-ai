## Omniverse란 무엇인가

NVIDIA Omniverse는 Pixar가 개발한 OpenUSD(Universal Scene Description)를 공통 언어로 삼아, 3D 씬(scene) 데이터를 실시간으로 공유하고 시뮬레이션할 수 있도록 설계된 개방형 플랫폼입니다. 단순한 시각화 도구가 아니라, 로봇 공학·자율주행·산업 시뮬레이션·물리 AI(Physical AI) 연구를 위한 **디지털 트윈(digital twin)** 인프라 전체를 아우릅니다.

Omniverse의 핵심 설계 철학은 **"실세계와 동일한 물리 법칙이 적용되는 가상 세계"** 를 만드는 것입니다. 단순히 그래픽이 예쁜 에뮬레이터가 아니라, 빛의 반사·굴절·그림자·재질의 물리적 특성까지 경로 추적(path tracing) 방식으로 정확하게 재현합니다. 이 물리적 정확도가 시뮬레이션-to-실세계(sim-to-real) 전환의 성패를 좌우하는 핵심 요소입니다.

## 핵심 구성 요소

### Nucleus — 협업의 중심 서버

Nucleus는 Omniverse의 **씬 데이터 협업 서버**입니다. 여러 클라이언트(사용자, 애플리케이션, 시뮬레이터)가 동일한 USD 씬 데이터에 동시에 접근하고 변경 사항을 실시간으로 주고받을 수 있도록 합니다.

Nucleus가 해결하는 문제는 명확합니다. 기존 3D 작업 환경에서 여러 팀이 동일한 씬을 다룰 때는 파일을 복사하고 병합하는 수작업이 필요했습니다. Nucleus는 **변경 델타(delta)만 전송하는 라이브 동기화** 방식으로 이 문제를 해결합니다. 로봇 개발 맥락에서는 설계 팀이 로봇 모델을 수정하면 시뮬레이션 팀이 즉시 업데이트된 씬을 받아볼 수 있습니다.

Nucleus는 로컬 머신에서도 실행 가능하고, 팀 단위 협업을 위해 서버에 배포할 수도 있습니다. 데이터 전송에는 **Omniverse 전용 프로토콜(omniverse://)**을 사용하며, 씬 파일·머티리얼·텍스처 등을 체계적으로 관리합니다.

### Omniverse Kit — 확장 가능한 애플리케이션 프레임워크

Kit은 Omniverse 기반 애플리케이션을 만들기 위한 **개발 프레임워크**입니다. Isaac Sim, Drive Sim 같은 NVIDIA의 시뮬레이터들은 모두 Kit 위에 구축된 Extension(확장) 모음입니다.

Kit의 핵심 설계 원칙은 **Extension 기반 모듈화**입니다. 모든 기능은 독립적인 Extension으로 구현되며, 애플리케이션은 필요한 Extension들을 조합해서 만들어집니다. Python으로 커스텀 Extension을 작성해 기능을 추가하거나 기존 UI를 수정할 수 있습니다.

```text
Omniverse Kit Extension 구조

┌─────────────────────────────────────────────────────┐
│                   Isaac Sim App                     │
├────────────────┬────────────────┬───────────────────┤
│  Physics Ext   │  Sensors Ext   │  ROS2 Bridge Ext  │
├────────────────┴────────────────┴───────────────────┤
│            Omniverse Kit 코어 런타임                 │
│   (렌더링 루프 / 이벤트 시스템 / Python 바인딩)      │
└─────────────────────────────────────────────────────┘
```

### RTX 렌더러 — 물리 기반 시각화

Omniverse의 RTX 렌더러는 두 가지 모드를 제공합니다.

**RTX 실시간 모드**: 레이트레이싱을 GPU에서 가속해 고품질 영상을 실시간으로 생성합니다. 물리 시뮬레이션과 연동해 빠른 속도로 렌더링이 필요할 때 사용합니다.

**RTX 경로 추적(Path Tracing) 모드**: 빛이 씬 내에서 반사·굴절·산란하는 과정을 물리적으로 정확하게 시뮬레이션합니다. 합성 데이터(synthetic data) 생성 시 실제 카메라 이미지와 최대한 유사한 결과물을 만들기 위해 사용합니다. 렌더링 속도는 느리지만, 생성된 이미지의 사실성(photorealism)이 높아 도메인 갭(domain gap)을 줄이는 데 효과적입니다.

### Connector 생태계 — 기존 도구와의 연결

Omniverse Connector는 Blender, Maya, 3ds Max, Revit, AutoCAD 같은 기존 DCC(Digital Content Creation) 툴이나 CAD 소프트웨어에서 USD 데이터를 바로 내보내고 가져올 수 있도록 하는 플러그인 모음입니다.

로봇 공학 맥락에서 중요한 Connector로는 **URDF Importer**가 있습니다. ROS 생태계에서 표준으로 사용하는 URDF(Unified Robot Description Format) 파일을 Isaac Sim의 USD 기반 씬으로 변환해줍니다. 이 외에도 MJCF(MuJoCo 포맷) Importer가 제공됩니다.

## Isaac Sim과 Omniverse의 관계

Isaac Sim은 독립된 소프트웨어가 아니라 **Omniverse Kit 위에 구축된 Extension 모음**입니다. 이 구조적 특성은 몇 가지 중요한 함의를 갖습니다.

첫째, Isaac Sim은 Omniverse의 모든 인프라—RTX 렌더러, Nucleus 협업, USD 파이프라인—를 그대로 활용합니다. 별도의 렌더링 엔진이나 파일 시스템을 구현하지 않고 플랫폼의 능력을 이어받습니다.

둘째, Python Extension API를 통해 Isaac Sim의 모든 기능을 프로그래밍 방식으로 제어할 수 있습니다. 강화학습 훈련 루프, 합성 데이터 생성 파이프라인, 센서 데이터 수집 등을 스크립트로 자동화할 수 있습니다.

셋째, Omniverse의 확장성 덕분에 Isaac Sim 자체에도 커스텀 Extension을 추가할 수 있습니다. 특정 도메인에 특화된 센서 모델, 로봇 컨트롤러, 작업 환경을 Extension으로 구현해 Isaac Sim에 통합하는 방식으로 연구 및 제품 개발에 활용됩니다.

```text
Isaac Sim 구성 요소 계층

┌──────────────────────────────────────────────────────┐
│                   Isaac Sim                          │
│                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐  │
│  │  Physics     │  │   Sensors    │  │   Tasks   │  │
│  │  (PhysX)     │  │  (카메라/라이다)│  │  (RL 환경)│  │
│  └──────────────┘  └──────────────┘  └───────────┘  │
│                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐  │
│  │  ROS2 Bridge │  │  Replicator  │  │  Isaac Lab│  │
│  │  (통신 브릿지)│  │  (합성 데이터)│  │  (RL 프레임)│ │
│  └──────────────┘  └──────────────┘  └───────────┘  │
│                                                      │
├──────────────────────────────────────────────────────┤
│              Omniverse Kit 런타임                     │
├──────────────────────────────────────────────────────┤
│         RTX 렌더러 │ Nucleus 클라이언트 │ OpenUSD    │

```

## Omniverse가 Physical AI에 중요한 이유

Physical AI 연구에서 Omniverse의 가치는 세 가지 축으로 정리됩니다.

**물리적 정확도**: PhysX 물리 엔진과 RTX 렌더러의 조합은 중력·마찰·충돌·관성은 물론 빛의 물리적 거동까지 정확하게 재현합니다. 실제 로봇이 경험하는 환경과 최대한 유사한 시뮬레이션을 제공하는 것이 목표입니다.

**대규모 병렬 훈련**: 단일 GPU 서버에서 수천 개의 로봇을 동시에 시뮬레이션하고 강화학습을 진행할 수 있습니다. Isaac Lab은 이 병렬 훈련 능력을 활용하기 위해 특별히 설계된 프레임워크입니다.

**합성 데이터 생성**: Omniverse Replicator를 사용해 인식 모델 훈련에 필요한 대규모 레이블 데이터를 자동 생성할 수 있습니다. 실제 환경에서 수집하기 어렵거나 위험한 엣지 케이스도 시뮬레이션에서는 쉽게 만들어낼 수 있습니다.

이 세 능력을 하나의 통합 플랫폼에서 제공한다는 점이 Omniverse를 Physical AI 개발의 핵심 인프라로 만드는 이유입니다.
