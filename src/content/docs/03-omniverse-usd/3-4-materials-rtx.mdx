## 왜 머티리얼이 Physical AI에서 중요한가

로봇이 물체를 인식하고 조작하는 능력은 시각 정보에 크게 의존합니다. 물체의 색상, 광택, 반사율, 질감은 카메라 이미지에서 물체를 구분하고 자세(pose)를 추정하는 데 핵심적인 단서입니다.

시뮬레이션에서 학습한 모델이 실제 환경에서 제대로 작동하려면 두 가지가 필요합니다. 첫째, 시뮬레이션의 물체가 **실제 물체와 시각적으로 충분히 유사**해야 합니다(사실성). 둘째, 훈련 과정에서 다양한 조명 조건·재질 변형을 경험해야 실제 환경의 변동성에 견고한 모델이 만들어집니다(일반화).

MDL 머티리얼과 RTX 렌더링은 이 두 목표를 동시에 달성하기 위한 기술 기반입니다.

## MDL — Material Definition Language

MDL(Material Definition Language)은 NVIDIA가 개발한 **물리 기반 머티리얼 정의 언어**입니다. MDL은 머티리얼이 빛과 어떻게 상호작용하는지를 수학적으로 정확하게 기술합니다. Omniverse의 모든 머티리얼은 MDL로 정의되며, USD 씬에서 `UsdShadeMaterial`과 `UsdShadeShader` Prim을 통해 메시에 바인딩됩니다.

MDL의 핵심 특성은 **렌더러 독립성**입니다. MDL로 정의된 머티리얼은 RTX 경로 추적, 실시간 래스터라이저, 심지어 외부 렌더러에서도 물리적으로 일관된 결과를 냅니다. 이 특성이 시뮬레이션-to-실세계 갭을 줄이는 데 중요합니다.

### PBR — 물리 기반 렌더링 파라미터

MDL의 대표적인 머티리얼 모델은 **PBR(Physically Based Rendering)** 입니다. PBR은 실세계 재료의 광학적 특성을 몇 가지 직관적인 파라미터로 추상화합니다.

```text
PBR 머티리얼 핵심 파라미터

┌──────────────────────────────────────────────────────────┐
│                    PBR 머티리얼                           │
├─────────────────────┬────────────────────────────────────┤
│ Albedo (기반 색상)  │ 빛이 없는 순수 색상              │
│                     │ RGB 값 또는 텍스처 맵              │
├─────────────────────┼────────────────────────────────────┤
│ Metallic (금속성)   │ 0.0 = 비금속 (절연체)            │
│                     │ 1.0 = 금속                        │
│                     │ 금속/비금속의 반사 특성 결정       │
├─────────────────────┼────────────────────────────────────┤
│ Roughness (거칠기)  │ 0.0 = 완전히 매끄러운 경면 반사   │
│                     │ 1.0 = 완전히 거친 확산 반사        │
│                     │ 표면의 미세 구조를 모델링           │
├─────────────────────┼────────────────────────────────────┤
│ Normal Map          │ 표면 법선 방향을 픽셀 단위로 변형  │
│ (법선 맵)           │ 실제 지오메트리 없이 디테일 표현   │
├─────────────────────┼────────────────────────────────────┤
│ Emissive (발광)     │ 스스로 빛을 내는 표면              │
│                     │ LED 패널, 화면, 발광 소재          │
└─────────────────────┴────────────────────────────────────┘
```

도메인 랜덤화 관점에서 이 파라미터들은 **연속적인 값 범위**를 가지므로, 균등 분포나 정규 분포에서 샘플링해 다양한 재질 변형을 자동으로 생성할 수 있습니다.

### OmniPBR — Omniverse 기본 머티리얼

Omniverse는 `OmniPBR`이라는 내장 MDL 머티리얼을 제공합니다. 이는 PBR 파라미터를 직관적인 UI로 조작할 수 있게 해주는 표준 셰이더입니다. Isaac Sim에서 물체에 머티리얼을 적용하고 도메인 랜덤화를 설정할 때 가장 자주 사용하는 시작점입니다.

```python
# 개념 이해용 의사코드 — MDL 머티리얼 파라미터 접근 예시
# 실제 API는 Isaac Sim/Omniverse 버전에 따라 다를 수 있으므로 공식 문서 기준 확인 필요

material_prim = stage.GetPrimAtPath("/World/Objects/Box/Looks/Material")
shader = UsdShade.Shader(material_prim.GetChild("Shader"))

# OmniPBR 파라미터 조작 예시 (의사코드)
shader.GetInput("albedo_brightness").Set(0.8)    # 밝기
shader.GetInput("reflection_roughness_constant").Set(0.3)  # 거칠기
shader.GetInput("metallic_constant").Set(0.0)   # 금속성
```

## RTX 렌더링 모드 비교

Omniverse는 두 가지 RTX 렌더링 모드를 제공하며, 각각 적합한 용도가 다릅니다.

```text
RTX 렌더링 모드 비교

┌─────────────────┬────────────────────┬────────────────────┐
│    특성         │   RTX 실시간 모드  │  RTX 경로 추적 모드 │
├─────────────────┼────────────────────┼────────────────────┤
│ 렌더링 속도     │ 높음 (실시간 가능) │ 낮음 (프레임당 수초)│
├─────────────────┼────────────────────┼────────────────────┤
│ 빛 정확도       │ 근사 (하이브리드)  │ 물리적으로 정확     │
├─────────────────┼────────────────────┼────────────────────┤
│ 반사/굴절       │ 제한적 근사        │ 다중 반사 정확 처리 │
├─────────────────┼────────────────────┼────────────────────┤
│ 그림자          │ 근사 (레이 트레이싱)│ 정확한 소프트 섀도우│
├─────────────────┼────────────────────┼────────────────────┤
│ 간접 조명       │ 제한적             │ 완전한 전역 조명    │
├─────────────────┼────────────────────┼────────────────────┤
│ 주요 용도       │ RL 훈련 루프       │ 합성 데이터 생성    │
│                 │ 실시간 시각화      │ 고품질 렌더링       │
└─────────────────┴────────────────────┴────────────────────┘
```

### RTX 실시간 모드 — 훈련 속도 우선

강화학습 훈련 루프에서는 초당 수십~수백 스텝의 씬 업데이트가 필요합니다. 이 경우 렌더링 품질보다 속도가 우선이므로 RTX 실시간 모드를 사용합니다. GPU 레이트레이싱 가속을 활용해 실시간에 가까운 속도로 고품질 이미지를 생성합니다.

### RTX 경로 추적 모드 — 데이터 품질 우선

합성 데이터 생성에서는 생성된 이미지가 실제 카메라 이미지와 최대한 유사해야 합니다. RTX 경로 추적 모드는 빛의 물리적 경로를 시뮬레이션해 실제 사진과 구분하기 어려운 이미지를 생성합니다.

경로 추적의 핵심 효과들:

```text
경로 추적이 재현하는 물리 효과

광원
 │
 ├──▶ 직접 조명 (Direct Illumination)
 │    물체 표면에 직접 닿는 빛
 │
 ├──▶ 간접 조명 (Indirect/Global Illumination)
 │    다른 표면에서 반사된 빛 (예: 파란 벽 옆 물체가 파랗게 물드는 현상)
 │
 ├──▶ 코스틱 (Caustics)
 │    투명체/반사면을 통과한 빛이 만드는 집중 패턴
 │    (물컵 아래 밝은 패턴)
 │
 └──▶ 서브서피스 스캐터링 (SSS)
      빛이 반투명 재질 내부를 산란하는 효과
      (피부, 과일, 얇은 플라스틱)
```

## 도메인 랜덤화를 위한 머티리얼 전략

도메인 랜덤화(domain randomization)에서 머티리얼 변형은 가장 영향력 있는 요소 중 하나입니다. 동일한 형태의 물체라도 재질이 달라지면 카메라 이미지에서 완전히 다르게 보입니다.

### 머티리얼 랜덤화 파라미터 공간

```text
도메인 랜덤화 머티리얼 파라미터 예시

Albedo 색상: 균등 분포 샘플링 (전체 색상 공간)
    또는 실제 물체 색상 분포에서 샘플링

Roughness: 균등 분포 [0.1, 0.9]
    ├── 0.1-0.3: 광택 있는 표면 (플라스틱, 유리)
    ├── 0.3-0.6: 반광택 (페인트, 코팅)
    └── 0.6-0.9: 무광 (고무, 직물)

Metallic: 이진 분포 또는 균등 [0.0, 1.0]
    ├── ~0.0: 비금속 (플라스틱, 도자기, 나무)
    └── ~1.0: 금속 (철, 알루미늄, 구리)

Normal Map 강도: 균등 분포 [0.5, 2.0]
    (표면 디테일의 강조 정도 변형)
```

### VariantSet과 머티리얼의 결합

앞서 배운 USD VariantSet을 활용해 미리 정의된 재질 세트를 효율적으로 전환할 수 있습니다. 예를 들어 "금속 컵", "플라스틱 컵", "세라믹 컵" 변형을 VariantSet으로 정의하면 프로그래밍 방식으로 랜덤하게 선택해 데이터를 생성할 수 있습니다.

```python
# 개념 이해용 의사코드 — VariantSet으로 머티리얼 전환 예시

def randomize_material(stage, prim_path, variant_set_name):
    prim = stage.GetPrimAtPath(prim_path)
    variant_set = prim.GetVariantSets().GetVariantSet(variant_set_name)
    
    available_variants = variant_set.GetVariantNames()
    selected = random.choice(available_variants)
    variant_set.SetVariantSelection(selected)
```

### 합성 데이터 품질과 도메인 갭

합성 데이터로 학습한 모델이 실제 환경에서 잘 작동하지 않는 현상을 **도메인 갭(domain gap)**이라고 합니다. 머티리얼 관련 도메인 갭의 원인과 대응 전략을 정리하면 다음과 같습니다.

| 도메인 갭 원인 | 대응 전략 |
|----------------|----------|
| 머티리얼 파라미터가 실제와 다름 | 실제 물체를 3D 스캔해 재질 측정 후 MDL로 재현 |
| 조명 조건이 너무 단순 | 다양한 HDR 환경 맵과 조명 위치 랜덤화 |
| 그림자·반사가 비현실적 | RTX 경로 추적 모드로 전환 |
| 배경·환경이 고정됨 | 배경 텍스처와 환경 조명도 랜덤화 |

RTX 경로 추적 모드는 물리적으로 정확한 반사와 간접 조명을 제공해 도메인 갭의 주요 원인 중 하나인 렌더링 비현실성을 크게 줄여줍니다. 단, 렌더링 속도와의 트레이드오프가 있으므로 합성 데이터 생성은 오프라인으로 진행하고 훈련은 별도로 수행하는 방식이 일반적입니다.
