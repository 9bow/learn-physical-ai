## USD란 무엇인가

USD(Universal Scene Description)는 Pixar Animation Studios가 개발하고 2016년에 오픈소스로 공개한 3D 씬 기술 형식입니다. 픽사는 대규모 영화 제작 과정에서 수십 명의 아티스트가 동일한 씬 데이터를 비파괴적으로 편집할 수 있는 시스템이 필요했고, 그 결과물이 USD입니다. "Coco(2017)", "Soul(2020)" 같은 작품도 이 파이프라인을 활용했습니다.

USD가 픽사의 영화 파이프라인을 넘어 로봇 공학과 Physical AI 분야의 표준으로 자리 잡은 이유는 세 가지입니다.

첫째, **레이어 기반 비파괴 편집**입니다. 원본 데이터를 건드리지 않고 위에 오버라이드 레이어를 쌓아 변경 사항을 적용할 수 있습니다. 로봇 씬의 경우 베이스 환경은 그대로 두고 시험별로 다른 머티리얼·조명·물체 배치를 별도 레이어로 관리할 수 있습니다.

둘째, **참조(reference)와 인스턴싱(instancing)**을 통해 대규모 씬을 효율적으로 구성할 수 있습니다. 동일한 로봇 모델을 수천 개 배치할 때도 원본 데이터는 하나만 유지하면서 각 인스턴스의 위치·자세만 다르게 지정할 수 있습니다.

셋째, **확장 가능한 스키마(Schema)** 시스템입니다. USD는 물리·렌더링·애니메이션·로봇 관절 같은 도메인별 데이터를 타입화된 스키마로 표현합니다. NVIDIA는 PhysicsSchema, IsaacSensorSchema 등을 추가해 로봇 시뮬레이션에 필요한 정보를 USD 안에서 표현합니다.

## USD 핵심 데이터 모델

### Stage — 씬의 최상위 컨테이너

Stage는 USD의 **씬 전체를 담는 최상위 컨테이너**입니다. 여러 레이어(layer)가 조합되어 하나의 Stage를 구성합니다. 프로그래밍 관점에서 Stage는 씬에 접근하는 진입점(entry point)입니다.

```text
USD Stage 구조

Stage
├── Root Layer (루트 레이어 - 주 .usd 파일)
│   ├── SubLayer A (보조 레이어)
│   │   └── /World/Robot (프림)
│   └── SubLayer B (오버라이드 레이어)
│       └── /World/Robot.material (머티리얼 오버라이드)
│
└── Composed Scene (최종 합성 결과)
    └── /World/Robot (A + B 합성된 최종 상태)
```

### Prim — 씬 객체의 기본 단위

Prim(Primitive)은 USD 씬의 **기본 구성 단위**입니다. 3D 메시, 카메라, 라이트, 로봇 관절, 스코프(Scope, 논리적 그룹) 등 씬의 모든 요소가 Prim으로 표현됩니다.

모든 Prim은 씬 계층에서 경로(path)로 식별됩니다. 예를 들어 `/World/Robot/LeftArm/Gripper`처럼 파일 시스템과 유사한 계층 구조를 가집니다.

Prim의 중요한 속성으로 **타입(type)**이 있습니다. `Xform`(변환), `Mesh`(메시), `Scope`(논리 그룹), `Camera`(카메라) 등의 내장 타입과, 스키마를 통해 정의된 `PhysicsRigidBodyAPI`, `PhysicsCollisionAPI` 같은 API 스키마가 있습니다.

```text
Prim 계층 예시

/World                       (Xform - 씬 루트)
├── /World/Environment        (Scope - 환경 그룹)
│   ├── /World/Environment/Ground   (Mesh - 바닥)
│   └── /World/Environment/Light    (DistantLight)
│
└── /World/Robot              (Xform - 로봇 루트)
    ├── /World/Robot/Base     (Mesh + PhysicsRigidBodyAPI)
    ├── /World/Robot/Joint0   (PhysicsRevoluteJoint)
    └── /World/Robot/Link1    (Mesh + PhysicsRigidBodyAPI)
```

### Property — Attribute와 Relationship

각 Prim은 **Property**를 가집니다. Property는 두 종류입니다.

**Attribute(속성)**: 값을 가지는 데이터입니다. 위치(`xformOp:translate`), 색상(`primvars:displayColor`), 질량(`physics:mass`) 같은 것들이 Attribute입니다. Attribute는 시간에 따라 값이 변하는 **시계열(time-sampled)** 데이터를 지원합니다. 로봇 관절 각도의 시간에 따른 변화를 USD Attribute로 표현할 수 있습니다.

**Relationship(관계)**: 한 Prim이 다른 Prim을 참조하는 관계입니다. 머티리얼을 메시에 바인딩하거나, 관절이 연결하는 두 링크를 지정할 때 Relationship을 사용합니다.

### Schema — 타입화된 데이터 계약

Schema는 **특정 도메인의 Attribute와 Relationship 집합을 정의한 계약**입니다. Schema를 통해 USD는 범용 컨테이너이면서도 도메인 특화 데이터를 체계적으로 표현할 수 있습니다.

Schema는 크게 두 종류입니다.

- **Typed Schema**: Prim의 타입 자체를 결정합니다. `UsdGeomMesh`, `UsdLuxDistantLight` 등이 여기에 해당합니다.
- **API Schema**: 기존 Prim에 추가적인 기능을 붙이는 방식입니다. `UsdPhysicsRigidBodyAPI`, `UsdPhysicsCollisionAPI`처럼 API를 적용해 물리 시뮬레이션에 필요한 속성을 추가합니다.

```text
Schema 적용 예시

/World/Robot/Link1 (Prim)
│
├── 타입: UsdGeomMesh         ← Typed Schema (메시 형상)
│   ├── points: [....]        ← 정점 좌표
│   └── faceVertexIndices: [.] ← 폴리곤 인덱스
│
├── API: UsdPhysicsRigidBodyAPI ← API Schema (물리 특성)
│   ├── physics:mass: 1.5 kg
│   └── physics:velocity: [0,0,0]
│
└── API: UsdPhysicsCollisionAPI ← API Schema (충돌 검출)
    └── physics:approximation: "convexHull"
```

## USD 파일 형식

USD는 네 가지 파일 형식을 지원합니다(.usdz는 패키지 형식으로 별도 분류).

| 형식 | 확장자 | 특징 | 권장 용도 |
|------|--------|------|----------|
| USD ASCII | `.usda` | 사람이 읽을 수 있는 텍스트 | 디버깅·학습·버전 관리 |
| USD Crate(Binary) | `.usdc` | 이진 압축 형식, 빠른 로딩 | 프로덕션 씬, 대용량 데이터 |
| USD (복합) | `.usd` | 런타임이 자동으로 형식 선택 | 일반 사용 |
| USDZ | `.usdz` | Zip 패키징, AR/배포용 | 모바일 AR, 씬 배포 |

로봇 개발 워크플로에서는 보통 디버깅과 검토 단계에서 `.usda`를, 실제 시뮬레이션 씬은 성능을 위해 `.usdc`로 저장합니다. Isaac Sim은 두 형식을 모두 지원합니다.

간단한 `.usda` 예시를 보면 USD의 텍스트 구조를 이해할 수 있습니다:

```text
# 실제 .usda 파일 구조 예시
#usda 1.0
(
    upAxis = "Y"
    metersPerUnit = 1.0
)

def Xform "World"
{
    def Mesh "Ground"
    {
        float3[] extent = [(-5, 0, -5), (5, 0, 5)]
        point3f[] points = [(-5, 0, -5), (5, 0, -5), (5, 0, 5), (-5, 0, 5)]
        int[] faceVertexCounts = [4]
        int[] faceVertexIndices = [0, 1, 2, 3]
    }
}
```

## Physical AI에서 USD가 중요한 이유

### 도구 간 상호운용성

Physical AI 개발 파이프라인은 CAD 소프트웨어, DCC 툴, 시뮬레이터, 학습 프레임워크 등 다양한 도구를 사용합니다. USD는 이 모든 도구가 **단일 씬 표현을 공유**할 수 있는 공통 언어 역할을 합니다.

```text
Physical AI 데이터 흐름에서의 USD

CAD/설계 툴          DCC 툴               시뮬레이터
(SolidWorks, etc)  (Blender, Maya)     (Isaac Sim)
       │                  │                  │
       ▼                  ▼                  ▼
  URDF/CAD → USD    OBJ/FBX → USD     USD ← → USD
                          │
                     USD 공통 포맷
                          │
              ┌───────────┼───────────┐
              ▼           ▼           ▼
         학습 데이터    디지털 트윈    배포 패키지
         (어노테이션)   (실시간 감시)  (로봇 URDF)
```

### 비파괴 실험 관리

로봇 훈련 실험에서는 같은 환경에 다양한 조건 변형이 필요합니다. USD의 레이어 시스템을 활용하면 기본 씬은 유지하면서 머티리얼·조명·물체 위치 같은 변수만 레이어로 오버라이드하는 방식으로 수백 가지 변형 씬을 관리할 수 있습니다. 이는 도메인 랜덤화(domain randomization) 파이프라인의 기반이 됩니다.

### 대규모 씬의 효율적 관리

로봇 공장 같은 복잡한 환경은 수만 개의 객체로 구성됩니다. USD의 참조와 인스턴싱 메커니즘을 통해 동일한 부품은 메모리에 한 번만 로드하면서 씬 전체에 수천 개를 배치할 수 있습니다. 이는 대규모 합성 데이터 생성 및 디지털 트윈 구현에 직접적으로 기여합니다.

## USD를 배우는 실전 접근법

USD는 개념이 많고 처음에는 복잡하게 느껴집니다. 다음 순서로 접근하는 것이 효과적입니다.

1. **usda 파일 직접 읽기**: Isaac Sim에서 간단한 씬을 만들고 `.usda`로 저장해 내용을 확인합니다.
2. **Python USD API 실습**: `pxr.Usd` 모듈(OpenUSD Python 바인딩)로 Stage를 열고 Prim을 순회하는 간단한 스크립트를 작성합니다.
3. **Isaac Sim Stage 트리 탐색**: Isaac Sim UI의 Stage 패널에서 로봇 씬의 Prim 계층을 직접 탐색합니다.

이후 섹션에서는 USD의 레이어 합성 시스템(Composition Arcs)과 Isaac Sim에서의 실전 활용을 더 깊이 다룹니다.
