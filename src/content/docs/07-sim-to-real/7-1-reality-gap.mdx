## Reality Gap이란 무엇인가

로보틱스 강화학습(RL) 연구에서 "Reality Gap(현실 갭)"은 시뮬레이션에서 학습한 정책(policy)이 실제 로봇에서는 성능이 크게 저하되거나 완전히 작동하지 않는 현상을 가리킵니다. 이 문제는 시뮬레이터가 아무리 정교해도 완전히 사라지지 않습니다. 현실의 물리 법칙은 수식으로 근사할 수 있을 뿐, 완벽히 재현할 수 없기 때문입니다.

Reality Gap을 단순히 "시뮬레이터가 부정확해서" 라고 이해하는 것은 틀리지 않지만 충분하지 않습니다. 실제로는 여러 독립적인 원인이 복합적으로 작용하며, 각 원인의 성격과 대처 방법도 다릅니다. 이 챕터에서는 그 원인들을 하나씩 분해합니다.

## 원인 1: 물리 모델 부정확성

시뮬레이터는 현실 세계의 물리 법칙을 **수치적으로 근사**합니다. 가장 핵심적인 근사 오차는 **접촉 역학(contact dynamics)** 과 **마찰(friction)** 에서 발생합니다.

### 접촉 역학

로봇이 물체를 집거나 발이 지면에 닿는 순간, 현실에서는 수 마이크로초 단위의 복잡한 탄성 변형, 진동, 에너지 분산이 일어납니다. 시뮬레이터는 이를 단순한 충돌 모델(rigid body contact, spring-damper 등)로 처리합니다. Isaac Sim, MuJoCo, PyBullet 등 각 시뮬레이터는 서로 다른 접촉 모델을 사용하므로, 같은 정책도 시뮬레이터에 따라 다른 결과를 냅니다.

### 마찰

마찰 계수(friction coefficient)는 측정 방법과 환경에 따라 크게 달라집니다. 바닥재 종류, 습도, 로봇 발바닥의 마모 정도에 따라 실제 마찰력은 시뮬레이션 값과 수십 퍼센트 차이가 날 수 있습니다. 보행 로봇에서는 이 오차가 미끄러짐이나 넘어짐으로 직결됩니다.

```text
┌──────────────────────────────────────────────────────────┐
│              물리 모델 부정확성의 주요 원천               │
├──────────────────────┬───────────────────────────────────┤
│ 원인                 │ 영향                              │
├──────────────────────┼───────────────────────────────────┤
│ 접촉 모델 단순화     │ 그리핑/보행 시 예상 밖 거동      │
│ 마찰 계수 오차       │ 미끄러짐, 안정성 저하             │
│ 강체 가정            │ 유연 부재(soft body) 무시         │
│ 공기 저항 무시       │ 고속 운동 시 궤적 오차            │
│ 관절 순응도 무시     │ 힘 제어 정확도 저하               │
└──────────────────────┴───────────────────────────────────┘
```

## 원인 2: 센서 노이즈

실제 센서는 이상적인 수치를 출력하지 않습니다. 카메라에는 렌즈 왜곡, 노출 변화, 모션 블러가 있습니다. IMU(관성 측정 장치)는 온도에 따른 드리프트(drift)와 고주파 노이즈를 포함합니다. 힘/토크 센서는 진동에 민감하고 제로 오프셋이 시간에 따라 변합니다.

시뮬레이터는 이런 노이즈를 가우시안(Gaussian) 분포나 단순한 확률 모델로 근사합니다. 하지만 실제 노이즈는 비선형적이고, 주변 온도·전자기 간섭·물리적 충격 등 환경 변수에 의존적입니다.

특히 **시각 기반 정책(vision-based policy)** 에서는 센서 노이즈가 중요합니다. 시뮬레이션의 렌더링은 물리 엔진과 별개의 문제이고, 실제 카메라 이미지와는 텍스처·조명·반사 특성이 다릅니다. 이 시각적 차이는 "시각적 도메인 갭(visual domain gap)"이라고 별도로 부르기도 합니다.

## 원인 3: 액추에이터 동역학

액추에이터(actuator)는 로봇의 관절을 움직이는 장치입니다. 시뮬레이터에서는 관절 위치나 토크 명령이 즉각적이고 완벽하게 실행된다고 가정하는 경우가 많습니다. 그러나 실제 액추에이터는 다음과 같은 특성을 가집니다.

- **전류 제한**: 과전류 방지를 위한 물리적 상한이 있습니다.
- **백래시(backlash)**: 기어 조립 공차로 인해 명령 방향 전환 시 유격이 발생합니다.
- **모터 열**: 지속 작동 시 코일 저항이 증가하여 토크 출력이 감소합니다.
- **비선형 마찰**: 정적 마찰(static friction)과 동적 마찰(kinetic friction)의 차이, 스틱-슬립(stick-slip) 현상.

이런 특성들은 단순한 PID 모델로는 포착되지 않습니다. 이를 더 정확히 모델링하기 위해 **액추에이터 네트워크(actuator network)** 를 별도로 학습하는 방법이 등장했으며, 7-3 챕터에서 자세히 다룹니다.

## 원인 4: 지연(Latency)

```text
실제 로봇 제어 루프의 지연 발생 지점:

센서 측정
    │
    ▼
[센서 읽기 지연 ~1-5ms]
    │
    ▼
신호 전송 (USB/EtherCAT/CAN)
    │
    ▼
[통신 지연 ~0.1-2ms]
    │
    ▼
정책 추론 (CPU/GPU)
    │
    ▼
[추론 지연 ~5-50ms]
    │
    ▼
액추에이터 명령 전송
    │
    ▼
[명령 실행 지연 ~1-10ms]
    │
    ▼
실제 관절 이동
```

시뮬레이터에서는 이 전체 루프가 사실상 즉각적으로 일어납니다. 그러나 실제 시스템에서 총 지연은 수십 밀리초에 달할 수 있습니다. 고속 보행 로봇에서 20ms 지연은 한 보행 사이클의 상당 부분에 해당합니다.

지연 불일치는 특히 **고게인(high-gain) 정책** 에서 치명적입니다. 정책이 최신 상태가 아닌 오래된 관측을 바탕으로 명령을 내리면, 진동이나 발산(divergence)이 발생합니다. 이를 완화하기 위해 학습 시 의도적으로 지연을 추가하거나(action delay randomization), 상태에 과거 행동 히스토리를 포함시키는 방법을 사용합니다.

## 원인 5: 시각적 차이

물리 엔진의 정확도와 별개로, **렌더링 품질**도 Reality Gap의 독립적인 원인입니다. 실제 카메라로 찍은 이미지와 시뮬레이션 렌더링의 차이는 다음 요소에서 발생합니다.

- **조명**: 자연광의 스펙트럼, 방향, 강도 변화
- **텍스처**: 실제 재질의 미세 구조와 시뮬레이션 텍스처의 차이
- **반사와 그림자**: 광선 추적(ray tracing)도 현실을 완벽히 재현하지 않음
- **렌즈 특성**: 카메라 모델의 왜곡 파라미터 오차

Isaac Sim과 같은 고품질 렌더러를 사용하면 시각적 갭을 크게 줄일 수 있지만, 완전히 제거할 수는 없습니다.

## Reality Gap의 복합 효과

각 원인이 독립적이라면 개별적으로 해결하면 될 것입니다. 문제는 이 원인들이 **복합적으로 상호작용**한다는 점입니다. 예를 들어, 액추에이터 동역학의 오차가 관절 상태 추정 오차를 만들고, 이것이 다시 잘못된 힘 명령을 유발하며, 그 결과 접촉 역학에서 예상치 못한 거동이 나타납니다. 정책은 이 연쇄 오차가 누적된 상황에 맞닥뜨립니다.

이런 복합 효과 때문에, Reality Gap 해결은 단일 기술로 끝나지 않습니다. 도메인 랜덤화(domain randomization), 시스템 식별(system identification), 적응 제어(adaptive control) 등 여러 방법을 조합하는 것이 일반적입니다. 이어지는 챕터들에서 각 방법을 상세히 다룹니다.

## 정리: Reality Gap의 다섯 가지 원인

```text
┌─────────────────────────────────────────────────────────────┐
│                   Reality Gap 원인 지도                      │
├──────────────────────────┬──────────────────────────────────┤
│ 원인                     │ 주요 대응 방법 (미리 보기)       │
├──────────────────────────┼──────────────────────────────────┤
│ 물리 모델 부정확         │ System ID, Domain Randomization  │
│ 센서 노이즈              │ 노이즈 모델 추가, 감지 필터링    │
│ 액추에이터 동역학        │ Actuator Network 학습            │
│ 지연(Latency)            │ Action delay 랜덤화, 히스토리 관측│
│ 시각적 차이              │ Domain Randomization (visual),   │
│                          │ 고품질 렌더러                    │
└──────────────────────────┴──────────────────────────────────┘
```

Reality Gap은 시뮬레이션 로보틱스의 근본적 도전입니다. 그러나 이를 이해하고 체계적으로 대응하면, 수백만~수억 스텝의 시뮬레이션 학습으로 얻은 정책을 실제 로봇에 이전하는 것이 가능합니다. 이것이 현대 로보틱스 연구의 핵심 목표 중 하나입니다.
