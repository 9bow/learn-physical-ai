## Domain Randomization의 핵심 직관

Domain Randomization(도메인 랜덤화, DR)의 핵심 아이디어는 단순합니다. **현실 세계는 시뮬레이션의 파라미터 공간 어딘가에 존재한다**는 가정 아래, 그 파라미터를 다양하게 변화시켜 학습시키면 정책이 실제 환경도 하나의 "변형(variant)"으로 처리할 수 있다는 것입니다.

달리 말하면, 시뮬레이터를 정확하게 보정하는 대신 **의도적으로 부정확하게** 만들어 일반화 능력을 키우는 전략입니다. 이 접근은 모델 오차를 줄이는 것이 아니라, 정책이 오차에 **robust(강건)** 해지도록 훈련하는 것입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│               Domain Randomization 개념도                     │
│                                                              │
│   파라미터 공간                                              │
│   ┌────────────────────────────────────────────┐            │
│   │  θ_low ●────────────────────────● θ_high  │            │
│   │         │  랜덤화 범위           │          │            │
│   │         │   (학습 시 샘플링)     │          │            │
│   │                    ★ 현실                   │            │
│   │         (현실은 이 범위 어딘가)             │            │
│   └────────────────────────────────────────────┘            │
│                                                              │
│   정책은 모든 θ에서 작동하도록 학습 → 현실도 커버          │
└──────────────────────────────────────────────────────────────┘
```

## 어떤 파라미터를 랜덤화할 것인가

랜덤화 대상 파라미터 선정은 DR에서 가장 중요한 설계 결정입니다. 모든 파라미터를 랜덤화하면 너무 어려운 과제가 되고, 너무 적게 하면 현실과의 갭이 남습니다.

### Dynamics Randomization (물리 파라미터)

물리 파라미터 랜덤화는 시뮬레이터의 동역학 설정을 에피소드마다 바꾸는 방식입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│           Dynamics Randomization 파라미터 예시               │
├──────────────────────┬──────────────────────────────────────┤
│ 카테고리             │ 구체 파라미터                         │
├──────────────────────┼──────────────────────────────────────┤
│ 질량/관성           │ 링크 질량, 관성 텐서                  │
│ 마찰                │ 발-지면, 손-물체 마찰 계수            │
│ 관절 특성           │ 댐핑, 강성, 토크 제한                 │
│ 액추에이터          │ 게인(KP, KD), 백래시, 지연            │
│ 외란                │ 외력(push perturbation), 지면 기울기  │
│ 물체 속성           │ 조작 대상의 질량, 크기, 마찰          │
└──────────────────────┴──────────────────────────────────────┘
```

보행 로봇에서 가장 중요한 파라미터는 일반적으로 **링크 질량**, **관절 댐핑**, **마찰 계수**, 그리고 **액추에이터 지연**입니다. 이 네 가지만 잘 랜덤화해도 실제 전이에서 상당한 개선을 볼 수 있습니다.

### Visual Randomization (시각 파라미터)

시각 기반 정책(카메라 입력)에는 시각 도메인 갭이 추가됩니다. Visual DR은 렌더링 파라미터를 변화시킵니다.

- **조명**: 광원 위치, 강도, 색온도, 다수 광원
- **텍스처**: 배경, 물체 표면에 무작위 텍스처 적용
- **카메라**: 위치 노이즈, FOV(시야각), 렌즈 왜곡
- **색상**: 무작위 색조(hue), 채도(saturation) 변화
- **무작위 물체**: 배경에 관련 없는 물체 배치 (distractor)

OpenAI Dactyl 프로젝트가 Visual DR을 본격적으로 사용하여 인-핸드 조작(in-hand manipulation)의 시각 기반 정책을 현실에 이전한 사례가 있습니다. 이 프로젝트는 7-4 챕터에서 더 자세히 다룹니다.

## 랜덤화 범위 설정

랜덤화 범위(distribution bounds)를 어떻게 설정하느냐는 실험적 판단이 필요합니다. 일반적인 접근 방식은 다음과 같습니다.

**1단계: 현실 측정값 기준점 확보**  
시스템 식별(7-3 챕터)을 통해 실제 파라미터의 기준값을 측정합니다.

**2단계: 물리적으로 그럴듯한 범위 설정**  
기준값의 ±20~50% 범위를 초기값으로 사용합니다. 예를 들어, 실제 링크 질량이 1kg이면 [0.7, 1.3]kg 범위로 랜덤화합니다.

**3단계: 점진적 확대**  
좁은 범위에서 학습이 수렴되면 범위를 점차 넓힙니다. 이것이 Automatic DR(ADR)의 기본 아이디어입니다.

## 과도 랜덤화(Over-Randomization)의 트레이드오프

DR은 강력하지만 지나치면 역효과가 납니다. 이를 **과도 랜덤화(over-randomization)** 라 합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│         랜덤화 범위와 성능의 관계                            │
│                                                              │
│  실제 성능                                                   │
│    │                   ★ 최적 범위                          │
│    │             ╱─────────╲                                 │
│    │           ╱             ╲                               │
│    │         ╱                 ╲  과도 랜덤화:               │
│    │       ╱  부족한 랜덤화:     ╲  너무 어려운 과제,        │
│    │     ╱    현실 갭 잔존        ╲ 정책 수렴 실패           │
│    └────────────────────────────────────────────▶           │
│                            랜덤화 범위                       │
└──────────────────────────────────────────────────────────────┘
```

과도 랜덤화가 발생하면:
- 정책 학습이 수렴하지 않거나 성능이 낮아집니다.
- 물리적으로 불가능한 조합(예: 마찰 계수 0에 가까운 미끄러운 표면에서 정밀 조작)을 배우려 하면 정책이 현실적이지 않은 보상 전략을 학습합니다.
- 불필요하게 보수적인 행동(conservative policy)이 나타납니다.

이 트레이드오프를 수동으로 조율하는 것은 어렵습니다. 이를 자동화하려는 시도가 Automatic DR입니다.

## Automatic Domain Randomization (ADR)

ADR은 랜덤화 범위를 **정책의 현재 성능에 따라 자동으로 조정**하는 방법입니다. OpenAI의 Dactyl 팀이 손 안 조작(in-hand manipulation) 연구에서 제안했습니다.

기본 작동 원리:

```text
ADR 알고리즘 개요 (개념 이해용 의사코드):

초기화: 각 파라미터 θ_i에 초기 범위 [lo_i, hi_i] 설정

반복:
  1. 현재 범위에서 θ를 샘플링하여 에피소드 진행
  2. 최근 N 에피소드의 평균 성공률 P 계산
  3. P > threshold_high 이면:
       랜덤화 범위 확대 (lo_i 감소, hi_i 증가)
  4. P < threshold_low 이면:
       랜덤화 범위 축소
  5. threshold_low ≤ P ≤ threshold_high 이면:
       범위 유지
```

ADR의 장점:
- 정책이 감당할 수 있는 난이도에 맞게 범위가 자동으로 조정됩니다.
- 처음부터 넓은 범위를 설정해 학습이 실패하는 문제를 피할 수 있습니다.
- 사람의 개입 없이 점진적으로 더 도전적인 환경을 학습합니다.

ADR의 한계:
- 랜덤화 확대 속도, 임계값 설정 등 추가 하이퍼파라미터가 필요합니다.
- 파라미터 간 상호 의존성을 처리하기 어렵습니다.
- 수렴에 매우 많은 학습 시간이 필요합니다.

## Structured Randomization: 분포 선택

랜덤화 분포를 어떻게 선택하느냐도 중요합니다. 흔히 사용하는 분포:

| 분포 | 특징 | 적합한 경우 |
|------|------|-------------|
| 균등 분포(Uniform) | 범위 내 모든 값 동등 확률 | 측정 불확실성이 클 때 |
| 정규 분포(Normal) | 기준값 근처에 집중 | 기준값에 대한 신뢰가 있을 때 |
| 로그 균등(Log-uniform) | 스케일 변화가 큰 파라미터 | 질량, 강성 계수 등 |
| 이산 분포(Discrete) | 특정 조건 선택 | 지면 재질 유형 등 |

실용적으로는 균등 분포가 가장 널리 쓰입니다. 범위만 지정하면 되어 단순하고, 파라미터 분포에 대한 사전 지식이 없어도 됩니다.

## 정리

Domain Randomization은 Reality Gap의 핵심 대응 전략입니다. Dynamics DR로 물리 파라미터를, Visual DR로 시각적 다양성을 처리합니다. 랜덤화 범위는 너무 좁아도(갭 잔존), 너무 넓어도(과도 랜덤화) 문제가 됩니다. Automatic DR은 이 범위를 정책 성능에 따라 자동 조정하여 인간 튜닝 부담을 줄입니다.

다음 챕터(7-3)에서는 시뮬레이터 파라미터를 실제 로봇 측정값으로 보정하는 System Identification을 다룹니다. DR과 SysID는 상호 보완적인 관계로 함께 사용할 때 가장 효과적입니다.
