## 사례 연구의 의미

Sim-to-Real 이전은 이론적 방법론만으로는 충분하지 않습니다. 실제 시스템에서 작동하는 사례를 통해 무엇이 현실적으로 효과적인지 이해해야 합니다. 이 챕터에서는 각각 다른 접근법과 도전 과제를 가진 세 가지 대표 사례를 분석합니다.

## 사례 1: ANYmal 보행 (ETH Zurich / RSL)

스위스 취리히 연방 공과대학(ETH Zurich)의 로봇 시스템 연구소(Robotic Systems Lab, RSL)는 4족 보행 로봇 ANYmal에 강화학습 기반 보행 정책을 적용하여 Sim-to-Real 성공의 대표 사례를 만들었습니다.

### 핵심 접근법

**Actuator Network 통합:**  
단순 강체 시뮬레이터의 가장 큰 약점인 액추에이터 동역학 부정확성을 해결하기 위해, 실제 로봇에서 수집한 데이터로 Actuator Network를 학습하고 시뮬레이터에 통합했습니다. 이를 통해 단순 PD 제어 모델 대비 Reality Gap이 크게 줄었습니다.

**대규모 병렬 시뮬레이션:**  
Isaac Gym 등 GPU 병렬 시뮬레이션을 활용하여 수천 개의 환경에서 동시에 학습합니다. 빠른 샘플 수집으로 광범위한 Domain Randomization이 가능해집니다.

**계층적 제어 구조:**  
고수준 정책(보행 방향, 속도 명령)과 저수준 제어(관절 추적)를 분리하는 구조를 활용합니다. 이는 학습 공간을 줄이고 안전성을 높입니다.

### 성공 요인

```text
┌──────────────────────────────────────────────────────────────┐
│          ANYmal Sim-to-Real 성공 요인                        │
├───────────────────────────────────────────────────────────────┤
│ 1. Actuator Network로 액추에이터 비선형성 포착               │
│ 2. 다양한 지형 Randomization (경사, 계단, 불규칙 지면)      │
│ 3. 외란 랜덤화 (외력 적용, 페이로드 변화)                   │
│ 4. 관절 파라미터 SysID 기반 DR 범위 설정                    │
│ 5. 학습 후 추가 실제 데이터로 정책 파인튜닝                  │
└───────────────────────────────────────────────────────────────┘
```

후속 연구(RMA: Rapid Motor Adaptation)에서는 환경 적응 모듈을 추가하여, 처음 보는 지면(젖은 표면, 얼음 등)에도 온라인으로 적응하는 능력을 보였습니다. 이 연구는 Sim-to-Real의 범위를 "알려진 도메인 이전"에서 "미지 도메인 실시간 적응"으로 확장했습니다.

## 사례 2: OpenAI Dactyl (In-Hand Manipulation)

OpenAI의 Dactyl 프로젝트는 5손가락 로봇 손으로 루빅스 큐브를 풀거나 물체를 손 안에서 조작(in-hand manipulation)하는 데 성공했습니다. 이는 보행보다 훨씬 정밀하고 복잡한 접촉 역학이 필요한 과제입니다.

### 핵심 접근법: Automatic Domain Randomization (ADR)

Dactyl 팀의 가장 큰 기여는 ADR을 체계화한 것입니다. 랜덤화 범위를 정책 성능에 따라 자동으로 확대하는 커리큘럼 학습 방식으로, 처음에는 좁은 범위에서 기본 조작을 익히고 점차 더 극단적인 조건에서도 작동하도록 확장했습니다.

**랜덤화된 파라미터 규모:**  
물체 크기, 무게, 마찰, 관절 댐핑, 액추에이터 지연, 관측 노이즈, 조명 조건, 카메라 위치 등 수십 가지 파라미터를 동시에 랜덤화했습니다.

**Visual DR:**  
시각 기반 정책 학습에서 배경 텍스처, 물체 색상, 조명을 랜덤화하여 실제 카메라 이미지와의 시각적 갭을 줄였습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│          Dactyl의 랜덤화 구조                                │
│                                                              │
│   물리 파라미터 랜덤화          시각 파라미터 랜덤화        │
│   ┌──────────────────┐         ┌──────────────────┐         │
│   │ • 물체 질량/크기 │         │ • 배경 텍스처    │         │
│   │ • 손가락 마찰    │         │ • 물체 색상      │         │
│   │ • 관절 댐핑      │         │ • 조명 강도/방향 │         │
│   │ • 액추에이터 지연│         │ • 카메라 위치    │         │
│   │ • 관측 노이즈    │         │ • 렌더링 노이즈  │         │
│   └──────────────────┘         └──────────────────┘         │
│            └─────────────┬─────────────┘                    │
│                          ▼                                   │
│               ADR로 범위 자동 조정                           │
└──────────────────────────────────────────────────────────────┘
```

### 성공 요인

Dactyl의 성공을 이끈 핵심 요인:

1. **ADR으로 커리큘럼 자동화**: 수동 과제 설계 없이 점진적으로 어려운 조건 학습
2. **물리+시각 DR 동시 적용**: 두 갭을 함께 처리
3. **대규모 컴퓨팅**: 수천 CPU 코어로 장기간 학습
4. **별도의 상태 추정 없음**: 정책이 직접 이미지에서 행동을 예측하는 end-to-end 구조

### 한계점

Dactyl은 고정된 피험 로봇 손(Shadow Dexterous Hand)과 특정 물체(정육면체)에 최적화되었습니다. 다른 물체 형태나 다른 로봇 손으로의 일반화는 별도 학습이 필요합니다. 또한 수백만 CPU 시간이라는 막대한 계산 비용이 필요했습니다.

## 사례 3: Unitree 계열 보행 로봇

Unitree(유니트리)의 Go1, Go2, H1 등은 상업용 사족·이족 보행 로봇으로, 연구 커뮤니티에서 Sim-to-Real 검증 플랫폼으로 널리 사용됩니다.

### 대표 연구 패턴

Unitree 플랫폼에서의 Sim-to-Real 연구는 대체로 ETH/RSL 방법론을 참조합니다.

- **Isaac Lab 또는 MuJoCo 기반 학습**: GPU 병렬화된 RL 학습
- **DR 기반**: 지면 마찰, 페이로드, 관절 마찰 랜덤화
- **학습된 속도 명령 정책(velocity command policy)**: 조이스틱 명령을 보행 동작으로 변환

### Sim-to-Real 이전의 일반적 절차

```text
Unitree 기반 Sim-to-Real 일반 절차:

1. Isaac Lab / MuJoCo에서 정책 학습
   (수백만 스텝, GPU 병렬)
        │
        ▼
2. 시뮬레이션 내 평가
   (평탄 지면 → 불규칙 지면 → 계단)
        │
        ▼
3. 소프트웨어 배포 준비
   (ONNX/TorchScript 변환, latency 측정)
        │
        ▼
4. 실제 로봇 안전 테스트
   (안전 구역에서 핸들러와 함께)
        │
        ▼
5. 점진적 실환경 테스트
   (실내 → 실외 → 복잡 지형)
```

Unitree 로봇의 실제 배포 사례에서 일관되게 나타나는 점은, **지면 접촉 파라미터(마찰, 강성)**와 **관절 댐핑**에 대한 DR이 성공에 가장 큰 기여를 한다는 것입니다.

## 세 사례의 비교 분석

```text
┌──────────────────────────────────────────────────────────────────────────┐
│                  사례 비교: Sim-to-Real 접근법                           │
├────────────────┬────────────────────┬───────────────────┬────────────────┤
│ 항목           │ ANYmal (ETH RSL)   │ Dactyl (OpenAI)  │ Unitree 계열   │
├────────────────┼────────────────────┼───────────────────┼────────────────┤
│ 로봇 유형      │ 사족 보행          │ 5손가락 조작 손  │ 사족/이족      │
│ 과제 복잡도    │ 중 (보행)          │ 매우 높음(조작)  │ 중 (보행)      │
│ 핵심 기법      │ Actuator Network   │ ADR, Visual DR   │ DR, PPO        │
│ SysID 활용     │ 적극 활용          │ 제한적           │ 보통           │
│ 계산 규모      │ 대규모             │ 매우 대규모      │ 중간           │
│ 상업 활용      │ 연구/산업          │ 연구             │ 광범위한 상업  │
│ 일반화         │ 다양한 지형        │ 특정 물체 형태   │ 일반 지형      │
└────────────────┴────────────────────┴───────────────────┴────────────────┘
```

## 공통 성공 요인 도출

세 사례를 관통하는 공통 요인:

**1. 충분히 넓은 DR**  
모든 사례에서 좁은 랜덤화 범위는 실패로 이어졌습니다. 처음에 "너무 넓다"고 느껴질 정도의 범위가 실제로는 필요합니다.

**2. 물리적으로 타당한 DR 설계**  
아무 파라미터나 랜덤화하는 것이 아니라, Reality Gap의 실제 원인에 해당하는 파라미터를 선택했습니다.

**3. 대규모 병렬 학습**  
실제 로봇으로는 얻을 수 없는 수백만~수억 스텝의 경험을 시뮬레이션으로 확보했습니다.

**4. 안전 설계**  
실제 로봇 테스트에서 단계적으로 위험도를 높이는 절차를 따랐습니다.

**5. 지속적인 실제-시뮬레이션 피드백**  
실제 로봇 실험에서 얻은 관찰을 시뮬레이션 설정에 반영하는 반복 루프를 유지했습니다.

## 정리

ANYmal은 Actuator Network와 체계적 SysID로 액추에이터 갭을 줄였고, Dactyl은 ADR과 대규모 Visual DR로 고난도 조작을 실현했으며, Unitree 연구들은 DR 기반 보행 정책의 범용 가능성을 보여주었습니다. 세 사례 모두 대규모 시뮬레이션, 잘 설계된 DR, 단계적 실제 테스트가 공통 성공 요인이었습니다.
