## System Identification이란

**System Identification(시스템 식별, SysID)** 은 실제 물리 시스템을 실험적으로 관측하여, 그 수학적 모델의 파라미터를 추정하는 방법론입니다. 로보틱스에서는 실제 로봇을 직접 동작시키거나 정적 측정을 통해, 시뮬레이터에 사용할 파라미터를 결정합니다.

Domain Randomization이 "어차피 정확한 값을 모르니 범위를 넓게 잡자"는 전략이라면, SysID는 "실제 값에 가능한 가까운 파라미터를 시뮬레이터에 넣자"는 전략입니다. 두 접근법은 상충하지 않으며, 실제 현장에서는 SysID로 기준값을 확보한 후 그 주변에서 DR을 수행하는 방식으로 병행합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│          SysID + Domain Randomization 결합 구조              │
│                                                              │
│   실제 로봇 ──SysID──▶ 기준 파라미터 θ*                    │
│                               │                              │
│                               ▼                              │
│                   θ* ± Δθ 범위에서 DR 수행                  │
│                               │                              │
│                               ▼                              │
│                     학습된 강건 정책 π                       │
│                               │                              │
│                               ▼                              │
│                         실제 로봇 배포                       │
└──────────────────────────────────────────────────────────────┘
```

## 주요 SysID 기법

### 1. 정적 측정 (Static Measurement)

가장 단순한 방법으로, 로봇을 정지 상태에서 측정합니다.

- **질량 측정**: 저울로 각 링크를 분리하여 측정합니다.
- **관성 텐서 추정**: CAD 모델 또는 진자 실험(pendulum test)으로 추정합니다.
- **중심 위치**: 적분 또는 평형점 실험으로 측정합니다.

정적 측정은 정확하지만, 동적 특성(댐핑, 마찰)은 파악하지 못합니다.

### 2. 동적 실험 (Dynamic Excitation)

로봇을 실제로 움직이면서 입력과 출력을 기록하여 파라미터를 역산합니다.

**관절 마찰 측정 예시:**  
다양한 속도로 관절을 회전시키면서 명령 토크와 실제 토크(관성만에 의한 예측값 대비)의 차이를 측정합니다. 이 차이가 마찰 토크이며, 쿨롱 마찰(Coulomb friction)과 점성 마찰(viscous friction) 모델에 피팅합니다.

```text
관절 마찰 모델 피팅 (개념 이해용 의사코드):

측정 데이터: (속도 v_i, 토크 차이 τ_friction_i) 쌍

모델: τ_friction = μ_c * sign(v) + μ_v * v
       μ_c: 쿨롱 마찰 계수
       μ_v: 점성 마찰 계수

최적화: min Σ (τ_friction_i - (μ_c * sign(v_i) + μ_v * v_i))²
        over μ_c, μ_v
```

**보행 로봇 SysID:**  
로봇을 다양한 속도로 보행시키면서 관성 측정 장치(IMU)와 관절 인코더 데이터를 수집합니다. 시뮬레이션 동일 동작의 출력과 비교하여 오차를 최소화하는 파라미터를 탐색합니다.

### 3. 최적화 기반 SysID

고차원 파라미터를 동시에 추정할 때는 최적화 기법을 사용합니다. 실제 로봇에서 수집한 궤적 데이터를 시뮬레이터에서 재현하려 할 때, 시뮬레이션 출력과 실제 데이터의 차이를 손실 함수로 정의하고 최적화합니다.

## Real-to-Sim 접근법

**Real-to-Sim**은 실제 환경/로봇의 데이터로부터 시뮬레이터 설정을 역으로 조정하는 개념입니다. Sim-to-Real이 "시뮬에서 학습해 현실에 이전"이라면, Real-to-Sim은 "현실을 보고 시뮬을 현실에 맞게 수정"하는 방향입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│          Real-to-Sim 루프                                    │
│                                                              │
│   실제 로봇 실험                                             │
│        │                                                     │
│        ▼                                                     │
│   데이터 수집 (관절 상태, IMU, 힘/토크)                     │
│        │                                                     │
│        ▼                                                     │
│   시뮬레이터에서 동일 조건 재생                              │
│        │                                                     │
│        ▼                                                     │
│   실제 vs 시뮬 오차 계산                                     │
│        │                                                     │
│        ▼                                                     │
│   파라미터 최적화 (역문제 풀기)                              │
│        │                                                     │
│        ▼                                                     │
│   보정된 시뮬레이터 ──────────▶ 정책 학습                  │
└──────────────────────────────────────────────────────────────┘
```

Real-to-Sim의 핵심 과제는 **역문제(inverse problem)** 의 비볼록성(non-convexity)입니다. 파라미터 공간이 넓을수록 전역 최적해를 찾기 어렵고, 다수의 파라미터 조합이 비슷한 궤적을 만들 수 있습니다. 이를 완화하기 위해 물리적 사전 지식(physical prior)으로 탐색 공간을 제한하거나, 베이지안 최적화를 사용합니다.

## Actuator Network: 액추에이터 모델 학습

전통적인 SysID는 물리 모델을 가정하고 그 파라미터를 추정합니다. 그러나 실제 액추에이터는 모델화하기 어려운 비선형성을 포함합니다. **Actuator Network(액추에이터 네트워크)** 는 이 문제를 학습 기반으로 해결합니다.

### 개념

액추에이터 네트워크는 작은 신경망으로, **관절 위치 명령 + 현재 상태 → 실제 토크/속도 출력** 관계를 데이터에서 직접 학습합니다. 물리 모델 없이도 액추에이터의 복잡한 비선형 동역학을 포착할 수 있습니다.

```text
Actuator Network 구조 (개념 이해용 의사코드):

입력:
  - 현재 관절 위치 q(t)
  - 현재 관절 속도 dq(t)
  - 명령 위치 q_des(t)
  - 이전 타임스텝 히스토리 [q(t-1), dq(t-1), ...]

신경망:
  - 소형 MLP (2-3 레이어)
  - 입력 → 숨겨진 층 → 출력 토크 τ

학습:
  - 실제 로봇에서 다양한 동작 수집
  - 시뮬레이터의 단순 PD 모델 출력과 실제 토크 차이를 타깃
  - 또는 직접 실제 토크 측정값을 타깃
```

### Actuator Network의 활용

ANYmal 보행 로봇 연구(ETH Zurich, RSL 연구팀)에서 액추에이터 네트워크를 시뮬레이터에 통합하는 방식을 사용하여 Reality Gap을 크게 줄인 사례가 있습니다. 학습된 액추에이터 네트워크는 PD 제어기 대신 시뮬레이터 내부에 삽입되어, 단순 강체 시뮬레이터를 실제 액추에이터에 훨씬 가깝게 만듭니다.

| 비교 항목 | 전통 PD 모델 | Actuator Network |
|-----------|-------------|-----------------|
| 정확도 | 낮음 (선형 근사) | 높음 (비선형 포착) |
| 해석 가능성 | 높음 | 낮음 (블랙박스) |
| 데이터 요구량 | 적음 | 수천~수만 샘플 |
| 일반화 | 모델 범위 내 | 학습 데이터 범위 내 |

## SysID의 한계와 실용적 고려

SysID가 완벽하지 않은 이유:

1. **측정 오차**: 센서 자체에도 오차가 있습니다.
2. **시간 변화**: 로봇 부품은 마모되고, 환경은 변합니다. SysID는 특정 시점의 스냅샷입니다.
3. **역문제 불완전성**: 관측 가능한 신호만으로는 모든 내부 파라미터를 유일하게 결정할 수 없습니다.
4. **비용**: 체계적인 SysID는 상당한 실험 시간과 전문성이 필요합니다.

이런 한계 때문에, 실용적 Sim-to-Real 파이프라인에서 SysID는 **DR의 기준점을 잡는 용도**로 사용하고, DR로 잔여 불확실성을 커버하는 조합이 표준적입니다. 완벽한 시뮬레이터를 만들기보다, "충분히 좋은" 시뮬레이터에서 강건한 정책을 학습하는 것이 현실적 목표입니다.

## 정리

System Identification은 실제 로봇의 물리 파라미터를 측정·추정하여 시뮬레이터를 보정하는 방법입니다. 정적 측정, 동적 실험, 최적화 기반 방법이 있으며, Actuator Network는 복잡한 액추에이터 동역학을 학습 기반으로 포착합니다. SysID와 DR은 상호 보완적 관계로, 실용적 Sim-to-Real 파이프라인에서 함께 사용됩니다.
