## 접촉이 왜 어려운가

로봇이 물체를 집거나 바닥을 밟는 순간, **접촉(contact)** 이 발생합니다. 접촉은 Physical AI에서 가장 모델링하기 어려운 현상 중 하나입니다.

직관적으로 이해하기 어렵지 않아 보입니다. "두 물체가 닿으면 서로 밀어내는 힘이 생긴다." 그러나 실제 접촉의 세부 사항은 매우 복잡합니다.

- 접촉 면적이 점인가, 면인가?
- 미끄러짐은 언제 시작되는가?
- 접촉 시 충격력은 얼마인가?
- 탄성 vs 비탄성 충돌?

이 복잡성이 **Sim-to-Real 갭의 주요 원인** 중 하나입니다. 시뮬레이터가 근사하는 접촉 모델이 현실과 다를 때 훈련된 정책이 실패합니다.

## 접촉 모델: 소프트 vs 하드

물리 엔진은 두 가지 기본적인 접촉 모델을 사용합니다.

### 하드 접촉 모델 (Hard Contact)

두 물체가 **절대로 서로 관통하지 않는다**는 제약을 완전히 적용합니다. 충돌 순간 즉각적인 충격(impulse)이 적용되어 속도가 불연속적으로 변합니다.

수학적으로는 **LCP(Linear Complementarity Problem)** 를 풀어야 합니다.

```
f >= 0,  g >= 0,  f * g = 0

f: 접촉 힘 (법선 방향)
g: 분리 거리 (gap)

해석: 접촉 힘은 비음수, 분리 거리는 비음수,
      하나가 0이면 다른 하나는 양수일 수 있음
      (접촉 중이면 g=0, 접촉 힘 f>0;
       분리 상태면 f=0, 거리 g>0)
```

하드 접촉의 특징: 물리적으로 정확하지만, LCP 풀기가 계산 비용이 크고 수치적으로 불안정할 수 있습니다. 특히 많은 물체가 동시에 접촉하는 상황에서 어렵습니다.

### 소프트 접촉 모델 (Soft Contact)

물체가 **약간의 관통(penetration)** 을 허용하고, 관통 깊이에 비례하는 복원 힘을 가합니다. 마치 실제 물체에 약간의 탄성이 있는 것처럼 근사합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              소프트 접촉 모델                                  │
│                                                              │
│  물체 A        물체 B                                        │
│   ────────   ────────                                        │
│              ←→ δ (관통 깊이)                                │
│                                                              │
│  복원 힘 F = k * δ + c * δ_dot                              │
│                                                              │
│  k: 강성 계수 (stiffness)   - 클수록 딱딱함                   │
│  c: 감쇠 계수 (damping)     - 클수록 에너지 흡수              │
│  δ: 관통 깊이                                                │
│  δ_dot: 관통 깊이 변화율                                     │
└──────────────────────────────────────────────────────────────┘
```

MuJoCo가 소프트 접촉 모델을 채택합니다. 수치적으로 안정하고 미분 가능하다는 장점이 있어 gradient-based 학습에 유리합니다. 하지만 물체가 약간 관통하는 부자연스러운 현상이 생길 수 있습니다.

## Coulomb 마찰과 마찰 콘

접촉에는 법선 방향 힘(Normal force) 외에 접선 방향 힘(Tangential force, 마찰)도 작용합니다.

**Coulomb 마찰 법칙**:

```
정적 마찰력: |f_t| <= μ_s * f_n   (미끄러짐 없을 때)
동적 마찰력: |f_t| = μ_k * f_n    (미끄러지는 동안)

f_t: 접선 방향 마찰력
f_n: 법선 방향 힘 (누르는 힘)
μ_s: 정적 마찰 계수 (static)
μ_k: 동적 마찰 계수 (kinetic, 보통 μ_k < μ_s)
```

**마찰 콘(Friction Cone)**은 마찰을 3D로 시각화하는 개념입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                   마찰 콘 시각화                               │
│                                                              │
│          ↑ f_n (법선 방향)                                   │
│          │                                                   │
│         /│\                                                  │
│        / │ \     마찰 콘 (반각 = arctan(μ))                  │
│       /  │  \                                                │
│      /   │   \                                               │
│     /    │    \                                              │
│    /     ●     \   ← 접촉점                                  │
│                                                              │
│  콘 안쪽: 미끄러짐 없음 (정적 마찰 상태)                       │
│  콘 표면: 미끄러지는 순간                                      │
│  콘 바깥: 물리적으로 불가능                                    │
│                                                              │
│  3D에서: 콘 = 원뿔, 중심이 법선 방향                          │
└──────────────────────────────────────────────────────────────┘
```

합력 벡터(법선 + 마찰)가 마찰 콘 안에 있으면 미끄러짐이 없고, 콘 밖으로 나가면 미끄러짐이 시작됩니다.

로봇 파지에서 마찰 콘은 매우 중요합니다. 물체를 안정적으로 잡으려면 그리퍼가 가하는 힘이 마찰 콘 안에 있어야 합니다. 이를 **마찰 기반 파지 분석(friction-based grasp analysis)** 이라 합니다.

## 충돌 감지: 브로드 단계와 내로우 단계

충돌 감지(collision detection)는 어떤 물체 쌍이 접촉하고 있는지 결정하는 과정입니다. 물체 수가 n개라면 O(n²) 쌍을 모두 검사하는 것은 비효율적입니다. 대신 두 단계로 나눕니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              충돌 감지 2단계                                   │
│                                                              │
│  ┌─────────────────────────────────────────┐                 │
│  │  브로드 단계 (Broad Phase)               │                 │
│  │                                         │                 │
│  │  간단한 바운딩 볼륨(AABB, 구)으로         │                 │
│  │  충돌 가능성 있는 쌍을 빠르게 선별         │                 │
│  │                                         │                 │
│  │  방법: BVH, Spatial Hashing             │                 │
│  │  비용: O(n log n) ~ O(n)                │                 │
│  └─────────────────────────────────────────┘                 │
│                    │ 후보 쌍만                                │
│                    ▼                                         │
│  ┌─────────────────────────────────────────┐                 │
│  │  내로우 단계 (Narrow Phase)              │                 │
│  │                                         │                 │
│  │  실제 기하형상으로 정밀 충돌 검사         │                 │
│  │  접촉점, 법선 방향, 관통 깊이 계산        │                 │
│  │                                         │                 │
│  │  방법: GJK, EPA, SAT                    │                 │
│  │  비용: 쌍당 O(1) ~ O(복잡도)             │                 │
│  └─────────────────────────────────────────┘                 │
└──────────────────────────────────────────────────────────────┘
```

### 브로드 단계

**AABB(Axis-Aligned Bounding Box)**: 각 물체를 축 정렬 직육면체로 근사합니다. AABB끼리의 겹침 테스트는 6번의 비교만으로 가능합니다.

**BVH(Bounding Volume Hierarchy)**: AABB의 계층 구조로, 공간을 트리 방식으로 분할하여 O(n log n) 충돌 검사를 가능하게 합니다.

**Spatial Hashing**: 공간을 균일 격자로 나누고 물체를 격자에 해시. 인접 격자의 물체만 검사합니다.

### 내로우 단계

**GJK(Gilbert-Johnson-Keerthi) 알고리즘**: 두 볼록 물체 사이의 최소 거리를 계산합니다. 볼록 물체의 Minkowski 차집합을 이용한 효율적인 알고리즘입니다.

**EPA(Expanding Polytope Algorithm)**: GJK 이후 실제 관통 깊이와 법선 방향을 계산합니다.

## 접촉이 Sim-to-Real에서 어려운 이유

접촉 모델링의 불완전함은 Sim-to-Real 갭의 주요 원인입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│          접촉 관련 Sim-to-Real 갭 원인                         │
│                                                              │
│  현실의 접촉                   시뮬레이터의 근사               │
│  ─────────────────────────    ─────────────────────────      │
│  연속 접촉면 → 분산 힘          단일 접촉점 또는 소수 점        │
│  탄성-비탄성 혼합 충격          단순 반발 계수 (COR)            │
│  온도 의존 마찰                 상수 마찰 계수                  │
│  표면 조도, 비등방성 마찰       등방성 Coulomb 마찰             │
│  접촉면 변형                   강체 근사                        │
│  롤링/피벗 마찰                 종종 무시됨                     │
└──────────────────────────────────────────────────────────────┘
```

**실제 사례**: 로봇 손가락이 물컵을 집는 시뮬레이션에서는 성공하지만, 실제 로봇에서는 컵이 미끄러지거나 떨어지는 경우가 흔합니다. 실제 손가락-유리 접촉의 마찰 계수와 접촉 면적을 시뮬레이터에서 정확히 재현하기 어렵기 때문입니다.

### 완화 전략

**마찰 계수 랜덤화**: 훈련 중 마찰 계수를 넓은 범위에서 무작위로 변화시킵니다. 다양한 마찰 조건에서 작동하는 강건한 정책을 학습합니다.

**힘/토크 센서 피드백**: 실제 로봇에서 접촉 힘을 직접 측정하여 제어에 반영합니다. 시뮬레이터의 접촉 모델 오차를 실시간 센서로 보상합니다.

**소프트 그리퍼**: 실제 그리퍼를 연성 재료로 만들면 접촉 면적이 늘어나고 미끄러짐이 줄어듭니다. 물리 모델링이 어렵지만 현실에서 더 강건합니다.

**촉각 센서(Tactile Sensor)**: 접촉 분포를 고해상도로 측정하는 센서입니다. GelSight, Digit 같은 연구용 촉각 센서가 파지 안정성 향상에 활용됩니다.

## 핵심 요약

- **하드 접촉**: 관통 없음, LCP로 계산, 정확하지만 불안정 가능
- **소프트 접촉**: 약간의 관통 허용, 스프링-댐퍼 근사, 수치 안정
- **Coulomb 마찰**: 정적/동적 마찰 계수, 마찰 콘으로 시각화
- **충돌 감지**: 브로드 단계(빠른 선별) + 내로우 단계(정밀 계산)
- **Sim-to-Real**: 접촉 모델 오차가 현실에서 파지 실패의 주원인

접촉과 마찰의 복잡성은 Physical AI 연구에서 계속 다루어야 할 미해결 문제입니다. 다음 챕터에서는 이 물리 환경을 "인식"하는 **센서 시뮬레이션** 을 살펴봅니다.
