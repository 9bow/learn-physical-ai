## 물리 엔진이란

**물리 엔진(physics engine)** 은 물리 법칙을 수치적으로 시뮬레이션하는 소프트웨어 라이브러리입니다. 뉴턴의 운동 법칙, 충돌, 마찰, 관절 제약 등을 시간 순서대로 계산하여 물체의 운동을 예측합니다.

로봇 시뮬레이터가 "가상 로봇"을 움직일 수 있는 것은 물리 엔진 덕분입니다. Isaac Sim은 PhysX를 내장하고, MuJoCo는 자체 물리 엔진을 가지며, Gazebo는 ODE와 Bullet을 지원합니다.

## 물리 시뮬레이션의 기본 루프

모든 물리 엔진은 본질적으로 동일한 루프를 반복합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              물리 시뮬레이션 메인 루프                          │
│                                                              │

│       │                                                      │
│  ┌────▼──────────────────────────────────────────────────┐   │
│  │  ① 충돌 감지 (Collision Detection)                     │   │
│  │     어떤 물체 쌍이 접촉하고 있는가?                     │   │
│  └────┬──────────────────────────────────────────────────┘   │
│       ▼                                                      │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  ② 충돌 해소 / 제약 처리 (Constraint Solving)           │  │
│  │     접촉 힘, 관절 제약 힘 계산                           │  │
```

## 수치 적분기

연속 미분방정식을 이산 시간 스텝으로 근사하는 것이 수치 적분입니다. 선택하는 적분 방법에 따라 정확도, 안정성, 계산 비용이 달라집니다.

### 오일러 적분 (Explicit Euler)

```
x(t + Δt) = x(t) + Δt * f(x(t))
```

가장 간단하지만 **수치적으로 불안정**합니다. Δt가 크면 에너지가 증가하여 물체가 폭발적으로 튀어나가는 현상이 발생합니다.

### 심플렉틱 오일러 (Symplectic Euler)

```
v(t + Δt) = v(t) + Δt * a(t)
x(t + Δt) = x(t) + Δt * v(t + Δt)   ← 업데이트된 속도 사용
```

에너지를 장기적으로 보존하는 경향이 있어, 게임과 실시간 시뮬레이션에서 널리 사용됩니다. PhysX의 기본 적분기입니다.

### Runge-Kutta 4차 (RK4)

```
k1 = f(x, t)
k2 = f(x + Δt/2 * k1, t + Δt/2)
k3 = f(x + Δt/2 * k2, t + Δt/2)
k4 = f(x + Δt * k3, t + Δt)
x(t + Δt) = x(t) + Δt/6 * (k1 + 2k2 + 2k3 + k4)
```

더 높은 정확도이지만, 스텝당 4번의 함수 평가가 필요합니다.

### 시간 스텝의 중요성

```text
┌──────────────────────────────────────────────────────────────┐
│              시간 스텝 크기와 시뮬레이션 품질                   │
│                                                              │
│  Δt 너무 크면:                                               │
│    • 수치 불안정 → 물체가 "폭발"                               │
│    • 접촉 관통(penetration) 발생                               │
│    • 물리적으로 비현실적 거동                                   │
│                                                              │
│  Δt 너무 작으면:                                              │
│    • 계산 비용 증가 (1/Δt에 비례)                              │
│    • 훈련 속도 저하                                           │
│                                                              │
│  일반적 권장값:                                               │
│    보행 로봇 제어: 1ms ~ 4ms (250Hz ~ 1kHz)                  │
│    조작 태스크: 2ms ~ 10ms                                    │
│    RL 탐색(빠른 훈련): 5ms ~ 20ms                             │
└──────────────────────────────────────────────────────────────┘
```

## 세 가지 주요 물리 엔진

### PhysX 5 (NVIDIA)

NVIDIA가 개발하는 GPU 가속 물리 엔진입니다. 원래 게임 물리용으로 시작했지만, PhysX 5에서는 과학 시뮬레이션과 로보틱스를 위한 기능이 크게 강화되었습니다.

**주요 특징**:
- GPU 가속 병렬 시뮬레이션 (수천 개 씬 동시 실행)
- 강체, 연성체(soft body), 천(cloth), 유체(fluid) 지원
- ArticulationAPI를 통한 로봇 관절 체인 정밀 처리
- Isaac Sim과 통합, CUDA 텐서 인터페이스로 RL 훈련 지원

**적합한 용도**: NVIDIA 생태계에서의 대규모 RL 훈련, 산업 시뮬레이션

```python
# PhysX ArticulationAPI 개념 (Isaac Sim 스타일, 의사코드)
# 개념 이해용 의사코드
robot_prim = stage.GetPrimAtPath("/World/Robot")
articulation = UsdPhysics.ArticulationRootAPI.Apply(robot_prim)

# 관절 드라이브 설정
joint = stage.GetPrimAtPath("/World/Robot/Joint1")
drive = UsdPhysics.DriveAPI.Apply(joint, "angular")
drive.GetStiffnessAttr().Set(10000.0)   # PD 제어 강성
drive.GetDampingAttr().Set(1000.0)       # PD 제어 감쇠
```

### MuJoCo (DeepMind)

2010년대 초 Emo Todorov가 개발하고, 2021년 DeepMind가 인수하여 2022년 오픈소스로 공개한 물리 엔진입니다. 로보틱스 RL 연구의 표준 환경을 제공합니다.

**주요 특징**:
- 접촉 모델이 특히 우수 (소프트 접촉 모델)
- 근육 모델 지원 (생체역학 시뮬레이션)
- 역동역학(inverse dynamics) 계산 내장
- MJX: JAX 기반 GPU/TPU 가속 버전

**표준 벤치마크 환경**: Ant, HalfCheetah, Humanoid, Hopper, Walker2D 등은 수백 편의 RL 논문에서 사용된 표준입니다.

```python
import mujoco
import numpy as np

# MuJoCo 모델 로드 및 시뮬레이션
model = mujoco.MjModel.from_xml_path("robot.xml")
data = mujoco.MjData(model)

# 시뮬레이션 한 스텝
data.ctrl[0] = 1.0   # 관절 0에 제어 입력
mujoco.mj_step(model, data)

# 상태 읽기
print("관절 각도:", data.qpos)
print("관절 속도:", data.qvel)
print("EE 위치:", data.xpos[model.body("end_effector").id])
```

### Bullet Physics

오픈소스 물리 엔진으로, 게임 산업과 영화 시각 효과에서도 사용됩니다. ROS와의 통합이 잘 되어 있어 Gazebo의 기본 엔진으로 사용됩니다.

**주요 특징**:
- PyBullet Python 바인딩으로 쉬운 프로토타이핑
- 강체, 연성체, 차량 물리 지원
- CPU 기반 (GPU 가속 제한적)
- URDF, SDF 로봇 모델 로딩 지원

```python
import pybullet as p
import pybullet_data

# PyBullet 초기화
p.connect(p.DIRECT)   # 헤드리스 (또는 p.GUI로 시각화)
p.setAdditionalSearchPath(pybullet_data.getDataPath())
p.setGravity(0, 0, -9.8)

# 바닥과 로봇 로드
p.loadURDF("plane.urdf")
robot = p.loadURDF("kuka_iiwa/model.urdf", basePosition=[0, 0, 0])

# 한 스텝 시뮬레이션
p.stepSimulation()
joint_state = p.getJointState(robot, 0)
print("관절 각도:", joint_state[0])
```

## 강체 vs 연성체 시뮬레이션

지금까지 다룬 것은 주로 **강체(rigid body)** 시뮬레이션입니다. 그러나 로보틱스에서는 **연성체(soft body)** 도 중요합니다.

| 대상 | 시뮬레이션 방법 | 물리 엔진 지원 |
|------|----------------|-------------|
| 금속 링크, 하우징 | 강체 | 모든 엔진 |
| 고무 그리퍼, 피부 | 연성체 (FEM) | PhysX, MuJoCo |
| 케이블, 밧줄 | 1D 연성체 | PhysX, Bullet |
| 직물, 의류 | 천(cloth) 시뮬레이션 | PhysX |
| 액체, 분말 | 입자 기반(SPH, MPM) | PhysX |

연성체 시뮬레이션은 계산 비용이 훨씬 크며, 현재 대부분의 로봇 RL 연구는 강체 근사를 사용합니다. 하지만 소프트 로봇(soft robot)이나 정교한 조작(dexterous manipulation)에서는 연성체 모델링이 중요해집니다.

## 시뮬레이터 선택 기준

- **NVIDIA 생태계 + 대규모 병렬 RL**: Isaac Sim + PhysX
- **RL 연구 벤치마크**: MuJoCo
- **ROS2 통합 프로토타이핑**: Gazebo + Bullet
- **빠른 교육용 실험**: PyBullet
- **최신 GPU 네이티브 연구**: Genesis(자체 GPU 네이티브 엔진)

다음 챕터에서는 물리 시뮬레이션에서 가장 어려운 부분 중 하나인 **접촉과 마찰 모델링** 을 깊이 살펴봅니다.
