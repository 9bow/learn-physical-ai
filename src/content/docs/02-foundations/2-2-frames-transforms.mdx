## 좌표 프레임이란

물리 세계에서 "로봇 팔 끝이 어디 있는가"를 표현하려면 **기준(reference)** 이 필요합니다. 이 기준이 바로 **좌표 프레임(coordinate frame)** 입니다.

로봇 시스템에는 여러 개의 좌표 프레임이 공존합니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              로봇 시스템의 좌표 프레임들                        │
│                                                              │
│  {W} 월드 프레임     : 전체 환경의 절대 기준                   │
│  {B} 로봇 베이스 프레임: 로봇 기저에 고정                      │
│  {L1}, {L2}... 링크 프레임: 각 링크에 고정                    │
│  {E} 엔드이펙터 프레임: 로봇 팔 끝에 고정                      │
│  {C} 카메라 프레임   : 카메라에 고정                           │
│  {O} 물체 프레임     : 조작 대상 물체에 고정                   │
│                                                              │
│  월드     ◀── 외부 캘리브레이션 ──▶  로봇 베이스             │
│  로봇 베이스 ──▶ 링크1 ──▶ 링크2 ──▶ EE                      │
│  EE ──▶ 카메라 (손목 카메라인 경우)                           │
└──────────────────────────────────────────────────────────────┘
```

**변환(transform)** 은 한 프레임에서 다른 프레임으로 좌표를 변환하는 수학적 연산입니다. "물체가 카메라 프레임에서 (x=0.3, y=-0.1, z=0.5)에 있다"는 정보를 "월드 프레임에서는 어디인가"로 변환하려면 카메라 → 로봇 베이스 → 월드 프레임으로의 변환 체인이 필요합니다.

## 회전 행렬 (Rotation Matrix)

3D 공간에서의 순수 회전은 3x3 **회전 행렬(rotation matrix)** R로 표현됩니다. 회전 행렬은 다음 성질을 만족합니다.

```
R^T * R = I  (직교 행렬)
det(R) = +1  (행렬식이 1)

이를 만족하는 3x3 행렬의 집합을 SO(3)이라 합니다.
```

예를 들어 z축 중심의 θ 라디안 회전:

```text
         ┌  cos(θ)  -sin(θ)  0 ┐
Rz(θ) = │  sin(θ)   cos(θ)  0 │
         └    0        0     1 ┘
```

복잡한 회전은 기본 회전 행렬을 곱하여 표현합니다. 예를 들어 x축 α 회전 후 y축 β 회전: R = Ry(β) * Rx(α).

## 동차변환행렬 (Homogeneous Transformation Matrix)

회전만으로는 부족합니다. 이동(translation)도 함께 표현해야 합니다. **동차변환행렬(homogeneous transformation matrix)** T는 4x4 행렬로 회전과 이동을 하나로 통합합니다.

```text
         ┌         ┬     ┐
         │    R    │  p  │     R: 3x3 회전행렬
    T =  │  (3x3) │(3x1)│     p: 3x1 이동벡터(translation)
         ├─────────┼─────┤
         │  0  0  0│  1  │
         └         ┴     ┘

즉:       ┌ r11 r12 r13  px ┐
          │ r21 r22 r23  py │
    T =   │ r31 r32 r33  pz │
          └  0   0   0    1 ┘
```

동차변환행렬이 강력한 이유는 **행렬 곱으로 변환을 합성**할 수 있기 때문입니다.

```
{A}에서 {B}로의 변환: T_AB
{B}에서 {C}로의 변환: T_BC
{A}에서 {C}로의 변환: T_AC = T_AB * T_BC
```

로봇 팔에서는 베이스에서 엔드이펙터까지의 변환이 각 관절 변환의 곱입니다.

```
T_0E = T_01 * T_12 * T_23 * ... * T_(n-1)n
```

이것이 바로 **순기구학(forward kinematics)** 계산의 핵심입니다.

## 오일러각 (Euler Angles)

**오일러각**은 세 번의 순차적 회전으로 임의의 자세를 표현하는 방법입니다. 가장 흔한 표기는 **ZYX 오일러각** 또는 **롤-피치-요(Roll-Pitch-Yaw, RPY)**입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                 롤-피치-요 (RPY)                              │
│                                                              │
│  Yaw (요, ψ)   : z 축 중심 회전 - 좌우 방향 전환             │
│  Pitch (피치, θ): y 축 중심 회전 - 앞뒤 기울임               │
│  Roll (롤, φ)  : x 축 중심 회전 - 좌우 기울임               │
│                                                              │
│  항공기로 비유:                                               │
│  Yaw = 방향 전환 (코 좌우)                                   │
│  Pitch = 상승/하강 (코 위아래)                               │
│  Roll = 날개 기울기 (좌우로 기움)                            │
└──────────────────────────────────────────────────────────────┘
```

오일러각의 장점: 직관적이고 이해하기 쉽습니다. 3개의 숫자로 자세를 표현합니다.

### 짐벌 락(Gimbal Lock) 문제

오일러각의 치명적 단점이 있습니다. 특정 자세에서 두 회전 축이 일치하면 자유도를 하나 잃는 **짐벌 락** 현상이 발생합니다.

ZYX 표기에서 Pitch = ±90°가 되면 Roll과 Yaw가 동일한 회전축을 공유합니다. 이 순간 두 자유도가 하나의 축을 공유하여 특정 방향의 자세를 표현할 수 없게 됩니다.

짐벌 락의 실제 문제:
- 로봇 팔이 특이 자세에서 불연속적으로 움직일 수 있습니다.
- 경로 보간(interpolation) 중에 예상치 못한 갑작스런 회전이 발생합니다.
- 수치 계산에서 불안정성이 생깁니다.

## 쿼터니언 (Quaternion)

**쿼터니언**은 수학자 Hamilton이 1843년에 도입한 4차원 수 체계입니다. 형태:

```
q = w + xi + yj + zk = [w, x, y, z]

여기서 i, j, k는 허수 단위:
  i² = j² = k² = ijk = -1
```

단위 쿼터니언(|q| = 1)은 3D 회전을 표현합니다. 회전축 n = (nx, ny, nz)와 회전 각도 θ에 대해:

```
q = [cos(θ/2), sin(θ/2)*nx, sin(θ/2)*ny, sin(θ/2)*nz]
```

### 쿼터니언의 장점

```text
┌──────────────────────────────────────────────────────────────┐
│           회전 표현 방법 비교                                  │
├───────────────┬──────────┬──────────┬───────────────────────┤
│ 속성          │ 오일러각 │ 회전행렬 │ 쿼터니언              │
├───────────────┼──────────┼──────────┼───────────────────────┤
│ 파라미터 수   │   3개    │  9개     │   4개                 │
│ 짐벌 락       │  발생    │ 없음     │  없음                 │
│ 보간 편의성   │  나쁨    │ 어려움   │  매우 좋음 (SLERP)    │
│ 계산 효율     │  보통    │ 나쁨     │  좋음                 │
│ 직관성        │  좋음    │ 보통     │  낮음                 │
│ 유일성        │  비유일  │ 유일     │  이중 덮개(±q 동치)   │
└───────────────┴──────────┴──────────┴───────────────────────┘
```

쿼터니언은 **짐벌 락이 없고**, **보간이 쉽습니다**. 두 자세 사이의 부드러운 보간(Spherical Linear Interpolation, SLERP)이 쿼터니언에서 자연스럽게 계산됩니다.

```
SLERP(q1, q2, t) = q1 * (q1^{-1} * q2)^t     (0 ≤ t ≤ 1)
```

t=0이면 q1, t=1이면 q2, 중간값은 두 자세 사이를 일정 각속도로 보간합니다.

### 쿼터니언 곱

두 회전의 합성은 쿼터니언 곱으로 표현됩니다.

```
q_total = q2 * q1    (q1 먼저, q2 나중)
```

쿼터니언 곱은 교환 법칙이 성립하지 않습니다(q2*q1 ≠ q1*q2). 회전의 순서가 결과에 영향을 미치기 때문입니다. 이는 회전 행렬 곱과 동일한 성질입니다.

## 로보틱스 라이브러리에서의 사용

현대 로봇 시뮬레이터와 라이브러리는 내부적으로 쿼터니언을 사용하지만, 사용자 인터페이스에서 오일러각을 지원하기도 합니다.

개념 이해용 의사코드 (아래는 실제 scipy/numpy를 활용한 예시):

```python
import numpy as np
from scipy.spatial.transform import Rotation

# 오일러각 (ZYX 순서, 단위: 라디안)
euler_angles = [0.1, 0.2, 0.3]   # roll, pitch, yaw

# 오일러각 → 쿼터니언 변환
r = Rotation.from_euler('xyz', euler_angles)
quat = r.as_quat()   # [x, y, z, w] 순서
print(f"쿼터니언: {quat}")

# 쿼터니언 → 회전 행렬 변환
rot_matrix = r.as_matrix()   # 3x3 행렬

# 두 회전의 합성
r1 = Rotation.from_euler('z', np.pi/4)   # z축 45도
r2 = Rotation.from_euler('x', np.pi/6)   # x축 30도
r_combined = r2 * r1   # r1 먼저, r2 나중
```

Isaac Sim에서는 `pxr.Gf.Quatf` 또는 `torch.Tensor`로 쿼터니언을 표현하며, Isaac Lab에서는 GPU 텐서 연산으로 수천 개의 로봇 자세를 동시에 처리합니다.

## 변환 체인의 예시

카메라로 물체를 인식하고, 그 위치를 로봇 제어에 사용하는 전형적인 시나리오:

```text
물체 위치 (카메라 프레임)
        │ T_camera_to_ee  (카메라 외부 보정 행렬)
        ▼
물체 위치 (엔드이펙터 프레임)
        │ T_ee_to_base    (순기구학으로 계산)
        ▼
물체 위치 (로봇 베이스 프레임)
        │ T_base_to_world (로봇 위치 캘리브레이션)
        ▼
물체 위치 (월드 프레임)
```

이 체인의 각 변환이 동차변환행렬로 표현되고, 최종 변환은 행렬의 곱입니다. 어느 한 단계의 오차가 최종 오차로 누적되므로, 각 캘리브레이션(카메라 외부 파라미터, 로봇 위치 측정)의 정확도가 중요합니다.

## 핵심 요약

- **좌표 프레임**: 위치/자세의 기준, 로봇 시스템에는 여러 프레임이 공존
- **동차변환행렬**: 4x4 행렬로 회전+이동을 통합, 곱으로 합성
- **오일러각**: 직관적이지만 짐벌 락 문제
- **회전 행렬**: 짐벌 락 없음, 9개 원소로 중복성 있음
- **쿼터니언**: 4개 원소, 짐벌 락 없음, 보간 우수, 실무에서 표준

다음 챕터에서는 이 변환 수학을 직접 활용하는 **순기구학과 역기구학** 을 다룹니다.
