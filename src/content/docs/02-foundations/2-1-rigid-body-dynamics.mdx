## 왜 물리학 기초를 배워야 하는가

로봇 시뮬레이터를 열면 로봇 팔이 물체를 집고, 보행 로봇이 계단을 오르는 것을 볼 수 있습니다. 이 모든 동작의 배경에는 **강체 동역학(rigid body dynamics)** 이 있습니다. 물리 엔진이 내부적으로 계산하는 바로 그 수식들입니다.

이 기초를 이해하면 두 가지 이점이 있습니다. 첫째, 시뮬레이터가 왜 특정 방식으로 작동하는지(또는 실패하는지) 이해할 수 있습니다. 둘째, 보상 함수 설계, 제어 알고리즘 선택, Sim-to-Real 갭 분석 같은 실용적 문제에서 올바른 판단을 내릴 수 있습니다.

## 강체(Rigid Body)란 무엇인가

**강체(rigid body)** 는 어떤 힘을 받더라도 내부 점들 사이의 거리가 변하지 않는 이상적인 물체입니다. 즉, 변형이 없는 물체입니다.

현실 세계의 모든 물체는 어느 정도 변형됩니다. 금속 빔은 하중에 의해 휘고, 고무는 늘어납니다. 그러나 대부분의 로봇 구조물과 조작 대상 물체는 **변형이 무시할 수 있을 정도로 작아**, 강체 가정이 충분히 유효합니다. 강체 가정 덕분에 복잡한 탄성 역학 없이도 물체의 운동을 예측할 수 있습니다.

### 6자유도(6-DOF)

3차원 공간에서 강체의 위치와 자세(orientation)를 완전히 기술하려면 **6개의 독립 변수**가 필요합니다.

```text
┌────────────────────────────────────────────────────────────┐
│            강체의 6 자유도 (6 Degrees of Freedom)           │
│                                                            │
│  병진 운동 (Translation):                                   │
│    • x 방향 이동                                            │
│    • y 방향 이동                                            │
│    • z 방향 이동                                            │
│                                                            │
│  회전 운동 (Rotation):                                      │
│    • x 축 회전 (roll, 롤)                                   │
│    • y 축 회전 (pitch, 피치)                                │
│    • z 축 회전 (yaw, 요)                                    │
│                                                            │
│  총 6개 = 3(병진) + 3(회전)                                 │
└────────────────────────────────────────────────────────────┘
```

로봇 팔의 각 링크(link)는 하나의 강체입니다. 6자유도 로봇 팔은 기저(base)에 고정된 채 6개의 관절을 통해 엔드이펙터를 3차원 공간의 임의 위치와 자세로 이동시킬 수 있습니다.

## 질량과 무게중심

**질량(mass, m)** 은 물체의 관성의 척도입니다. 질량이 클수록 동일한 힘에 의해 더 작은 가속도를 얻습니다(뉴턴 제2법칙: F = ma).

**무게중심(center of mass, CoM)** 은 물체의 질량 분포의 평균 위치입니다. 균일한 밀도의 구라면 기하학적 중심이 무게중심과 일치합니다. 복잡한 형상의 물체에서는 다를 수 있습니다.

로보틱스에서 무게중심이 중요한 이유는 **지지 다각형(support polygon)** 과의 관계 때문입니다. 보행 로봇이 안정적으로 서 있으려면 무게중심의 수직 투영이 발 접촉점들이 이루는 다각형 안에 있어야 합니다. 이 조건이 깨지면 로봇은 쓰러집니다.

## 관성 텐서(Inertia Tensor)

선형 운동에서 질량이 "힘에 대한 저항"을 나타내듯, 회전 운동에서는 **관성 텐서(inertia tensor)** 가 "토크에 대한 저항"을 나타냅니다.

관성 텐서는 3x3 대칭 행렬 I로 표현됩니다.

```text
        ┌                    ┐
        │  Ixx  -Ixy  -Ixz  │
    I = │ -Ixy   Iyy  -Iyz  │
        │ -Ixz  -Iyz   Izz  │
        └                    ┘

대각 원소 Ixx, Iyy, Izz: 각 축 중심의 관성 모멘트
비대각 원소 Ixy, Ixz, Iyz: 관성곱(products of inertia)
```

관성 모멘트(moment of inertia)는 물체의 질량이 회전축에서 멀리 분포할수록 큽니다. 피겨 스케이팅 선수가 팔을 오므리면 관성 모멘트가 줄어들고 회전 속도가 빨라지는 것이 직관적 예입니다.

물리 시뮬레이터는 각 물체의 질량, 무게중심, 관성 텐서를 저장하고, 이를 이용해 힘과 토크에 의한 운동을 계산합니다.

## 운동학(Kinematics) vs 동역학(Dynamics)

이 구분은 로보틱스 전반에서 반복적으로 등장하므로 명확히 이해해야 합니다.

```text
┌─────────────────────────────────────────────────────────────┐
│            운동학 vs 동역학 비교                              │
│                                                             │
│  운동학 (Kinematics)                                        │
│  ─────────────────                                          │
│  • 힘/토크를 고려하지 않음                                   │
│  • 위치, 속도, 가속도만 다룸                                 │
│  • 질문: "어디로 움직이는가?"                                │
│  • 예: 관절 각도 → 엔드이펙터 위치 (순기구학)               │
│         엔드이펙터 위치 → 관절 각도 (역기구학)              │
│                                                             │
│  동역학 (Dynamics)                                          │
│  ─────────────────                                          │
│  • 힘과 토크를 포함함                                        │
│  • 질량, 관성, 에너지를 다룸                                 │
│  • 질문: "왜 그렇게 움직이는가?"                             │
│  • 예: 관절 토크 → 관절 가속도 (순동역학)                    │
│         원하는 가속도 → 필요한 토크 (역동역학)               │
└─────────────────────────────────────────────────────────────┘
```

**운동학적 계획**: "로봇 팔 끝을 A 위치에서 B 위치로 이동시킬 경로를 계획한다." 이때 필요한 힘은 고려하지 않습니다.

**동역학적 계획**: "이 경로를 따라 이동하기 위해 각 관절에 어느 토크를 가해야 하는가?" 질량과 관성을 고려해 실제 모터 출력을 계산합니다.

고전적 로봇 제어는 두 단계를 순서대로 수행합니다. 먼저 운동학적 계획으로 경로를 결정하고, 그 경로를 추종하기 위한 제어 토크를 동역학적으로 계산합니다.

## 뉴턴-오일러 방정식

강체의 운동을 지배하는 기본 방정식입니다.

**뉴턴의 운동 방정식 (선형 운동)**:
```
F = m * a_com

F: 합력 벡터 (N)
m: 질량 (kg)
a_com: 무게중심의 가속도 벡터 (m/s²)
```

**오일러의 회전 방정식 (회전 운동)**:
```
τ = I * α + ω × (I * ω)

τ: 합 토크 벡터 (N·m)
I: 관성 텐서 (3×3 행렬)
α: 각가속도 벡터 (rad/s²)
ω: 각속도 벡터 (rad/s)
× : 벡터 외적(cross product)
```

오일러 방정식의 `ω × (I * ω)` 항은 **자이로스코픽 힘(gyroscopic effect)** 을 나타냅니다. 회전하는 팽이가 쓰러지지 않고 세차운동(precession)을 하는 것이 이 항 때문입니다. 빠르게 회전하는 드론 프로펠러에서도 이 효과가 비행 제어에 영향을 줍니다.

## 멀티바디 시스템 (로봇 팔)

로봇 팔은 여러 강체(링크)가 관절(joint)로 연결된 **멀티바디 시스템(multibody system)** 입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                 6자유도 로봇 팔 구조                           │
│                                                              │
│  Base (고정)                                                 │
│    │                                                         │
│    ├── 관절 1 (J1) ── 링크 1                                  │
│    │                    │                                    │
│    │               관절 2 (J2) ── 링크 2                     │
│    │                                 │                       │
│    │                            관절 3 (J3) ── 링크 3        │
│    │                                              │          │
│    │                                         관절 4~6 ── EE  │
│    │                                                         │
│    각 링크: 강체 (질량, 관성 텐서)                             │
│    각 관절: 운동 제약 (회전 또는 병진)                         │
└──────────────────────────────────────────────────────────────┘
```

멀티바디 시스템의 동역학을 기술하는 방법에는 여러 가지가 있습니다.

**뉴턴-오일러 재귀 알고리즘(Recursive Newton-Euler Algorithm)**: 각 링크에 대해 뉴턴-오일러 방정식을 순방향(기저→끝)으로 운동학을 계산하고, 역방향(끝→기저)으로 힘을 계산합니다. 계산 효율이 O(n)으로 링크 수에 선형 비례합니다.

**라그랑지안 역학(Lagrangian Mechanics)**: 시스템의 운동에너지와 퍼텐셜에너지 차이(라그랑지안)를 이용해 운동방정식을 유도합니다. 구속 조건을 자연스럽게 처리할 수 있는 장점이 있습니다.

## 물리 시뮬레이터에서의 구현

Isaac Sim의 PhysX, MuJoCo 등 물리 엔진은 이 방정식들을 수치적으로 적분합니다. 작은 시간 스텝(time step) Δt마다:

1. 현재 상태(위치, 속도)에서 힘과 토크 계산
2. 뉴턴-오일러 방정식으로 가속도 계산
3. 수치 적분으로 다음 상태 예측
4. 접촉/충돌 처리

사용하는 적분 방법(오일러, Runge-Kutta, 심플렉틱 오일러 등)에 따라 수치 안정성과 정확도가 달라집니다. 시간 스텝이 너무 크면 수치 불안정이 발생하고, 너무 작으면 계산 비용이 증가합니다.

개념 이해용 의사코드 (아래는 실제 라이브러리가 아닌 알고리즘 설명):

```python
# 개념 이해용 의사코드
def simulate_rigid_body_step(body, dt):
    # 현재 힘/토크 계산 (중력 + 접촉 + 제어)
    force = compute_total_force(body)
    torque = compute_total_torque(body)

    # 뉴턴 방정식: 선형 가속도
    linear_acc = force / body.mass

    # 오일러 방정식: 각가속도 (월드 프레임)
    gyroscopic = cross(body.omega, body.inertia @ body.omega)
    angular_acc = inv(body.inertia) @ (torque - gyroscopic)

    # 심플렉틱 오일러 적분
    body.linear_vel += linear_acc * dt
    body.angular_vel += angular_acc * dt
    body.position += body.linear_vel * dt
    body.orientation = integrate_rotation(body.orientation,
                                           body.angular_vel, dt)
    return body
```

## 핵심 요약

강체 동역학의 핵심 개념을 정리하면:

- **강체**: 변형 없는 이상 물체, 6자유도로 기술
- **질량/무게중심**: 선형 운동의 관성
- **관성 텐서**: 회전 운동의 관성, 3x3 행렬
- **운동학**: 힘 없이 위치/속도/가속도만
- **동역학**: 힘/토크 포함, 뉴턴-오일러 방정식
- **멀티바디**: 로봇 = 강체들의 연결, 재귀 알고리즘으로 효율적 계산

다음 챕터에서는 이 강체들의 위치와 자세를 수학적으로 표현하는 방법, 즉 좌표계와 변환 행렬을 다룹니다.
