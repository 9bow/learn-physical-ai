## 왜 아직도 어려운가

Physical AI는 지난 10년간 눈부신 발전을 이루었습니다. 시뮬레이터에서 보행 로봇이 장애물을 넘고, 로봇 팔이 복잡한 조립 작업을 학습합니다. 그런데 왜 공장이나 일반 가정에 완전 자율 로봇이 보편화되지 않을까요?

이 챕터는 Physical AI가 여전히 직면한 핵심 난제들을 정직하게 살펴봅니다. 과장도, 과소평가도 없이, 연구 현장의 실질적 어려움을 이해하는 것이 목표입니다.

## 난제 1: Sim-to-Real 갭

**Sim-to-Real 갭(Reality Gap)** 은 시뮬레이터에서 학습된 정책이 실제 로봇에서 성능이 저하되는 현상입니다. 이는 Physical AI에서 가장 널리 연구되는 문제 중 하나입니다.

### 갭의 원인

```text
┌──────────────────────────────────────────────────────────────┐
│              Sim-to-Real 갭의 원인 분류                       │
├───────────────────────┬──────────────────────────────────────┤
│ 범주                  │ 구체적 예시                           │
├───────────────────────┼──────────────────────────────────────┤
│ 물리 모델 오차         │ 마찰 계수, 관성, 유연성 근사         │
│ 렌더링/시각 차이       │ 조명, 텍스처, 반사, 그림자           │
│ 센서 모델 오차         │ 카메라 왜곡, 노이즈 특성, LiDAR 반사 │
│ 접촉 모델 오차         │ 변형, 미끄러짐, 파지 안정성          │
│ 제어 지연              │ 통신 지연, 모터 응답 특성            │
│ 물체 특성 오차         │ 질량, 무게중심, 표면 마찰            │
└───────────────────────┴──────────────────────────────────────┘
```

가장 까다로운 것은 **접촉(contact) 모델**입니다. 로봇 손가락이 물체를 집을 때 발생하는 복잡한 접촉 역학은 현재의 물리 시뮬레이터도 완벽히 재현하기 어렵습니다. 시뮬레이터에서 완벽히 작동하는 파지 정책이 실제 로봇에서 미끄러지거나 떨어뜨리는 일이 흔합니다.

### 완화 기법

**도메인 랜덤화(Domain Randomization)**: 훈련 중 물리 파라미터(마찰, 질량, 조명, 텍스처)를 무작위로 변화시킵니다. 다양한 조건에서 훈련된 정책은 실제 환경의 "알 수 없는" 파라미터에 더 강인합니다. OpenAI의 Dactyl 손가락 로봇이 이 기법을 극단적으로 적용하여 루빅스 큐브를 맞추는 데 성공했습니다.

**도메인 적응(Domain Adaptation)**: 소량의 실제 데이터를 이용해 시뮬레이션에서 학습된 표현을 실제 도메인으로 적응시킵니다. 픽셀 수준 이미지를 도메인 무관 특징으로 변환하는 접근입니다.

**시스템 식별(System Identification)**: 실제 로봇의 물리 파라미터를 측정하고, 시뮬레이터를 실제에 가깝게 교정합니다. 자동화된 방법으로 로봇을 작동시키며 관찰된 행동과 시뮬레이션을 비교해 파라미터를 추정합니다.

**실세계 미세 조정(Real-world Fine-tuning)**: 시뮬레이션에서 사전 훈련하고, 소량의 실세계 데이터로 추가 학습합니다. 이 접근은 실세계 데이터 수집을 줄이면서도 시뮬레이션의 한계를 보정합니다.

## 난제 2: 데이터 수집의 병목

**"데이터 병목(data bottleneck)"** 은 Physical AI에서 반복적으로 등장하는 주제입니다.

생성형 AI는 인터넷이라는 무한한 데이터 소스를 활용합니다. GPT-4는 수조 토큰의 텍스트를 학습했습니다. 반면 로봇 조작 데이터를 수집하려면:

1. 물리 로봇을 준비하고 (비용: 수천~수십만 달러)
2. 숙련된 조작자가 원격 조종(teleoperation)으로 시연하고 (시간: 에피소드당 수분)
3. 데이터를 정리하고 라벨링한다 (추가 인력)

이 과정을 통해 수집하는 데이터는 하루에 수백 에피소드입니다. 언어 모델 규모의 학습에 필요한 데이터와는 비교할 수 없이 작습니다.

### 대안적 접근

```text
┌────────────────────────────────────────────────────────────┐
│              데이터 병목 완화 전략                           │
│                                                            │
│  ① 합성 데이터                                             │
│     시뮬레이터 + 물리 기반 생성 → 무한한 에피소드           │
│                                                            │
│  ② 크라우드소싱 수집                                        │
│     Stanford ALOHA, UMI (Universal Manipulation Interface) │
│     → 저비용 원격조종 하드웨어로 대규모 수집                │
│                                                            │
│  ③ 인터넷 비디오 활용                                       │
│     인간 시연 영상 → 의사행동(pseudo-action) 학습           │
│     (VPT: Video Pre-Training, OpenAI)                      │
│                                                            │
│  ④ 자율 데이터 수집                                         │
│     로봇이 스스로 탐색 → 다양한 경험 자동 수집              │
└────────────────────────────────────────────────────────────┘
```

ALOHA(A Low-cost Open-source Hardware System for Bimanual Dexterous Manipulation) 는 Stanford에서 개발한 저비용 양팔 원격조종 시스템으로, 데이터 수집 비용을 크게 낮췄습니다. LeRobot 등 오픈소스 프로젝트들이 이 방향을 계속 발전시키고 있습니다.

## 난제 3: 안전성과 신뢰성

**기능 안전(functional safety)** 은 시스템이 오작동할 때 안전한 상태로 유지되도록 설계하는 것을 의미합니다. 소프트웨어 버그, 센서 오류, 예상치 못한 환경 조건에서도 인명 피해가 발생하지 않아야 합니다.

딥러닝 기반 정책의 안전성을 보장하는 것은 현재 미해결 문제입니다.

**문제 1: 분포 이탈과 실패 모드**
신경망은 훈련 분포 내에서는 잘 작동하지만, 분포 밖(out-of-distribution)에서는 예측 불가능하게 실패할 수 있습니다. "이 물체는 훈련에 없었던 형태"임을 모델이 인식하지 못하고 잘못된 행동을 자신 있게 실행할 수 있습니다.

**문제 2: 불확실성 정량화**
고전적 제어 시스템은 잘 정의된 오차 범위 안에서 작동합니다. 신경망 정책은 자신의 불확실성을 정량화하기 어렵습니다. "이 상황에서 내 결정이 얼마나 신뢰할 수 있는가"를 답하기 어렵습니다.

**현재 접근법**:
- **안전 레이어(safety layer)**: 정책 출력을 물리적 한계(속도, 힘, 관절 범위) 내로 클리핑
- **배리어 함수(Control Barrier Function)**: 안전 제약을 수학적으로 보장하는 제어 이론 기법
- **점진적 배포**: 제어된 환경에서 점차 복잡한 환경으로 확장, 각 단계에서 안전성 검증
- **인간 감독(human-in-the-loop)**: 불확실성이 높을 때 인간 확인 요청

## 난제 4: 일반화

**일반화(generalization)** 는 학습된 정책이 훈련에 없었던 새로운 상황에서도 작동하는 능력입니다.

```text
┌──────────────────────────────────────────────────────────────┐
│              일반화의 여러 차원                               │
│                                                              │
│  객체 수준   : 훈련에 없던 색상/형태/재질의 물체 처리         │
│  환경 수준   : 새로운 방, 새로운 조명, 새로운 배치            │
│  태스크 수준 : 새로운 조합의 하위 태스크 처리                │
│  로봇 수준   : 다른 로봇 플랫폼(embodiment transfer)         │
│  언어 수준   : 새로운 형식의 자연어 명령 이해                │
└──────────────────────────────────────────────────────────────┘
```

현재 최고 수준의 모델들도 **훈련 분포에 민감**합니다. "파란 컵을 집어라"로 훈련된 모델이 "파란 머그잔을 집어라"에서 실패하는 경우가 있습니다. 범용 파운데이션 모델(GR00T, RT-2)은 이 일반화 문제를 대규모 데이터로 완화하려 하지만, 완전히 해결된 상태는 아닙니다.

## 난제 5: 긴 수평선 태스크(Long-horizon Task)

단순한 작업(단일 물체 집기, 짧은 경로 이동)에서 복잡한 작업(여러 부품을 조립하여 완성품 만들기, 부엌에서 요리 준비)으로 갈수록 **긴 수평선 태스크** 문제가 중요해집니다.

20단계로 구성된 조립 작업을 생각해봅시다. 각 단계의 성공률이 95%라면, 전체 성공률은 0.95^20 ≈ 0.36입니다. 즉, 긴 태스크에서는 각 단계의 소폭 실패도 전체 실패로 이어집니다.

또한 초기 단계의 오류가 이후 단계에서 복구 불가능한 상태를 만들 수 있습니다. "나사를 잘못된 위치에 조인 것"은 이후 과정에서 인식이 어렵고, 자동 복구는 더욱 어렵습니다.

**현재 접근**:
- **계층적 계획(hierarchical planning)**: 고수준 언어 모델이 하위 태스크를 분해하고, 각 하위 태스크는 전문화된 정책이 담당
- **오류 복구 정책(error recovery policy)**: 실패 상태를 인식하고 복구하는 별도 정책 훈련
- **태스크와 모션 계획(TAMP: Task and Motion Planning)**: 논리적 태스크 구조와 물리 동작 계획을 통합

## 난제들의 상호작용

이 난제들은 독립적이지 않습니다.

```text
┌─────────────────────────────────────────────────────────────┐
│              난제 간 상호작용                                 │
│                                                             │
│  Sim-to-Real 갭 ◀──── 데이터 병목                           │
│       │                  │                                  │
│       ▼                  ▼                                  │
│  일반화 실패 ─────▶ 안전성 저하                              │
│       │                  │                                  │
│       └──────────────────┘                                  │
│                  ▼                                          │
│         긴 수평선 태스크 실패                                │
└─────────────────────────────────────────────────────────────┘
```

데이터가 부족하면 Sim-to-Real 의존도가 높아지고, Sim-to-Real 갭이 크면 실제 환경에서의 일반화가 어렵습니다. 일반화가 어려우면 예상치 못한 상황에서 안전성이 저하되고, 긴 태스크에서는 이 모든 문제가 누적됩니다.

Physical AI 연구의 진보는 이 난제들을 독립적으로 해결하는 것이 아니라, **상호 보완적인 접근을 통해 함께 완화**하는 방향으로 이루어지고 있습니다. 대규모 합성 데이터로 사전훈련하고, 실세계 미세조정으로 Sim-to-Real 갭을 줄이고, 안전 레이어로 실패를 방지하는 통합 파이프라인이 점점 표준이 되어가고 있습니다.

다음 챕터에서는 이 과제들을 해결하기 위한 현재의 도구와 생태계 전체 지형을 살펴봅니다.
