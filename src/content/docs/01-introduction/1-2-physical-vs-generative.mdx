## 생성형 AI의 등장과 혼동

ChatGPT, Midjourney, Stable Diffusion의 등장 이후 "AI"라는 단어는 대중에게 거의 동의어처럼 쓰입니다. 하지만 이 시스템들은 **생성형 AI(Generative AI)** 라는 특정 범주에 속하며, Physical AI와는 근본적으로 작동 방식이 다릅니다. 두 유형을 명확히 구분하지 않으면, 왜 GPT-4가 로봇을 직접 제어할 수 없는지, 왜 Physical AI 연구에 별도의 시뮬레이터와 하드웨어가 필요한지 이해하기 어렵습니다.

## 생성형 AI: 분포에서 샘플링하기

생성형 AI의 핵심 아이디어는 **데이터의 분포를 학습**하고, 그 분포에서 새로운 샘플을 생성하는 것입니다. 대규모 언어 모델(LLM)은 수천억 개의 텍스트 토큰을 학습하여 다음 토큰의 확률 분포를 예측합니다. 이미지 생성 모델은 수억 장의 이미지-텍스트 쌍을 학습하여, 텍스트 프롬프트에 조건화된 이미지 분포에서 샘플을 추출합니다.

이 과정의 특징은 다음과 같습니다.

- **비동기(asynchronous)**: 사용자가 프롬프트를 보내고 몇 초 또는 몇 분 후에 응답이 옵니다. 실시간성이 요구되지 않습니다.
- **단방향(open-loop)**: 모델의 출력이 다시 입력으로 피드백되지 않습니다. 생성된 텍스트가 "잘못되었는지" 모델 스스로 알 수 없습니다.
- **물리 무관(physics-agnostic)**: 텍스트는 중력의 영향을 받지 않습니다. "컵이 테이블에서 떨어진다"는 문장을 생성할 때 모델은 실제 물리 시뮬레이션을 수행하지 않습니다.

```text
┌──────────────────────────────────────────────────────────────┐
│                 생성형 AI 작동 방식 (개루프)                  │
│                                                              │
│  사용자 프롬프트 ──▶ [LLM / Diffusion Model] ──▶ 결과 출력  │
│                              │                              │
│                    (피드백 루프 없음)                         │
│                    출력이 환경에 영향 안 줌                   │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│                Physical AI 작동 방식 (폐루프)                 │
│                                                              │
│  센서 ──▶ [정책 모델] ──▶ 액추에이터 명령                    │
│   ▲                              │                          │
│   └──────── 환경 변화 피드백 ◀───┘                          │
│                                                              │
│        루프가 초당 수십~수백 회 반복됨                        │
└──────────────────────────────────────────────────────────────┘
```

## 핵심 차이 1: 실시간성

생성형 AI에서 응답 속도는 품질 문제입니다. 응답이 3초 걸리든 10초 걸리든, 결과의 "정답 여부"는 응답 시간과 무관합니다.

Physical AI에서 응답 속도는 **안전과 기능의 문제**입니다. 보행 로봇의 균형 제어 알고리즘이 1초 지연되면 로봇은 쓰러집니다. 자율주행 차량이 앞차의 급정거를 감지하고 200ms 안에 제동을 시작하지 못하면 추돌합니다. 이를 **하드 실시간(hard real-time)** 요구라고 합니다.

이 차이는 하드웨어 선택에도 영향을 미칩니다. 생성형 AI는 대규모 데이터센터 GPU에서 배치(batch) 처리로 실행됩니다. Physical AI 추론은 경량 엣지 GPU(예: NVIDIA Jetson)에서 저지연(low-latency)으로 실행되어야 합니다.

## 핵심 차이 2: 물리 제약과 안전

생성형 AI가 "틀린" 출력을 낼 때의 비용은 대체로 낮습니다. 환각(hallucination)된 사실을 포함한 문단, 해부학적으로 부정확한 손가락을 가진 이미지는 품질 문제이지 안전 문제가 아닙니다.

Physical AI의 오류는 **물리 세계에서 돌이킬 수 없는 결과**를 낳을 수 있습니다.

- 의료 로봇의 위치 오차 → 환자 부상
- 자율주행 오판단 → 교통사고
- 산업 로봇 제어 실패 → 작업자 안전 위협

이로 인해 Physical AI 시스템에는 훨씬 엄격한 **검증(verification)** 과 **안전 케이스(safety case)** 가 요구됩니다. ISO 26262(자동차 기능 안전), IEC 62061(기계 안전) 같은 산업 표준이 적용됩니다.

## 핵심 차이 3: 데이터의 특성

```text
┌─────────────────────────────────────────────────────────────┐
│               학습 데이터 비교                               │
├──────────────────┬──────────────────────────────────────────┤
│ 생성형 AI        │ Physical AI                              │
├──────────────────┼──────────────────────────────────────────┤
│ 인터넷 텍스트    │ 로봇 시연(demonstration)                  │
│ 이미지-텍스트쌍  │ 센서 로그(LiDAR, IMU, RGB-D)             │
│ 수십억~수조 토큰 │ 수천~수십만 에피소드                      │
│ 자동 수집 가능   │ 물리 로봇 필요, 수동 수집                 │
│ 정적 데이터셋    │ 상호작용 통한 온라인 학습                 │
│ 라벨링 비교적 쉬움│ 시연자 전문성 필요                       │
└──────────────────┴──────────────────────────────────────────┘
```

생성형 AI의 성공은 인터넷에 이미 존재하는 방대한 텍스트와 이미지 덕분입니다. "스케일링 법칙(scaling laws)"이 적용되어, 더 많은 데이터와 더 큰 모델이 일관되게 더 나은 성능을 보입니다.

Physical AI의 데이터는 **수집 자체가 연구 과제**입니다. 물리 로봇을 운영하며 사람이 시연(teleop, kinesthetic teaching)을 수행해야 하고, 각 에피소드는 수 분에서 수십 분이 걸립니다. Open X-Embodiment 데이터셋은 이 문제를 공동 데이터 수집으로 완화하려는 시도입니다.

## 핵심 차이 4: 일반화의 방향

생성형 AI의 일반화 문제는 주로 **도메인 이동(domain shift)** 입니다. 영어 텍스트로 훈련된 모델이 한국어에서 성능이 저하되거나, 사진으로 훈련된 이미지 분류기가 스케치에서 실패하는 식입니다.

Physical AI의 일반화 문제는 훨씬 복잡합니다.

- **Sim-to-Real 갭**: 시뮬레이터의 물리 모델이 현실과 다릅니다. 마찰 계수, 조명, 물체 형태의 미세한 차이가 정책을 실패하게 만듭니다.
- **분포 이탈(distribution shift)**: 훈련 시 보지 못한 물체 모양, 새로운 작업 환경에서 성능이 급격히 저하됩니다.
- **긴 수평선(long-horizon) 실패**: 10단계로 이루어진 조립 작업에서 7단계까지 성공했더라도 8단계에서 실패하면 전체가 무너집니다.

## 그렇다면 두 AI는 서로 무관한가?

전혀 그렇지 않습니다. 최근 연구의 방향은 **생성형 AI의 언어/시각 이해 능력을 Physical AI에 통합**하는 것입니다.

**언어 조건화 정책(language-conditioned policy)**: "빨간 컵을 집어서 오른쪽 선반에 놓아라"와 같은 자연어 명령을 로봇 동작으로 변환합니다. OpenVLA, RT-2 같은 모델이 이 방향을 개척했습니다.

**시각-언어-행동 모델(VLA: Vision-Language-Action)**: 비전 인코더, 언어 모델, 행동 디코더를 하나의 모델로 통합합니다. NVIDIA GR00T N1이 이 아키텍처를 채택합니다.

**합성 데이터 생성**: 생성형 AI 기술(확산 모델, NeRF)을 이용해 로봇 훈련 데이터를 합성합니다. 현실적인 씬(scene)을 대량 생성하여 Physical AI의 데이터 부족 문제를 완화합니다.

```text
┌─────────────────────────────────────────────────────────────┐
│         생성형 AI + Physical AI 융합 아키텍처               │
│                                                             │
│  자연어 명령 ──▶ [언어 모델(LLM)]                           │
│                        │                                   │
│                        ▼ 고수준 계획                        │
│  카메라 이미지 ──▶ [시각 인코더] ──▶ [행동 정책] ──▶ 로봇   │
│                                          │                  │
│                        ◀── 센서 피드백 ──┘                  │
└─────────────────────────────────────────────────────────────┘
```

이처럼 생성형 AI는 Physical AI의 "두뇌" 역할, 즉 고수준의 계획과 언어 이해를 담당하고, Physical AI의 저수준 제어 루프는 실시간 센서 피드백에 반응하는 구조로 발전하고 있습니다.

## 정리: 무엇이 본질적으로 다른가

| 기준 | 생성형 AI | Physical AI |
|------|-----------|-------------|
| 피드백 루프 | 개루프(open-loop) | 폐루프(closed-loop) |
| 시간 제약 | 없음 | 하드 실시간 |
| 실패 비용 | 낮음(품질 문제) | 높음(안전 문제) |
| 데이터 규모 | 수십억 샘플 | 수천~수만 에피소드 |
| 핵심 과제 | 환각, 정렬 | Sim-to-Real, 안전 |

두 AI는 경쟁 관계가 아니라 **상호 보완적**입니다. 생성형 AI의 발전이 Physical AI를 가속화하고, Physical AI에서 수집된 실세계 상호작용 데이터가 생성형 AI를 더 현실 인식적으로 만들 것입니다. 다음 챕터에서는 NVIDIA가 이 두 영역을 어떻게 하나의 통합 플랫폼으로 연결하는지 살펴봅니다.
