## 로보틱스의 두 세기

로봇이 처음 공장에 등장한 1960년대부터 오늘날의 학습 기반 로보틱스까지, 제어 패러다임의 역사는 "정해진 규칙에서 학습된 규칙으로"의 전환으로 요약할 수 있습니다. 이 전환은 단순한 기술 변화가 아니라, 로봇이 해결할 수 있는 문제의 근본적 확장을 의미합니다.

## 1세대: 프로그래밍 기반 자동화 (1960~1990년대)

초기 산업 로봇은 **위치 제어(position control)** 방식으로 작동했습니다. 엔지니어가 로봇 팔의 각 관절이 특정 시간에 도달해야 할 각도를 직접 프로그래밍합니다. 용접 로봇은 미리 입력된 수십~수백 개의 위치 좌표를 순서대로 반복합니다.

이 방식의 장점은 명확합니다. 예측 가능하고, 반복 정밀도가 높으며, 검증이 쉽습니다. 자동차 조립 라인처럼 **환경이 고도로 통제되고 반복적인** 작업에서 수십 년간 높은 신뢰성을 보였습니다.

한계도 명확합니다. 물체 위치가 조금만 달라지면, 또는 예상치 못한 장애물이 나타나면, 로봇은 실패합니다. 적응(adaptation)이 없습니다.

## 2세대: Sense-Plan-Act 패러다임 (1980~2000년대)

로봇에 센서를 장착하고 환경을 인식한 뒤 계획을 세우는 **Sense-Plan-Act(SPA)** 구조가 등장했습니다.

```text
┌──────────────────────────────────────────────────────┐
│            Sense-Plan-Act 파이프라인                  │
│                                                      │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐           │
│  │  Sense  │───▶│  Plan   │───▶│   Act   │           │
│  │         │    │         │    │         │           │
│  │ 센서 데이터│   │ 경로 계획│   │ 명령 실행│          │
│  │ 환경 인식│    │ 동작 순서│   │ 모터 제어│          │
│  └─────────┘    └─────────┘    └─────────┘           │
│                      │                               │
│              각 단계가 독립 모듈                       │
│         새 환경 = 각 모듈 재설계 필요                  │
└──────────────────────────────────────────────────────┘
```

이 시기에 중요한 기술들이 발전했습니다.

**역기구학(Inverse Kinematics)**: "엔드이펙터(end-effector, 로봇 팔 끝)를 이 위치에 놓아라"는 명령을 각 관절의 각도로 변환하는 수학적 방법입니다. 해석적 해가 없는 경우 Jacobian 기반 반복법을 사용합니다.

**경로 계획(Path Planning)**: A*, RRT(Rapidly-exploring Random Tree) 같은 알고리즘이 장애물을 피하는 경로를 계획합니다. 3D 공간에서 로봇 팔의 관절 공간을 탐색하는 것은 고차원 최적화 문제입니다.

**모델 예측 제어(MPC: Model Predictive Control)**: 미래 수 스텝의 시스템 상태를 예측하면서 최적 제어 입력을 실시간으로 계산합니다. 물리 모델이 정확할수록 성능이 좋습니다.

SPA는 구조화된 환경에서 인상적인 성과를 냈습니다. 하지만 근본적 한계가 있었습니다. **각 모듈의 오류가 누적**됩니다. 센서 노이즈 → 부정확한 환경 지도 → 잘못된 계획 → 실행 실패. 또한 환경이 달라지면 각 모듈을 처음부터 재설계해야 합니다.

## 3세대: 머신러닝의 도입 (2000년대~2010년대)

딥러닝 혁명이 로보틱스에 본격적으로 영향을 미치기 시작합니다.

### 행동 복제(Behavioral Cloning)

인간 전문가의 시연(demonstration)을 직접 학습하는 방식입니다. 카메라 이미지 → 행동 벡터의 매핑을 지도학습(supervised learning)으로 학습합니다. 자율주행 분야에서 Dave Pomerleau의 ALVINN(1989)이 선구적 연구였으며, 현대적 형태는 DAgger(Dataset Aggregation, 2011) 알고리즘으로 발전했습니다.

행동 복제의 치명적 문제는 **분포 이탈(distribution shift)** 입니다. 훈련 데이터의 분포(전문가 궤적)에서 조금만 벗어나면 오류가 급격히 누적됩니다. 전문가가 절대 경험하지 않는 상황(실수 후 복구)에 대한 데이터가 없기 때문입니다.

### 심층 강화학습(Deep RL)의 등장

2013년 DeepMind의 DQN이 Atari 게임에서 인간 수준의 성능을 달성한 이후, 강화학습이 로보틱스에 적극 적용되기 시작했습니다.

강화학습의 기본 구조는 다음과 같습니다.

```text
┌─────────────────────────────────────────────────────────┐
│                 강화학습 구조                             │
│                                                         │
│  ┌─────────┐  행동(action)  ┌─────────┐                 │
│  │  에이전트│──────────────▶│  환경   │                 │
│  │ (정책 π) │               │(로봇+세계│                 │
│  │         │◀──────────────│         │                 │
│  └─────────┘  보상(reward)  └─────────┘                 │
│               + 다음 상태(obs)                           │
│                                                         │
│  목표: 누적 보상을 최대화하는 정책 π 학습                  │
└─────────────────────────────────────────────────────────┘
```

시뮬레이터에서의 강화학습은 사람의 시연 없이도 **수백만 번의 시행착오**를 통해 정책을 학습할 수 있다는 장점이 있습니다. OpenAI Five(Dota 2), AlphaStar(StarCraft), 그리고 Boston Dynamics 보행 로봇이 이 접근법의 성과를 보여주었습니다.

그러나 문제도 있습니다.

- **보상 설계(reward engineering)**: "작업을 잘 수행했을 때 주는 보상"을 수치로 정의하는 것이 어렵습니다. 잘못 설계된 보상은 의도치 않은 행동(reward hacking)을 유발합니다.
- **샘플 비효율성**: 물리 로봇에서 실행하면 시간과 비용이 너무 많이 듭니다. 시뮬레이션에서 훈련하면 Sim-to-Real 갭이 문제가 됩니다.

## 4세대: 대규모 학습 기반 접근 (2020년대~현재)

언어 모델에서 증명된 **스케일링(scaling)** 의 힘이 로보틱스에도 적용되기 시작했습니다.

### 모방학습의 재발견: 확산 정책(Diffusion Policy)

Chi et al.의 Diffusion Policy(2023)는 확산 모델(diffusion model)을 행동 생성에 적용합니다. 다중 모달(multimodal) 행동 분포를 더 잘 포착하여, 전통적 행동 복제보다 훨씬 강인한 정책을 학습합니다.

### 대규모 로봇 데이터: Open X-Embodiment

Google과 여러 기관이 협력하여 공개한 Open X-Embodiment 데이터셋(2023)은 22개 로봇 플랫폼, 527가지 기술, 수십만 개의 실세계 에피소드를 포함합니다. 이 데이터로 훈련된 RT-X 모델은 다양한 로봇에서 새로운 태스크를 수행할 수 있었습니다.

### 비전-언어-행동 모델(VLA)

생성형 AI 기술과 로보틱스의 융합이 본격화되었습니다.

| 모델 | 기관 | 특징 |
|------|------|------|
| RT-2 | Google DeepMind | 웹 데이터로 사전훈련된 VLM + 로봇 행동 |
| OpenVLA | Stanford+Berkeley | 오픈소스 VLA 기반 모델 |
| GR00T N1 | NVIDIA | 휴머노이드 전용 기반 모델 |
| π0 (Pi Zero) | Physical Intelligence | 확산 기반 범용 조작 모델 |

### 엔드-투-엔드 학습(End-to-End Learning)

```text
┌─────────────────────────────────────────────────────────────┐
│         전통 파이프라인 vs 엔드-투-엔드 학습                  │
│                                                             │
│ 전통 방식:                                                   │
│ 카메라 → [인식 모듈] → [계획 모듈] → [제어 모듈] → 모터      │
│          (각각 별도 설계 및 튜닝)                             │
│                                                             │
│ 엔드-투-엔드:                                                │
│ 카메라 ──────────────▶ [단일 신경망] ──────────────▶ 모터    │
│ (픽셀)                  (정책 모델)                (토크)     │
│                                                             │
│ 장점: 중간 표현에 대한 인간의 편향 제거                        │
│ 단점: 해석 가능성 감소, 디버깅 어려움                         │
└─────────────────────────────────────────────────────────────┘
```

엔드-투-엔드 학습은 센서 입력에서 모터 명령까지 하나의 신경망이 전체 파이프라인을 처리합니다. 중간 표현(인식 결과, 경로 계획)을 인간이 설계하지 않아도 됩니다. Tesla의 FSD(Full Self-Driving) 시스템이 이 방향으로 전환한 것이 대표적 사례입니다.

## 고전과 학습 기반의 공존

현재 산업 현장에서는 **하이브리드 접근**이 주류입니다. 고수준 계획은 학습 기반 모델이, 저수준 실시간 제어는 고전적 MPC나 PID 제어기가 담당합니다. 두 방식의 장점을 결합하는 것이 현실적입니다.

예를 들어, 로봇 팔이 물체를 집는 태스크에서:
- **시각 기반 파지 계획(grasp planning)**: 학습된 CNN이 적절한 파지 위치와 자세를 예측
- **역기구학 + MPC**: 예측된 목표 자세까지 관절을 안전하게 이동
- **힘/토크 제어**: 접촉 후 섬세한 힘 조절

각 단계가 가장 잘 풀 수 있는 도구를 씁니다.

## 이 역사가 주는 교훈

로보틱스 패러다임의 전환을 보면 일관된 패턴이 있습니다. **규칙 기반 → 통계적 학습 → 대규모 데이터 + 딥러닝**. 각 전환은 이전 방식의 한계를 일부 극복하지만 새로운 도전을 가져왔습니다. 오늘날의 학습 기반 로보틱스도 마찬가지입니다. 일반화, 안전성, 해석 가능성이라는 해결되지 않은 과제가 남아 있으며, 이것이 다음 패러다임 전환의 씨앗이 될 것입니다.

다음 챕터에서는 현재 Physical AI 연구의 가장 어려운 핵심 난제들을 구체적으로 살펴봅니다.
