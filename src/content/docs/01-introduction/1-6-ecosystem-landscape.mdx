## 생태계의 구조

Physical AI 생태계는 크게 네 레이어로 구성됩니다. **시뮬레이터** 레이어가 가상 환경을 제공하고, **훈련 프레임워크** 레이어가 정책 학습을 담당하며, **데이터/모델** 레이어가 사전훈련된 지식을 제공하고, **하드웨어** 레이어에서 실제 로봇이 작동합니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│                 Physical AI 생태계 레이어                        │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  데이터 / 기반 모델                                      │    │
│  │  GR00T, OpenVLA, LeRobot, Open X-Embodiment(OXE)        │    │
│  └─────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  훈련 프레임워크                                          │    │
│  │  Isaac Lab, Stable Baselines3, RLlib, Tianshou          │    │
│  └─────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  시뮬레이터                                               │    │
│  │  Isaac Sim, MuJoCo, Genesis, Gazebo, PyBullet           │    │
│  └─────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  하드웨어 / 미들웨어                                      │    │
│  │  Jetson AGX, ROS2, Isaac ROS, DRIVE AGX                 │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

## 시뮬레이터 비교

물리 시뮬레이터는 Physical AI 개발의 핵심 인프라입니다. 선택은 사용 목적, 필요한 물리 정확도, GPU 병렬화 지원 여부에 따라 달라집니다.

| 시뮬레이터 | 개발사 | 물리 엔진 | GPU 병렬화 | 특징 | 주요 용도 |
|----------|-------|---------|-----------|------|---------|
| Isaac Sim | NVIDIA | PhysX 5 | O (수천 환경) | Omniverse 통합, 포토리얼 렌더링 | 산업 로봇, 휴머노이드 |
| MuJoCo | DeepMind | 자체 | O (MJX) | 접촉 정확도, RL 벤치마크 표준 | 연구, 보행, 조작 |
| Genesis | MIT 등 | 자체 (GPU native) | O (매우 빠름) | 완전 GPU, 생성형 통합 | 빠른 실험, 연구 |
| Gazebo/Ignition | Open Robotics | ODE/Bullet | 제한적 | ROS 통합 표준 | ROS 개발, 프로토타이핑 |
| PyBullet | Erwin Coumans | Bullet | 제한적 | 가볍고 설치 간단 | 교육, 빠른 프로토타이핑 |
| Webots | Cyberbotics | ODE | X | 오픈소스 완성도 높음 | 교육, 소형 로봇 |

**Isaac Sim** 은 이 코스의 주요 시뮬레이터입니다. NVIDIA PhysX 5를 기반으로 GPU 가속 병렬 시뮬레이션을 지원하며, 포토리얼리스틱 렌더링으로 합성 센서 데이터 품질이 높습니다.

**MuJoCo(MuJoCo Physics)** 는 로보틱스 RL 연구의 사실상 표준 벤치마크 환경을 제공합니다. Ant, Humanoid, Hopper 같은 표준 환경이 수백 편의 논문에서 사용되어 결과 비교가 용이합니다.

**Genesis** 는 2024년 말 공개된 신규 시뮬레이터로, 완전 GPU 네이티브 설계로 매우 빠른 시뮬레이션 속도를 자랑합니다. 아직 생태계는 성숙하지 않았지만 주목할 만한 프로젝트입니다.

## 강화학습 훈련 프레임워크

| 프레임워크 | 언어 | 특징 | 적합 규모 |
|-----------|------|------|---------|
| Isaac Lab | Python | Isaac Sim 통합, GPU 병렬 RL | 대규모, 산업 |
| Stable Baselines3 (SB3) | Python | 간단한 API, 교육용 | 소~중규모 |
| RLlib | Python | 분산 RL, 유연한 커스텀 | 중~대규모 |
| CleanRL | Python | 단일 파일 구현, 가독성 | 연구, 학습 |
| RL Games | Python | Isaac과 통합, GPU 가속 | 대규모 |

**Isaac Lab** 은 이 코스에서 다루는 주요 프레임워크입니다. GPU 병렬 환경에서 수천 개의 로봇을 동시에 훈련할 수 있어, 몇 시간 안에 보행이나 조작 정책을 학습할 수 있습니다.

## 데이터셋과 기반 모델

### 주요 데이터셋

| 데이터셋 | 규모 | 특징 |
|---------|------|------|
| Open X-Embodiment | 22개 로봇, 527가지 기술 | 다중 로봇 플랫폼 데이터 |
| DROID | ~76,000 에피소드 | 대규모 단일 로봇 데이터 |
| BridgeData V2 | ~60,000 데모 | 저비용 로봇 조작 |
| RoboSet | 다양한 씬 | 구조화된 조작 태스크 |

### 주요 기반 모델

**GR00T(NVIDIA)**: 휴머노이드 로봇을 위한 기반 모델. 다중 모달 입력(비전, 언어, 고유감각)을 처리하고, 다양한 조작 태스크로 파인튜닝 가능합니다.

**OpenVLA(Stanford+Berkeley)**: 오픈소스 비전-언어-행동 모델. 7B 파라미터 Llama 기반으로, 자연어 명령과 카메라 이미지를 입력받아 로봇 행동을 출력합니다.

**LeRobot(Hugging Face)**: Hugging Face가 개발한 로봇 학습 라이브러리. ACT, Diffusion Policy 등 최신 모방학습 알고리즘을 표준화된 인터페이스로 제공합니다.

## 하드웨어와 미들웨어

### 엣지 컴퓨팅 플랫폼

| 플랫폼 | AI 성능 | 전력 | 사용 사례 |
|-------|---------|------|---------|
| Jetson Orin Nano | ~40 TOPS | 5~15W | 소형 로봇, 드론 |
| Jetson AGX Orin | ~275 TOPS | 15~60W | AMR, 산업 로봇 |
| DRIVE AGX Thor | ~2000 TOPS | ~150W | 자율주행 |

TOPS(Tera Operations Per Second)는 AI 추론 성능 지표입니다. 숫자가 클수록 더 복잡한 신경망을 실시간으로 처리할 수 있습니다.

### 미들웨어: ROS2

**ROS2(Robot Operating System 2)** 는 로봇 소프트웨어 개발의 표준 미들웨어입니다. 센서 드라이버, 시각화 도구(RViz2), 로봇 기술 언어(URDF), 패키지 에코시스템을 제공합니다. Isaac ROS는 NVIDIA가 제공하는 ROS2 패키지 모음으로, Jetson에서 GPU 가속 인식 파이프라인을 실행합니다.

## 이 학습 사이트의 커리큘럼 지도

이 시리즈는 Physical AI를 처음 접하는 독자가 실습 가능한 수준까지 도달하도록 설계되었습니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│                  커리큘럼 학습 경로                               │
│                                                                 │
│  [01] 입문        개념 이해, 생태계 파악          ◀ 현재 위치    │
│    │                                                            │
│  [02] 수학 기초   강체 동역학, 기구학, 물리 엔진                 │
│    │                                                            │
│  [03] Omniverse/USD  3D 씬 구성, USD 워크플로                   │
│    │                                                            │
│  [04] Isaac Sim   시뮬레이터 실습, 센서 시뮬레이션              │
│    │                                                            │
│  [05] Isaac Lab   GPU 병렬 RL 훈련, 환경 커스텀                 │
│    │                                                            │
│  [06] RL/제어     PPO, SAC, 보상 설계, 계층 제어                │
│    │                                                            │
│  [07] Sim-to-Real 도메인 랜덤화, 실제 배포                      │
│    │                                                            │
│  [08] 모방/데이터 행동 복제, Diffusion Policy, ALOHA            │
│    │                                                            │
│  [09] 파운데이션  VLA 모델, GR00T, OpenVLA                      │
│    │                                                            │
│  [10] 오픈 시뮬   MuJoCo, Genesis, Webots 비교                  │
│    │                                                            │
│  [11] ROS2 통합   Isaac ROS, 실제 로봇 연결                      │
│    │                                                            │
│  [12] 프로젝트    통합 실습 프로젝트                             │
│    │                                                            │
│  [13] 고급 주제   멀티에이전트, 소프트 로봇, 연구 동향           │
│    │                                                            │
│  [14] 국내 산업   한국 로봇 산업, 스타트업, 채용 트렌드         │
```

### 추천 학습 경로

**기초부터 탄탄히 (권장)**: 01 → 02 → 03 → 04 → 05 → 06 순으로 진행합니다. 수학 기초(02)를 먼저 학습한 뒤 시뮬레이터로 넘어가는 것이 이해에 도움이 됩니다.

**실습 우선**: 수학 배경이 있는 독자라면 01 → 04 → 05 → 06 → 07로 빠르게 진행하고, 필요할 때 02~03을 참조할 수 있습니다.

**모델/데이터 중심**: AI/ML 배경이 있는 독자라면 01 → 02 → 08 → 09 → 05 순서로 데이터와 모델 중심으로 진행하고 시뮬레이터 실습을 나중에 추가할 수 있습니다.

## Physical AI를 시작하기 위한 준비물

**소프트웨어 환경**:
- Python 3.10 이상
- CUDA 지원 NVIDIA GPU (Isaac Sim 실행 시 RTX GPU 권장)
- Ubuntu 22.04 (권장) 또는 Windows WSL2

**사전 지식**:
- Python 프로그래밍 기초
- 선형대수 기초 (행렬, 벡터 연산)
- 딥러닝 기초 (유익하지만 필수는 아님)

**하드웨어 없이도 시작 가능**: Isaac Sim은 클라우드(NVIDIA Cloud / Google Colab Pro+)에서도 실행 가능하며, MuJoCo와 PyBullet은 CPU에서도 기본 실습이 가능합니다.

Physical AI는 로보틱스, 제어 이론, 컴퓨터 비전, 기계학습이 교차하는 복잡한 분야입니다. 한 번에 모든 것을 이해하려 하지 않고, 각 챕터에서 핵심을 파악하며 점진적으로 쌓아나가는 것을 권장합니다. 다음 섹션에서는 이 모든 것의 물리적 기반, 즉 강체 동역학과 운동학으로 들어갑니다.
