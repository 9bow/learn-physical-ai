## 왜 세 가지 컴퓨터가 필요한가

Physical AI 시스템을 개발하고 배포하는 과정에는 서로 다른 성격의 컴퓨팅이 필요합니다. 대용량 데이터로 신경망을 훈련하는 작업, 물리 법칙에 따라 로봇 환경을 시뮬레이션하는 작업, 실제 로봇에서 밀리초 단위로 추론하는 작업은 각각 다른 하드웨어 특성을 요구합니다.

NVIDIA는 이 세 단계를 **3-컴퓨터 패러다임(Three-Computer Paradigm)** 으로 공식화했습니다. Jensen Huang CEO는 "로봇을 만들려면 세 종류의 컴퓨터가 필요하다"고 반복해서 강조합니다.

```text
┌────────────────────────────────────────────────────────────────┐
│               NVIDIA 3-컴퓨터 패러다임                          │
│                                                                │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │  컴퓨터 1   │    │  컴퓨터 2   │    │  컴퓨터 3   │         │
│  │    DGX      │    │    OVX      │    │  Jetson/AGX │         │
│  │             │    │             │    │             │         │
│  │  AI 훈련    │    │ 시뮬레이션  │    │  로봇 배포  │         │
│  │  모델 개발  │    │ 디지털트윈  │    │  엣지 추론  │         │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘         │
│         │                  │                  │                │
│         └──────────────────┼──────────────────┘                │
```

## 컴퓨터 1: DGX — AI 훈련

**DGX** 는 NVIDIA의 AI 훈련용 슈퍼컴퓨터 시스템입니다. H100, B200 등 최신 데이터센터 GPU를 다수 탑재하여, 수백만에서 수십억 파라미터 규모의 신경망을 훈련합니다.

Physical AI 맥락에서 DGX의 역할:

- **기반 모델 사전훈련**: GR00T와 같은 휴머노이드 기반 모델을 수백만 개의 시연 데이터로 훈련
- **강화학습 정책 훈련**: Isaac Lab과 연동하여 시뮬레이션 환경에서 수백만 번의 롤아웃(rollout)을 병렬 실행하고 정책을 개선
- **Fine-tuning**: 범용 기반 모델을 특정 로봇/태스크에 적응시키는 단계

DGX는 클라우드(NVIDIA DGX Cloud) 또는 온프레미스(on-premises)로 운영될 수 있습니다. 개인 연구자나 소규모 팀은 NVIDIA NGC(NVIDIA GPU Cloud)를 통해 접근합니다.

## 컴퓨터 2: OVX — 시뮬레이션과 디지털 트윈

**OVX** 는 NVIDIA Omniverse를 실행하기 위해 최적화된 시각화·시뮬레이션 서버입니다. RTX GPU를 기반으로 실시간 레이트레이싱(ray tracing)과 물리 시뮬레이션을 동시에 처리합니다.

### Omniverse와 USD

**Omniverse** 는 NVIDIA가 개발한 3D 협업 및 시뮬레이션 플랫폼입니다. 핵심은 **USD(Universal Scene Description)** 형식입니다. Pixar Animation Studios가 개발한 USD는 3D 씬의 표준 교환 형식으로, 복잡한 로봇 환경을 계층적으로 기술합니다.

```text
┌─────────────────────────────────────────────────────┐
│              Omniverse / USD 생태계                  │
│                                                     │
│  CAD 소프트웨어 (SolidWorks, Fusion360)              │
│         │                                           │
│         ▼ USD 변환                                  │
│  Omniverse Stage (3D 씬)                            │
│         │                                           │
│    ┌────┴────┐                                      │
│    │         │                                      │
│    ▼         ▼                                      │
│  Isaac Sim  시각화/협업 도구                          │
│  (로봇 시뮬레이션)                                   │
└─────────────────────────────────────────────────────┘
```

### 디지털 트윈(Digital Twin)

디지털 트윈은 **실제 물리 시스템의 정밀한 디지털 복제본**입니다. 단순한 3D 모델이 아니라, 물리 특성(질량, 마찰, 관성), 센서 특성(카메라 렌즈 왜곡, 노이즈 모델), 작업 흐름까지 현실을 최대한 충실하게 반영합니다.

Physical AI에서 디지털 트윈의 가치:

- **훈련 데이터 생성**: 실제 로봇을 운영하지 않고도 수백만 개의 합성 훈련 데이터를 생성할 수 있습니다.
- **안전한 실험**: 물리 로봇에서 실험하면 비용이 크고 위험하지만, 시뮬레이터에서는 안전하게 실패하고 재시도할 수 있습니다.
- **도메인 랜덤화(domain randomization)**: 물체 색상, 조명, 마찰 계수를 무작위로 변경하며 다양한 조건에서의 데이터를 생성하여 Sim-to-Real 갭을 줄입니다.

## 컴퓨터 3: Jetson/AGX — 로봇 배포

**Jetson** 과 **AGX** 는 엣지(edge) 환경에서 AI 추론을 실행하기 위한 임베디드 컴퓨팅 플랫폼입니다.

| 제품 | 주요 용도 | 특징 |
|------|----------|------|
| Jetson Nano/Orin Nano | 소형 로봇, 교육용 | 저전력, 저비용 |
| Jetson AGX Orin | 자율 이동 로봇, 드론 | 고성능 AI 추론 |
| DRIVE AGX Thor | 자율주행 차량 | 차량 안전 등급, 고대역폭 센서 처리 |

이 플랫폼들의 공통 특징은 **저전력에서 고성능 추론**을 달성한다는 것입니다. 로봇은 배터리로 구동되거나 열방산(thermal dissipation) 제약이 있어, 데이터센터 GPU처럼 수백 와트를 소비할 수 없습니다. NVIDIA의 TensorRT 추론 최적화 엔진은 훈련된 모델을 엣지 하드웨어에서 효율적으로 실행할 수 있도록 변환합니다.

## 데이터 플라이휠: 선순환 구조

세 컴퓨터가 개별적으로 작동한다면 그저 다른 역할의 하드웨어들일 뿐입니다. NVIDIA 전략의 핵심은 이 세 단계가 **데이터 플라이휠(data flywheel)** 을 통해 선순환을 이룬다는 점입니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│                      데이터 플라이휠                             │
│                                                                 │
│                   ┌─────────────────┐                           │
│               ┌──▶│  DGX: 모델 훈련 │──┐                        │
│               │   └─────────────────┘  │                        │
│               │                        ▼                        │
│   ┌───────────────────┐    ┌─────────────────────┐              │
│   │ Jetson: 실제 배포 │    │ OVX: 시뮬레이션 생성 │             │
│   │ (실세계 데이터 수집│◀───│ (합성 데이터 제공)   │             │
│   └───────────────────┘    └─────────────────────┘              │
│               │                        ▲                        │
│               └────────────────────────┘                        │
│          실제 로봇 경험 → 시뮬레이터 개선                        │
└─────────────────────────────────────────────────────────────────┘
```

플라이휠의 작동 방식:

1. **OVX(시뮬레이션)** 에서 다양한 환경의 합성 데이터를 대규모로 생성합니다.
2. **DGX(훈련)** 에서 이 데이터로 로봇 정책을 훈련합니다.
3. **Jetson(배포)** 에서 실제 로봇에 정책을 적용하고, 실세계 경험 데이터를 수집합니다.
4. 수집된 실세계 데이터가 다시 시뮬레이터의 현실성을 높이고, 더 나은 훈련 데이터로 환원됩니다.
5. 루프가 반복될수록 정책 품질이 향상됩니다.

이 플라이휠이 강력한 이유는 **실세계 데이터 수집의 병목을 시뮬레이션으로 완화**하기 때문입니다. 물리 로봇 운영 없이도 수백만 개의 훈련 에피소드를 생성할 수 있고, 실제 로봇은 주로 검증과 시뮬레이터 개선에 활용됩니다.

## Isaac 플랫폼: 3-컴퓨터를 연결하는 소프트웨어

NVIDIA는 3-컴퓨터 패러다임을 소프트웨어 수준에서 통합하는 **Isaac 플랫폼**을 제공합니다.

```text
┌────────────────────────────────────────────────────────┐
│                  NVIDIA Isaac 플랫폼                    │
├──────────────────┬───────────────────┬─────────────────┤
│  Isaac Sim       │  Isaac Lab        │  Isaac ROS      │
│  (시뮬레이터)    │  (RL 훈련 프레임워크)│ (ROS2 통합)   │
│  OVX 기반        │  DGX 기반         │  Jetson 기반    │
├──────────────────┴───────────────────┴─────────────────┤
│              공통 기반: USD, PhysX, CUDA               │
└────────────────────────────────────────────────────────┘
```

- **Isaac Sim**: Omniverse 기반 로봇 시뮬레이터. 물리 정확도 높은 환경에서 센서 시뮬레이션, 로봇 모델 임포트, 합성 데이터 생성을 지원합니다.
- **Isaac Lab**: GPU 병렬 강화학습 훈련 프레임워크. Isaac Sim 위에서 수천 개의 환경을 동시에 실행하며 정책을 훈련합니다.
- **Isaac ROS**: ROS2(Robot Operating System 2) 패키지 모음. 실제 로봇에서 지각(perception)·조작(manipulation) 파이프라인을 Jetson에서 실행합니다.

## 이 패러다임이 가져오는 변화

3-컴퓨터 패러다임 이전에는 각 단계가 분절되어 있었습니다. 연구자들은 각자 다른 시뮬레이터, 다른 훈련 코드, 다른 배포 파이프라인을 사용했습니다. NVIDIA의 접근은 이 파편화를 통합하려는 시도입니다.

물론 이 접근이 유일한 방법은 아닙니다. DeepMind의 MuJoCo, Google의 Drake, 오픈소스 Gazebo 등도 각자의 생태계를 형성하고 있습니다. 하지만 NVIDIA가 하드웨어(GPU, Jetson)에서 소프트웨어(Isaac, Omniverse)까지 수직 통합된 플랫폼을 제공한다는 점은 산업 채택 측면에서 강력한 이점입니다.

다음 챕터에서는 이 현대적 패러다임이 탄생하기까지, 고전 로보틱스 제어 방식에서 학습 기반 접근으로 어떻게 패러다임이 전환되었는지 역사적 흐름을 살펴보겠습니다.
