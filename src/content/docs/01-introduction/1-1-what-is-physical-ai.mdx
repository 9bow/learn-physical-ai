## Physical AI란 무엇인가

**Physical AI**는 물리 세계를 직접 인지(perceive)하고 이해(understand)하며, 그 세계 안에서 실제 행동(act)을 수행하는 인공지능 시스템을 가리킵니다. 카메라나 LiDAR로 주변 환경을 감지하고, 그 정보를 바탕으로 판단하며, 로봇 팔을 움직이거나 자율주행 차량을 조향하는 것이 전형적인 예입니다.

오늘날 "AI"라고 부르는 많은 시스템, 예컨대 대화형 언어 모델이나 이미지 생성 모델은 디지털 데이터만을 다룹니다. 이들은 텍스트를 받아 텍스트를 출력하거나, 픽셀 배열을 변환하는 방식으로 작동합니다. 물리 제약, 중력, 마찰, 실시간 피드백은 존재하지 않습니다. Physical AI는 이 경계를 넘어 **물리 법칙이 지배하는 현실 세계**에 직접 개입합니다.

## Embodied AI: 몸이 있는 인공지능

Physical AI의 핵심 개념 중 하나는 **체화 인지(embodied cognition)** 입니다. 인지과학에서 출발한 이 관점은, 지능이란 순수하게 추상적인 계산이 아니라 몸(body)과 환경 간의 상호작용 속에서 발생한다고 주장합니다. 로봇이 물체를 집으려면 손목 각도, 손가락 힘, 물체의 무게와 마찰을 동시에 고려해야 합니다. 이 과정은 단순한 패턴 매칭이 아니라 **피드백 루프 속에서의 연속적 의사결정**입니다.

**Embodied AI**는 Physical AI와 거의 동의어로 사용되지만, 학계에서는 다음과 같이 강조점이 다릅니다.

- **Embodied AI**: 인지와 행동이 물리적 신체를 통해 통합된다는 **철학적·인지과학적** 관점
- **Physical AI**: 실제 물리 환경에서 작동하는 AI 시스템의 **공학적·산업적** 명칭

NVIDIA는 "Physical AI"라는 용어를 공식적으로 사용하여, 로보틱스·자율주행·산업 자동화에 걸친 AI 플랫폼 전략을 설명합니다.

## 디지털 AI와 Physical AI의 차이

```text
┌─────────────────────────────────────────────────────────────┐
│                    AI 유형 비교                              │
├───────────────────┬─────────────────┬───────────────────────┤
│ 속성              │ 디지털 AI       │ Physical AI           │
├───────────────────┼─────────────────┼───────────────────────┤
│ 입력              │ 텍스트/이미지   │ 센서 스트림(실시간)   │
│ 출력              │ 텍스트/픽셀     │ 모터 명령/제어 신호   │
│ 시간 제약         │ 없음(비동기)    │ 엄격한 실시간 요구    │
│ 물리 법칙         │ 무관            │ 핵심 제약 조건        │
│ 안전 요구         │ 낮음            │ 매우 높음(인명 관련)  │
│ 데이터 피드백     │ 단방향          │ 폐루프(closed-loop)   │
│ 일반화 어려움     │ 도메인 이동     │ Sim-to-Real 갭        │
└───────────────────┴─────────────────┴───────────────────────┘
```

가장 큰 차이는 **폐루프 제어(closed-loop control)** 입니다. 언어 모델은 입력을 받아 출력을 내놓으면 끝입니다. 반면 Physical AI 시스템은 행동의 결과를 다시 센서로 감지하여, 다음 행동을 조정합니다. 이 루프는 초당 수십~수백 회 반복될 수 있습니다.

## 센서-인지-행동 루프

Physical AI의 작동 원리를 가장 간결하게 표현하면 **센서-인지-행동(Sense-Think-Act)** 루프입니다.

```text
       ┌─────────────────────────────────────┐
       │          Physical AI 제어 루프       │
       │                                     │
       │   ┌──────────┐    ┌──────────┐      │
       │   │  센서    │    │  인지    │      │
       │   │ Sensor   │───▶│  Think   │      │
       │   │          │    │          │      │
       │   └──────────┘    └────┬─────┘      │
       │        ▲               │            │
       │        │               ▼            │
       │   ┌────┴─────┐    ┌──────────┐      │
       │   │  환경    │◀───│  행동    │      │
       │   │ World    │    │  Act     │      │
       │   │          │    │          │      │
       │   └──────────┘    └──────────┘      │
       └─────────────────────────────────────┘
```

각 단계를 구체적으로 살펴보면:

**센서(Sense)**: RGB-D 카메라, LiDAR, IMU(관성 측정 장치), 촉각 센서, 힘/토크 센서 등이 환경 정보를 수집합니다. 데이터는 연속적이고, 노이즈가 포함되며, 실시간으로 처리되어야 합니다.

**인지(Think)**: 수집된 데이터를 바탕으로 현재 상태를 추정(state estimation)하고, 목표를 향한 행동 계획을 수립합니다. 딥러닝 모델, 강화학습 정책(policy), 고전적 계획 알고리즘이 모두 사용될 수 있습니다.

**행동(Act)**: 계획된 행동을 모터, 액추에이터, 스티어링 시스템 등을 통해 물리적으로 실행합니다. 실행 결과는 다시 센서로 감지되어 루프가 이어집니다.

이 루프의 **빈도(frequency)** 가 중요합니다. 보행 로봇의 균형 제어는 1kHz 이상의 제어 루프가 필요할 수 있고, 물체 파지(grasping)는 수십 Hz, 고수준 태스크 계획은 수 Hz로도 충분할 수 있습니다. 계층별로 다른 빈도의 루프를 구성하는 것을 **계층적 제어(hierarchical control)** 라고 합니다.

## 대표적인 Physical AI 사례

### 자율 이동 로봇(AMR)

물류 창고에서 선반을 나르거나 공장 바닥을 자율로 이동하는 로봇입니다. LiDAR와 카메라를 이용해 동적 환경에서 경로를 계획하고, 사람과의 충돌을 피하면서 작업을 수행합니다. Amazon Robotics, Boston Dynamics Spot 등이 대표 사례입니다.

### 휴머노이드 로봇

인간의 형태를 갖춘 로봇으로, 인간의 작업 환경(계단, 문손잡이, 다양한 높이의 작업대)에서 작동하도록 설계됩니다. NVIDIA GR00T는 휴머노이드 로봇을 위한 기반 모델(foundation model)로, 다양한 조작 태스크를 학습할 수 있도록 설계되었습니다. Figure AI, Agility Robotics Digit 등이 상용 휴머노이드 분야에서 주목받고 있습니다.

### 자율주행 차량

카메라, LiDAR, 레이더 등 다중 센서를 융합하여 도로 환경을 인식하고, 차선 유지·교차로 통과·보행자 회피 등 복잡한 주행 시나리오를 처리합니다. Waymo, Tesla Autopilot, NVIDIA DRIVE 플랫폼이 대표적입니다.

### 산업용 조작 로봇(Manipulation Robot)

공장 조립 라인에서 부품을 집고, 나사를 조이고, 용접하는 로봇입니다. 최근에는 고전적인 위치 제어 방식에서 벗어나, 비전 기반 학습 정책으로 유연성을 높이는 방향으로 발전하고 있습니다.

## Physical AI가 어려운 이유

Physical AI가 디지털 AI보다 훨씬 어려운 이유는 다음과 같습니다.

**현실 세계의 복잡성**: 조명 변화, 먼지, 예상치 못한 물체, 사람의 개입 등 통제되지 않은 변수가 무한히 존재합니다. 디지털 AI는 잘 정의된 입력을 받지만, Physical AI는 항상 "열린 세계(open world)"에서 작동합니다.

**실패의 비용**: 언어 모델이 오답을 내면 사용자가 불편을 겪습니다. 하지만 자율주행 차량이 오판하면 인명 피해가 발생할 수 있습니다. 안전(safety)과 신뢰성(reliability)에 대한 요구 수준이 근본적으로 다릅니다.

**데이터 수집의 어려움**: 텍스트 데이터는 인터넷에 수십억 건이 있습니다. 하지만 로봇 조작 데이터는 물리적 로봇을 운영하며 시연(demonstration)을 수집해야 하므로, 수집 비용과 시간이 매우 큽니다.

**시뮬레이션과 현실의 격차**: 시뮬레이터에서 완벽하게 작동하는 정책이 실제 로봇에서는 실패하는 경우가 많습니다. 이를 **Sim-to-Real 갭** 이라 하며, Physical AI 연구의 핵심 과제 중 하나입니다.
