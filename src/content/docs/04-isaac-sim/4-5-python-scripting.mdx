## Python API의 중요성

Isaac Sim을 GUI로만 사용하면 반복적인 실험, 대규모 데이터 생성, 강화학습 루프를 자동화할 수 없습니다. Python API를 통해 Isaac Sim의 모든 기능—씬 로딩, 물리 제어, 센서 데이터 수집, 렌더링—을 프로그래밍 방식으로 제어하는 것이 실제 연구와 개발에서 핵심입니다.

> **버전 주의**: Isaac Sim의 Python API 모듈 구조는 버전에 따라 변경되어 왔습니다. 특히 `omni.isaac.core`에서 `isaacsim` 계열 패키지로의 전환이 진행 중입니다. 아래 예시는 개념 이해를 위한 의사코드이며, 실제 모듈명과 함수명은 사용하는 버전의 공식 문서를 반드시 확인하세요.

## 실행 모드: Standalone vs Extension

Isaac Sim Python 코드를 실행하는 방식은 두 가지입니다.

```text
Standalone 모드 vs Extension 모드 비교

┌──────────────────┬─────────────────────┬────────────────────────┐
│  특성            │  Standalone 모드    │  Extension(GUI) 모드   │
├──────────────────┼─────────────────────┼────────────────────────┤
│  실행 방식       │ CLI에서 스크립트    │ Isaac Sim GUI 내       │
│                  │ 직접 실행           │ Script Editor 사용     │
├──────────────────┼─────────────────────┼────────────────────────┤
│  GUI             │ 없음 (headless)     │ 있음 (Viewport 표시)  │
│                  │ 선택적으로 활성화   │                        │
├──────────────────┼─────────────────────┼────────────────────────┤
│  Isaac Sim       │ 스크립트가 직접     │ 실행 중인 Isaac Sim에  │
│  라이프사이클    │ 시작·종료           │ 부착(attach) 형태      │
├──────────────────┼─────────────────────┼────────────────────────┤
│  주요 용도       │ 훈련 루프           │ 개발·디버깅            │
│                  │ 합성 데이터 생성    │ 빠른 프로토타이핑      │
│                  │ CI/CD 자동화        │ 대화형 테스트          │
├──────────────────┼─────────────────────┼────────────────────────┤
│  실행 명령 예    │ ./python.sh         │ GUI에서 Script Editor  │
│  (버전마다 다름) │ my_script.py        │ 열고 실행 버튼         │
└──────────────────┴─────────────────────┴────────────────────────┘
```

## Standalone 모드 기본 구조

Standalone 모드에서 Isaac Sim Python 스크립트의 전형적인 구조입니다.

```python
# 개념 이해용 의사코드
# 실제 import 경로와 클래스명은 Isaac Sim 버전에 따라 다릅니다
# 반드시 공식 문서의 해당 버전 예제를 참조하세요

# 1단계: Isaac Sim 애플리케이션 초기화
# (이 부분은 버전에 따라 방식이 크게 다름)
from isaacsim import SimulationApp  # 최신 버전 기준 (버전 확인 필요)

simulation_app = SimulationApp({"headless": False})  # GUI 표시
# headless=True 로 설정 시 GUI 없이 실행 (서버 환경)

# 2단계: Isaac Sim 모듈 import
# (SimulationApp 초기화 이후에 import해야 함 — 중요!)
import omni
from omni.isaac.core import World   # 버전에 따라 경로 다를 수 있음

# 3단계: World 생성 및 씬 구성
world = World(stage_units_in_meters=1.0)
world.scene.add_default_ground_plane()

# 4단계: 로봇 또는 기타 객체 추가
# (실제 클래스명은 버전과 모듈 구조에 따라 다름)
# robot = Franka(prim_path="/World/Franka", name="franka")
# world.scene.add(robot)

# 5단계: 씬 초기화
world.reset()

# 6단계: 시뮬레이션 루프
step_count = 0
while simulation_app.is_running():
    world.step(render=True)  # 한 스텝 진행

    if world.is_playing():
        # 여기에 제어 로직 작성
        step_count += 1
        if step_count >= 1000:
            break

# 7단계: 종료
simulation_app.close()
```

## World API — 씬 관리의 핵심

`World` 클래스(또는 버전에 따른 동등한 클래스)는 Isaac Sim에서 씬의 라이프사이클을 관리하는 핵심 객체입니다.

```python
# 개념 이해용 의사코드 — World API 주요 메서드

world = World(...)

# 씬 리셋 — 초기 상태로 복원
world.reset()

# 시뮬레이션 한 스텝 진행
world.step(render=True)   # render=True: 렌더링 포함
world.step(render=False)  # render=False: 렌더링 생략, 더 빠름

# 시뮬레이션 상태 확인
world.is_playing()    # 재생 중인지
world.is_stopped()    # 정지 상태인지

# 씬 제어
world.play()   # 시뮬레이션 시작
world.pause()  # 일시정지
world.stop()   # 정지 및 초기화

# 현재 시뮬레이션 시간
current_time = world.current_time  # 초 단위
```

## Articulation API — 로봇 관절 제어

임포트된 로봇을 Python으로 제어하려면 Articulation 인터페이스를 사용합니다. 아래는 관절 제어의 핵심 패턴을 보여주는 의사코드입니다.

```python
# 개념 이해용 의사코드
# 실제 클래스명/메서드명은 공식 문서의 해당 버전 API 참조 필수

# 씬에 로봇 추가 (USD 파일에서 로드)
# 방법은 버전에 따라 다양 — 공식 예제 참조

# 씬 리셋 후 Articulation 인터페이스 획득
world.reset()
# robot_articulation = world.scene.get_object("franka")  # 예시

# 관절 상태 읽기
joint_positions = robot_articulation.get_joint_positions()
# → numpy array: [q0, q1, q2, q3, q4, q5, q6, finger_l, finger_r]
# 단위: 라디안 (revolute), 미터 (prismatic)

joint_velocities = robot_articulation.get_joint_velocities()
# → numpy array: 각 관절의 현재 각속도 (rad/s)

# 관절 위치 제어 (Position Drive 모드)
import numpy as np
target_positions = np.array([0.0, -0.5, 0.0, -2.0, 0.0, 1.5, 0.0, 0.04, 0.04])
robot_articulation.set_joint_position_targets(target_positions)
# → 다음 world.step() 호출 시 드라이브 적용

# 관절 속도 제어 (Velocity Drive 모드)
target_velocities = np.zeros(9)
robot_articulation.set_joint_velocity_targets(target_velocities)

# 직접 토크 인가 (Effort 제어)
torques = np.zeros(9)
robot_articulation.apply_action(
    ArticulationAction(joint_efforts=torques)
)
```

## 제어 루프 패턴

실제 강화학습이나 제어 알고리즘에서 사용하는 전형적인 제어 루프 패턴입니다.

```python
# 개념 이해용 의사코드 — 제어 루프 패턴

world.reset()
robot_articulation = ...  # Articulation 객체 획득

for episode in range(num_episodes):
    world.reset()  # 각 에피소드 시작 전 환경 초기화

    for step in range(max_steps_per_episode):
        # 1. 현재 상태 관측
        obs = {
            "joint_pos": robot_articulation.get_joint_positions(),
            "joint_vel": robot_articulation.get_joint_velocities(),
            # 카메라 이미지, 센서 데이터 등 추가 가능
        }

        # 2. 정책/컨트롤러로 행동 계산
        action = policy.compute_action(obs)
        # action: numpy array of target joint positions or torques

        # 3. 행동 적용
        robot_articulation.set_joint_position_targets(action)

        # 4. 시뮬레이션 한 스텝 진행
        world.step(render=True)

        # 5. 보상·종료 조건 계산
        reward = compute_reward(obs, action)
        done = check_termination(obs)

        if done:
            break

    print(f"Episode {episode} completed")
```

## Extension 모드 — GUI 내 스크립팅

Isaac Sim GUI 내의 Script Editor를 통해 짧은 스크립트를 대화식으로 실행할 수 있습니다. 개발·디버깅 단계에서 유용합니다.

```text
Script Editor 접근 경로 (버전에 따라 다를 수 있음)

Window → Script Editor
또는
Isaac → Script Editor

Script Editor UI:
┌────────────────────────────────────────────────────┐
│  # Python 코드 입력                                 │
│  import omni                                       │
│  stage = omni.usd.get_context().get_stage()        │
│  prim = stage.GetPrimAtPath("/World/Robot")        │
│  print(prim)                                       │
│                                                    │
│  [실행 버튼]  [지우기]                              │
├────────────────────────────────────────────────────┤
│  출력:                                              │
│  Usd.Prim</World/Robot>                            │
└────────────────────────────────────────────────────┘
```

Extension 모드에서는 이미 실행 중인 Isaac Sim 인스턴스에 접근하므로 `SimulationApp` 초기화 없이 바로 `omni.usd`나 `omni.isaac.core` 모듈을 사용할 수 있습니다.

## USD Python API 직접 활용

Isaac Sim API와 함께 **OpenUSD Python 바인딩**(`pxr` 모듈)을 직접 사용하면 더 세밀한 씬 조작이 가능합니다.

```python
# 개념 이해용 의사코드 — USD Python API 활용 예시

from pxr import Usd, UsdGeom, UsdPhysics, Gf

# 현재 Stage 접근
import omni.usd
stage = omni.usd.get_context().get_stage()

# Prim 접근
robot_prim = stage.GetPrimAtPath("/World/Robot")

# Attribute 읽기
xform = UsdGeom.Xformable(robot_prim)
xform_ops = xform.GetOrderedXformOps()

# 물리 속성 읽기
rigid_body = UsdPhysics.RigidBodyAPI(robot_prim)
velocity = rigid_body.GetVelocityAttr().Get()

# Attribute 쓰기 (시뮬레이션 정지 상태에서)
translate_op = xform.GetXformOp(UsdGeom.XformOp.TypeTranslate)
translate_op.Set(Gf.Vec3d(1.0, 0.0, 0.76))

# 새 Prim 생성
new_prim = stage.DefinePrim("/World/NewObject", "Xform")
UsdPhysics.RigidBodyAPI.Apply(new_prim)
```

## 실용 팁: 빠른 개발 사이클

Isaac Sim Python 개발에서 효율적인 워크플로를 위한 팁입니다.

```text
개발 효율화 팁

1. Extension 모드로 빠른 프로토타이핑
   → Script Editor에서 코드 조각 테스트
   → 동작 확인 후 Standalone 스크립트로 정리

2. world.reset() 활용
   → 매 실험마다 reset()으로 깔끔하게 초기화
   → 씬 상태 누적 방지

3. render=False로 학습 속도 향상
   → 렌더링이 필요 없는 훈련 루프에서 사용
   → 렌더링을 끄면 학습 속도가 향상됩니다(향상폭은 워크로드에 따라 다름)

4. 오류 발생 시 콘솔 로그 확인
   → Window → Console 에서 오류 메시지 확인
   → omni.log.warn() / omni.log.error() 로 커스텀 로그

5. 공식 예제 코드 활용
   → Isaac Sim 설치 경로 내 standalone_examples/ 디렉토리
   → GitHub: github.com/isaac-sim/IsaacSim
```
