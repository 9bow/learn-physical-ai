## 합성 데이터가 필요한 이유

현실 세계에서 로봇 인식 모델을 훈련하기 위한 데이터를 수집하는 것은 여러 면에서 어렵습니다.

**비용**: 수만 장의 이미지에 사람이 직접 레이블을 붙이는 작업은 시간과 비용이 막대합니다.

**위험 상황**: 로봇이 실패하는 엣지 케이스(물체 낙하, 예상치 못한 충돌)는 데이터 수집 자체가 위험하거나 반복하기 어렵습니다.

**다양성 부족**: 실제 데이터는 특정 조명 조건, 배경, 물체 위치에 편향되어 있어 모델의 일반화 능력이 떨어질 수 있습니다.

합성 데이터(Synthetic Data)는 이 문제를 해결합니다. 시뮬레이터에서 다양한 조건의 이미지를 자동으로 생성하고 정확한 레이블을 동시에 얻을 수 있습니다. Omniverse Replicator는 이 파이프라인을 위한 전용 프레임워크입니다.

## Omniverse Replicator 개요

Replicator는 Omniverse의 **합성 데이터 생성 프레임워크**입니다. 핵심 기능은 다음과 같습니다.

```text
Replicator 핵심 기능

┌──────────────────────────────────────────────────────────────┐
│                     Omniverse Replicator                     │
├──────────────────┬───────────────────┬───────────────────────┤
│  도메인 랜덤화    │  어노테이션 생성  │  데이터 기록·저장      │
│  (Randomization) │  (Annotation)     │  (Writer)             │
├──────────────────┼───────────────────┼───────────────────────┤
│ • 위치/회전 무작위화│ • 바운딩 박스   │ • KITTI 포맷          │
│ • 머티리얼 변형  │ • 인스턴스 세그멘  │ • COCO 포맷           │
│ • 조명 강도/색상 │   테이션          │ • Pascal VOC 포맷     │
│ • 카메라 자세    │ • 깊이 맵         │ • 커스텀 포맷          │
│ • 물체 종류/수   │ • 법선 맵         │ • Nucleus에 직접 저장 │
│ • 배경 텍스처    │ • 키포인트        │                       │
└──────────────────┴───────────────────┴───────────────────────┘
```

## Replicator Python API 기본 구조

Replicator를 사용하는 스크립트는 세 부분으로 구성됩니다: **씬 설정**, **랜덤화 정의**, **실행 루프**.

```python
# 개념 이해용 의사코드
# 실제 API와 import 경로는 Replicator/Isaac Sim 버전에 따라 다름
# 공식 문서의 해당 버전 예제를 기준으로 사용할 것

import omni.replicator.core as rep  # Replicator 핵심 모듈 (버전 확인)

# ─────────────────────────────────────────
# 1단계: 씬 구성 (기존 씬 활용 또는 여기서 생성)
# ─────────────────────────────────────────
with rep.new_layer():  # 랜덤화용 새 레이어
    # 카메라 생성
    camera = rep.create.camera(
        position=(0, 0.5, 1.5),
        look_at=(0, 0, 0.5)
    )

    # 렌더 제품(Render Product) 생성
    # 카메라와 해상도를 연결하는 렌더링 설정
    render_product = rep.create.render_product(
        camera, (1280, 720)  # 해상도
    )

    # ─────────────────────────────────────────
    # 2단계: 랜덤화 정의
    # ─────────────────────────────────────────
    # 물체 위치 랜덤화
    with rep.randomizer.register(object_prim_path):
        rep.modify.pose(
            position=rep.distribution.uniform(
                (-0.3, -0.3, 0.76),   # 최솟값 (x, y, z) — 높이는 Z
                (0.3, 0.3, 0.76)      # 최댓값
            ),
            rotation=rep.distribution.uniform(
                (0, -180, 0),          # 회전 최솟값
                (0, 180, 0)            # 회전 최댓값
            )
        )

    # 조명 강도 랜덤화
    with rep.randomizer.register(light_prim_path):
        rep.modify.attribute(
            "inputs:intensity",
            rep.distribution.uniform(500, 3000)
        )

    # 머티리얼 색상 랜덤화
    with rep.randomizer.register(material_prim_path):
        rep.modify.attribute(
            "inputs:diffuse_tint",
            rep.distribution.uniform((0, 0, 0), (1, 1, 1))
        )

    # ─────────────────────────────────────────
    # 3단계: 어노테이션 설정
    # ─────────────────────────────────────────
    # 바운딩 박스 어노테이터
    bbox_annotator = rep.AnnotatorRegistry.get_annotator(
        "bounding_box_2d_tight"
    )
    bbox_annotator.attach(render_product)

    # 인스턴스 세그멘테이션 어노테이터
    seg_annotator = rep.AnnotatorRegistry.get_annotator(
        "instance_segmentation"
    )
    seg_annotator.attach(render_product)

    # RGB 이미지 어노테이터
    rgb_annotator = rep.AnnotatorRegistry.get_annotator("rgb")
    rgb_annotator.attach(render_product)

    # ─────────────────────────────────────────
    # 4단계: Writer 설정 (데이터 저장 형식)
    # ─────────────────────────────────────────
    writer = rep.WriterRegistry.get("BasicWriter")  # 기본 Writer
    writer.initialize(
        output_dir="/output/synthetic_data",
        rgb=True,
        bounding_box_2d_tight=True,
        instance_segmentation=True
    )
    writer.attach(render_product)

# ─────────────────────────────────────────
# 5단계: 데이터 생성 실행
# ─────────────────────────────────────────
with rep.trigger.on_frame(num_frames=1000):
    # 각 프레임마다 랜덤화 적용 후 렌더링·저장
    rep.randomizer.run_all()  # 등록된 모든 랜덤화 실행
```

## 어노테이션 타입 상세

Replicator가 생성할 수 있는 주요 어노테이션 타입을 정리합니다.

```text
Replicator 어노테이션 타입

┌─────────────────────────────┬───────────────────────────────────┐
│  어노테이션 타입             │  데이터 형태와 용도               │
├─────────────────────────────┼───────────────────────────────────┤
│  rgb                        │  컬러 이미지 (H×W×3)             │
│  (RGB 이미지)               │  학습 입력 데이터                 │
├─────────────────────────────┼───────────────────────────────────┤
│  bounding_box_2d_tight      │  픽셀 단위 정확한 2D 바운딩 박스 │
│  bounding_box_2d_loose      │  여유 있는 2D 바운딩 박스         │
│  bounding_box_3d            │  3D 공간 바운딩 박스               │
│  (물체 감지 레이블)          │  물체 감지 모델 학습              │
├─────────────────────────────┼───────────────────────────────────┤
│  instance_segmentation      │  각 인스턴스에 고유 색상 마스크   │
│  semantic_segmentation      │  클래스별 마스크                   │
│  (세그멘테이션 마스크)       │  세그멘테이션 모델 학습           │
├─────────────────────────────┼───────────────────────────────────┤
│  distance_to_camera         │  카메라까지의 거리 맵 (H×W×1)    │
│  distance_to_image_plane    │  이미지 평면 기준 깊이 맵         │
│  (깊이 맵)                  │  3D 인식, 깊이 추정 학습          │
├─────────────────────────────┼───────────────────────────────────┤
│  normals                    │  표면 법선 맵 (H×W×3)            │
│  (법선 맵)                  │  자세 추정, 표면 분석             │
├─────────────────────────────┼───────────────────────────────────┤
│  keypoints_2d               │  관절·특징점의 2D 이미지 좌표     │
│  (키포인트)                  │  자세 추정 모델 학습             │
└─────────────────────────────┴───────────────────────────────────┘
```

## 합성 데이터 파이프라인 전체 흐름

```text
합성 데이터 생성 → 모델 학습 전체 파이프라인

단계 1: 씬 준비
  Isaac Sim에서 기본 씬 구성
  (환경 자산 + 로봇 + 대상 물체 + 카메라)
         │
         ▼
단계 2: 도메인 랜덤화 설정
  Replicator API로 랜덤화 파라미터 정의
  (물체 위치/회전, 조명, 머티리얼, 배경, 카메라 자세)
         │
         ▼
단계 3: 대규모 데이터 생성 (headless 모드)
  예: 이미지당 약 5초로 가정하면, 10,000 이미지 × 5초/이미지 = 약 14시간
  (실제 소요 시간은 GPU 성능·씬 복잡도에 따라 크게 다름)
  다중 GPU 병렬화로 시간 단축 가능
  결과: 이미지 + 어노테이션 파일
         │
         ▼
단계 4: 데이터 후처리 및 검증
  생성된 어노테이션 시각화로 품질 확인
  이상치 제거 (극단적 오클루전 등)
  필요 시 형식 변환 (COCO, KITTI 등)
         │
         ▼
단계 5: 실제 데이터와 혼합 (선택)
  합성 데이터 80% + 실제 데이터 20%
  또는 합성 데이터만으로 사전학습 후 실제 데이터로 파인튜닝
         │
         ▼
단계 6: 모델 학습
  YOLOv8, DINO, Mask R-CNN 등
  합성 데이터로 학습된 모델 평가
         │
         ▼
단계 7: 실제 환경 검증
  실물 카메라로 테스트
  도메인 갭 측정 및 피드백
  → 도메인 랜덤화 파라미터 조정 후 반복
```

## Writer와 데이터 형식

Replicator Writer는 생성된 어노테이션을 표준 데이터셋 형식으로 저장합니다.

```python
# 개념 이해용 의사코드 — 다양한 Writer 설정

# KITTI 형식 (자율주행·로봇 분야 표준)
kitti_writer = rep.WriterRegistry.get("KittiWriter")
kitti_writer.initialize(
    output_dir="/output/kitti_data",
    bounding_box_2d_tight=True,
    semantic_segmentation=True
)

# COCO 형식 (컴퓨터 비전 표준)
coco_writer = rep.WriterRegistry.get("COCOWriter")  # 버전에 따라 지원 여부 확인
coco_writer.initialize(
    output_dir="/output/coco_data",
    bounding_box_2d_tight=True,
    instance_segmentation=True
)
```

## 생성 데이터 품질 검증

합성 데이터를 학습에 사용하기 전 품질을 검증하는 것이 중요합니다.

```python
# 개념 이해용 의사코드 — 어노테이션 시각화 예시

import cv2
import json
import numpy as np

def visualize_annotations(image_path, annotation_path):
    """생성된 어노테이션을 이미지 위에 시각화"""
    img = cv2.imread(image_path)

    with open(annotation_path) as f:
        annotations = json.load(f)

    for bbox in annotations.get("bounding_boxes", []):
        x_min = int(bbox["x_min"])
        y_min = int(bbox["y_min"])
        x_max = int(bbox["x_max"])
        y_max = int(bbox["y_max"])
        label = bbox.get("class", "unknown")

        # 바운딩 박스 그리기
        cv2.rectangle(img, (x_min, y_min), (x_max, y_max), (0, 255, 0), 2)
        cv2.putText(img, label, (x_min, y_min - 5),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 1)

    return img
```

## 합성 데이터의 한계와 현실적 기대치

Replicator와 합성 데이터에 대한 현실적인 시각을 갖추는 것이 중요합니다.

합성 데이터가 잘 작동하는 경우:
- 물체의 형태와 크기가 시뮬레이션에서 정확하게 재현될 때
- 도메인 랜덤화가 실제 환경의 변동 범위를 충분히 커버할 때
- RTX 경로 추적으로 물리적으로 정확한 렌더링을 사용할 때

여전히 어려운 경우:
- **반투명·투명 물체**: 굴절 효과를 완벽히 재현하기 어려움
- **매우 반사적인 표면**: 환경 반사의 정확한 재현이 도전적
- **질감이 복잡한 소재**: 직물, 식물 등 고주파 텍스처 재현 한계
- **새로운 환경 구조**: 훈련 환경과 전혀 다른 배경에서의 일반화

이 한계를 인식하고, 합성 데이터로 사전학습 후 소량의 실제 데이터로 파인튜닝하는 **합성+실제 데이터 혼합 전략**이 현실적으로 유효한 접근입니다.
