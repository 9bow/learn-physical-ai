## 렌더링 기술이 왜 로보틱스에 중요한가

로봇 정책을 시뮬레이션에서 학습한 뒤 실물로 전이할 때, **시각적 sim-to-real 갭**은 주요 실패 원인입니다. 시뮬레이터가 생성한 이미지가 실제 카메라 영상과 차이가 클수록, 시각 관측에 의존하는 정책의 성능이 저하됩니다.

**Neural rendering(신경 렌더링)** 은 실제 장면을 촬영한 사진으로부터 임의의 시점에서 사실적인 이미지를 합성하는 기술입니다. 이를 로보틱스에 적용하면 실제 환경을 스캔하여 시뮬레이션 자산으로 변환하는 **real-to-sim** 파이프라인이 가능해집니다.

## NeRF: 암시적 장면 표현

**NeRF(Neural Radiance Fields)** 는 2020년 제안된 기념비적 방법으로, 3차원 장면을 신경망의 가중치로 암시적으로 표현합니다.

### 핵심 아이디어

NeRF는 3D 공간의 한 점 $(x, y, z)$와 카메라 방향 $(\theta, \phi)$를 입력으로 받아, 그 방향에서 관측되는 색상(RGB)과 밀도(opacity $\sigma$)를 출력하는 MLP(다층 퍼셉트론)를 학습합니다.

렌더링은 **볼륨 렌더링(volume rendering)** 방정식으로 수행됩니다. 카메라 광선(ray)을 따라 여러 점에서 색상과 밀도를 쿼리하고, 밀도에 따른 가중합으로 최종 픽셀 색상을 계산합니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│                      NeRF 렌더링 파이프라인                       │
│                                                                  │
│  카메라 위치 + 방향                                               │
│       │                                                          │
│       ▼                                                          │
│  광선(ray) 생성 ──▶ 샘플링(N개 점) ──▶ MLP 쿼리                  │
│                                         │                        │
│                        (x,y,z,θ,φ) ──▶ (RGB, σ)               │
│                                         │                        │
│                       볼륨 렌더링 적분 ──▶ 픽셀 색상             │
│                                         │                        │
│                   실제 사진과 비교 ──▶ 손실 계산 ──▶ MLP 업데이트│
└──────────────────────────────────────────────────────────────────┘
```

입력으로 동일 장면의 다각도 사진(수십~수백 장)과 각 사진의 카메라 자세(pose)만 있으면 학습이 가능합니다. 학습 후에는 훈련 중 보지 못한 임의의 시점에서 사실적인 이미지를 합성할 수 있습니다.

### NeRF의 한계

- **학습 속도**: 원본 NeRF는 단일 장면 학습에 수 시간이 걸렸습니다.
- **렌더링 속도**: 실시간 렌더링이 어려웠습니다.
- **편집 어려움**: 암시적 표현이라 개별 물체를 분리하거나 편집하기 어렵습니다.

이후 Instant-NGP, Mip-NeRF 등 다양한 개선 버전이 등장했고, 학습 시간이 수분 이내로 단축되었습니다.

## 3D Gaussian Splatting(3DGS)

**3D Gaussian Splatting(3DGS)** 은 2023년에 제안되어 neural rendering 분야에 큰 주목을 받은 방법입니다. NeRF의 암시적 신경망 대신 **명시적(explicit) 가우시안(Gaussian) 원소들의 집합**으로 장면을 표현합니다.

### 표현 방식

각 3D Gaussian은 다음 파라미터를 갖습니다.
- **위치(position)**: 3D 공간에서의 중심점 $\mu$
- **공분산(covariance)**: 가우시안의 형태와 크기를 결정하는 행렬 $\Sigma$
- **색상(color/spherical harmonics)**: 방향에 따라 달라지는 색상 표현
- **불투명도(opacity)**: 가우시안의 투명도 $\alpha$

수백만 개의 작은 3D Gaussian들이 모여 장면 전체를 표현합니다.

### 렌더링: 스플래팅(Splatting)

렌더링은 카메라 관점에서 각 3D Gaussian을 2D 이미지 평면에 투영(projection)하여, 깊이 순서대로 알파 블렌딩(alpha blending)하는 방식입니다. 이 과정은 GPU 래스터화(rasterization) 파이프라인과 유사하여 **실시간 렌더링(수십~수백 fps)** 이 가능합니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│            NeRF vs. 3D Gaussian Splatting 비교                   │
├────────────────────────┬─────────────────────────────────────────┤
│ NeRF                   │ 3D Gaussian Splatting                   │
├────────────────────────┼─────────────────────────────────────────┤
│ 암시적 신경망          │ 명시적 가우시안 집합                    │
│ 학습: 수분~수시간      │ 학습: 수분 이내                         │
│ 렌더링: 느림(광선 추적)│ 렌더링: 실시간(래스터화)                │
│ 편집 어려움            │ 가우시안 직접 조작 가능                 │
│ 낮은 메모리            │ 높은 메모리(수백만 가우시안)            │
│ 부드러운 표면 잘 표현  │ 세밀한 질감 잘 표현                    │
└────────────────────────┴─────────────────────────────────────────┘
```

### 로보틱스에서의 3DGS 활용

3DGS의 **실시간 렌더링**과 **명시적 표현**은 로보틱스 응용에서 특히 유용합니다.

- **가우시안 분리(Gaussian Grouping)**: 각 가우시안에 세그멘테이션 레이블을 부여하여 개별 물체를 분리할 수 있습니다. 물체를 꺼내거나 배치를 바꾸는 장면 편집이 가능합니다.
- **동적 장면**: 움직이는 물체나 관절 로봇을 표현하기 위한 동적 3DGS 연구가 진행 중입니다.
- **물리 통합**: 가우시안에 물리 파라미터를 결합하여 상호작용 시뮬레이션을 지원하는 시도가 있습니다.

## Real-to-Sim 파이프라인

Neural rendering의 로보틱스 핵심 응용은 실제 환경을 시뮬레이션 자산으로 변환하는 **real-to-sim** 파이프라인입니다.

```text
┌────────────────────────────────────────────────────────────────┐
│                  Real-to-Sim 파이프라인                         │
│                                                                │
│  실제 장면 촬영               캡처 단계                        │
│  (스마트폰/RGBD 카메라)                                        │
│       │                                                        │
│       ▼                                                        │
│  카메라 자세 추정              COLMAP / SLAM                   │
│  (Structure-from-Motion)                                       │
│       │                                                        │
│       ▼                                                        │
│  NeRF / 3DGS 학습             Neural Rendering                │
│  (사실적 렌더링 모델)                                          │
│       │                                                        │
│       ▼                                                        │
│  메시/가우시안 추출            자산화                          │
│  (물체 분리, 텍스처 추출)                                      │
│       │                                                        │
│       ▼                                                        │
│  시뮬레이터 임포트             통합                            │
│  (USD, URDF, PhysX)                                           │
│       │                                                        │
│       ▼                                                        │
│  로봇 정책 학습                활용                            │
└────────────────────────────────────────────────────────────────┘
```

이 파이프라인의 장점은 3D 모델링 전문가 없이도 실제 환경을 시뮬레이터에 재현할 수 있다는 것입니다. 특히 산업 현장처럼 복잡하고 구체적인 환경을 빠르게 디지털 트윈(digital twin)으로 만들 수 있습니다.

## 한계와 미해결 문제

**물리 파라미터 부재**: NeRF나 3DGS는 시각적 외관을 잘 재현하지만, 물체의 질량, 마찰 계수, 탄성 같은 물리 파라미터는 포함하지 않습니다. 재질·물리 속성(색상·반사율 같은 PBR 파라미터 포함)은 별도의 추정 과정이 필요하며, 시뮬레이션에서 물리 상호작용을 재현하려면 별도의 시스템 식별이 필요합니다.

**투명·반사 표면**: 유리, 금속 표면처럼 시점에 따라 크게 달라 보이는 물질의 재현이 어렵습니다.

**대규모 장면**: 수천 평 규모의 공장 전체를 고품질로 재현하는 데는 계산·메모리 비용이 큽니다.

**동적 변화**: 물체가 자주 이동하거나 환경이 변하는 경우, 재촬영과 재학습이 필요합니다.

Neural rendering 기술은 빠르게 발전하고 있으며, 실시간 동적 재구성, 물리 통합 표현 등의 방향으로 연구가 진행 중입니다. 로보틱스 시뮬레이션의 시각적 사실성 문제를 해결하는 핵심 기술로 자리잡을 가능성이 높습니다.
