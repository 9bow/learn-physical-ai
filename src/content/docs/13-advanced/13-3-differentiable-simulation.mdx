## 시뮬레이션에 미분이 왜 필요한가

일반적인 물리 시뮬레이터는 "입력 제어→물리 시뮬레이션→결과 상태"의 방향으로만 동작합니다. 이를 **전향(forward) 시뮬레이션**이라고 합니다. 예를 들어 강화학습에서는 수천만 번의 시뮬레이션 스텝을 거쳐 보상 신호를 통해 점진적으로 좋은 행동을 학습합니다.

**미분가능 시뮬레이션(differentiable simulation)** 은 여기에 **역향(backward) 계산**, 즉 결과에 대한 그래디언트(gradient)를 시뮬레이션 전체를 통해 역전파(backpropagate)할 수 있게 합니다. 이를 통해 "결과가 좋아지려면 입력 제어를 어떻게 바꿔야 하는가"를 직접 계산할 수 있습니다.

```text
┌────────────────────────────────────────────────────────────────────┐
│        일반 시뮬레이션 vs. 미분가능 시뮬레이션                      │
├─────────────────────────┬──────────────────────────────────────────┤
│ 일반 시뮬레이션          │ 미분가능 시뮬레이션                      │
├─────────────────────────┼──────────────────────────────────────────┤
│ 제어 → 상태 (전향만)    │ 제어 → 상태 → 손실                       │
│ 그래디언트 없음          │ ∂손실/∂제어 계산 가능 (역전파)           │
│ RL: 수백만 샘플 필요    │ 그래디언트 기반: 수천 샘플로 최적화       │
│ 시뮬 자체는 블랙박스    │ 시뮬레이터가 미분 계산 그래프에 통합      │
└─────────────────────────┴──────────────────────────────────────────┘
```

## 핵심 아이디어: 자동 미분과 물리 시뮬레이션의 결합

미분가능 시뮬레이션의 구현 핵심은 **자동 미분(automatic differentiation, AD)** 입니다. 딥러닝 프레임워크(PyTorch, JAX)가 신경망의 그래디언트를 자동으로 계산하는 것과 동일한 원리를, 물리 시뮬레이션 코드에 적용합니다.

물리 시뮬레이션 한 스텝은 수학적으로 함수 $f: (q_t, \dot{q}_t, u_t) \to (q_{t+1}, \dot{q}_{t+1})$로 표현됩니다. 여기서 $q$는 위치, $\dot{q}$는 속도, $u$는 제어 입력입니다. 이 함수를 미분 가능하게 구현하면 전체 궤적에 대한 그래디언트를 계산할 수 있습니다.

## 주요 프레임워크

### DiffTaichi

**DiffTaichi**는 Python 기반 미분가능 프로그래밍 언어 Taichi 위에서 구현된 미분가능 물리 엔진입니다. 유체, 탄성체, 강체를 포함한 다양한 물리 현상을 지원하며, GPU 가속과 자동 미분을 결합합니다.

```text
┌─────────────────────────────────────────────────────────────────┐
│                     DiffTaichi 특징                              │
│                                                                 │
│  • Taichi 언어: Python 문법, GPU 커널 자동 생성                 │
│  • 지원 물리: 강체, 연체(MPM), 유체(SPH), 천 시뮬레이션        │
│  • 역전파: Taichi 커스텀 AD 엔진                                │
│  • 응용: 재료 최적화, 로봇 관절 제어                            │
└─────────────────────────────────────────────────────────────────┘
```

### Brax

**Brax**는 Google이 개발한 JAX 기반 미분가능 물리 엔진으로, 강체 동역학(rigid body dynamics)에 특화되어 있습니다. JAX의 `jit`, `vmap`, `grad` 변환을 그대로 활용할 수 있어 TPU/GPU에서 대규모 병렬 시뮬레이션이 가능합니다.

Brax는 MuJoCo와 유사한 인터페이스를 제공하면서 그래디언트 계산을 지원합니다. RL과 그래디언트 기반 최적화를 조합하거나, 모델 기반 RL(model-based RL)에서 세계 모델로 활용하는 연구에 많이 쓰입니다.

### NVIDIA Warp

**NVIDIA Warp**는 CUDA 커널을 Python 코드로 작성하고, 자동으로 순방향·역방향 패스를 생성하는 프레임워크입니다. 물리 시뮬레이션, 기하 처리, 로봇 동역학을 미분가능하게 구현하는 데 사용됩니다. NVIDIA Isaac Sim과 통합되어 있어 실제 로봇 시뮬레이션 파이프라인에서 바로 활용할 수 있습니다.

## 그래디언트 기반 제어와 응용

미분가능 시뮬레이션이 가능해지면 여러 중요한 응용이 열립니다.

### 궤적 최적화(Trajectory Optimization)

목표 상태에 도달하는 최적 제어 입력 시퀀스를 그래디언트 하강(gradient descent)으로 직접 찾을 수 있습니다. 기존의 반복적 LQR(iLQR), MPPI(Model Predictive Path Integral) 등 전통적 궤적 최적화가 근사하려 했던 것을 더 직접적으로 수행합니다.

손실 함수를 설계하는 유연성이 높습니다. "최종 위치 오차 최소화"뿐 아니라 "에너지 소비 최소화", "관절 가속도 부드럽게", "장애물과의 거리 최대화"를 동시에 목표로 설정할 수 있습니다.

### 시스템 식별(System Identification)

**시스템 식별**은 실제 로봇의 물리 파라미터(관절 마찰, 링크 질량, 액추에이터 특성 등)를 측정 데이터로부터 추정하는 과정입니다. 미분가능 시뮬레이션을 사용하면 이를 최적화 문제로 정형화할 수 있습니다.

실물 로봇 실험 데이터와 시뮬레이션 예측 사이의 오차를 손실 함수로 정의하고, 물리 파라미터에 대해 그래디언트를 계산하여 업데이트합니다. 이렇게 식별된 파라미터로 시뮬레이터를 조정하면 sim-to-real 갭을 줄일 수 있습니다.

### 정책 그래디언트 보강

순수 RL은 보상 신호만으로 학습하므로 샘플 효율이 낮습니다. 미분가능 시뮬레이션에서 그래디언트를 추가로 활용하면 학습 속도를 크게 높일 수 있습니다. 이를 **분석적 그래디언트 정책 최적화(analytic policy gradient)** 라고 부릅니다.

## 접촉 역학의 어려움

미분가능 시뮬레이션의 가장 큰 도전은 **접촉(contact)** 입니다.

### 불연속 접촉의 문제

물체가 바닥에 닿거나 로봇이 물체를 집는 순간, 물리 상태는 불연속적으로 변합니다. 수학적으로 이는 비미분점(non-differentiable point)을 의미합니다. 접촉 이전과 이후의 역학이 완전히 달라지므로 그래디언트를 통해 "접촉 여부"를 직접 최적화하기 어렵습니다.

```text
┌──────────────────────────────────────────────────────────────────┐
│               접촉 미분의 어려움                                  │
│                                                                  │
│  접촉 없음: 연속적, 미분 가능 ──▶ 그래디언트 정상 작동          │
│                                                                  │
│  접촉 발생:                                                      │
│    충격력 (impulse): 순간적, 불연속                              │
│    마찰력: 정적/동적 전환 불연속                                 │
│    ──▶ 그래디언트 폭발(explode) 또는 소실(vanish)               │
│    ──▶ "그래디언트 불안정" 문제                                        │
└──────────────────────────────────────────────────────────────────┘
```

### 해결 접근법들

**부드러운 접촉 모델(smooth contact model)**: 딱딱한 충격(hard contact) 대신 탄성 패널티(spring-like penalty) 모델을 사용하여 접촉을 연속적으로 근사합니다. 미분 가능성을 얻는 대신 물리적 정확도를 일부 포기합니다.

**완화(relaxation) 기법**: 접촉 조건을 연속적으로 완화하여 미분 가능한 대리 문제(surrogate)를 풉니다.

**혼합 접근(hybrid approach)**: 접촉 없는 구간에서는 미분가능 시뮬레이션의 그래디언트를 활용하고, 접촉이 포함된 구간에서는 샘플 기반 방법을 사용합니다.

## 미분가능 시뮬레이션의 현실적 한계

미분가능 시뮬레이션은 강력하지만, 현실적인 한계도 분명합니다.

- **긴 시간 궤적의 그래디언트 폭발**: 수십~수백 스텝에 걸친 역전파는 그래디언트 폭발(exploding gradient) 문제를 일으킵니다.
- **구현 복잡도**: 일반 시뮬레이터보다 구현과 디버깅이 훨씬 어렵습니다.
- **접촉 밀도 제한**: 접촉이 많은 조작 과제(천 다루기, 분말 다루기)에서는 안정적인 그래디언트를 얻기 어렵습니다.
- **모델 오차**: 시뮬레이션이 실제 물리를 완전히 대표하지 못하면, 그래디언트 방향이 실제 최적화 방향과 다를 수 있습니다.

이런 이유로 현재는 미분가능 시뮬레이션을 RL과 결합하거나, 접촉이 적은 특정 구간에 선택적으로 적용하는 방식이 실용적입니다. 기술의 발전으로 접촉 역학 미분의 안정성이 개선되면 더 넓은 응용이 가능해질 전망입니다.
