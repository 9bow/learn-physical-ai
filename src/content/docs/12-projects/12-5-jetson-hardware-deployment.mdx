## 프로젝트 개요

시뮬레이션에서 학습한 정책과 인식 모델은 결국 실제 로봇 하드웨어에서 동작해야 합니다. 이 프로젝트에서는 학습된 모델을 **NVIDIA Jetson Orin**에 배포하는 전체 과정을 다룹니다.

Jetson Orin은 로봇 엣지 컴퓨팅의 사실상 표준 플랫폼입니다. 256~2048 CUDA 코어(모델에 따라 다름), DLA(Deep Learning Accelerator), 통합 메모리 구조를 갖추고 있어 실시간 AI 추론에 적합합니다.

```text
┌──────────────────────────────────────────────────────────────────────┐
│               Jetson 하드웨어 배포 파이프라인                        │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  개발 서버 (데이터센터/워크스테이션)                                  │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Isaac Lab 학습 완료된 체크포인트 (.pt)                       │   │
│  │  Isaac Sim 합성데이터 학습 모델 (.onnx)                      │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                    │ 모델 파일 전송                                   │
│                    ▼                                                  │
│  Jetson Orin (엣지 디바이스)                                         │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  [1] TensorRT 변환    →   [2] 성능 검증                       │   │
│  │       (Jetson 위에서)         (정밀도/속도 측정)              │   │
│  │                                                              │   │
│  │  [3] Isaac ROS 통합   →   [4] 실시간 파이프라인 테스트        │   │
│  │       (GEM 연결)             (센서→추론→제어 루프)            │   │
│  │                                                              │   │
│  │  [5] 안전 메커니즘    →   [6] 실제 로봇 통합 테스트           │   │
│  │       (e-stop, 한계치)        (점진적 검증)                  │   │
│  └──────────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────────┘
```

## 준비물

- NVIDIA Jetson Orin 개발 키트 (AGX Orin, Orin NX, Orin Nano 중 하나)
- JetPack SDK 설치 (CUDA, cuDNN, TensorRT 포함)
- ROS 2 설치 (JetPack 호환 버전)
- 학습된 모델 파일 (이전 프로젝트 산출물)
- 실제 로봇 또는 Isaac Sim SIL 환경 (최종 테스트용)

## 단계 1: Jetson 환경 설정

```bash
# Jetson 기본 정보 확인
sudo apt-get install -y jetson-stats
sudo jtop  # GPU/CPU/메모리 사용률 모니터링 도구

# JetPack 버전 및 CUDA 확인
cat /etc/nv_tegra_release
nvcc --version

# TensorRT 설치 확인
dpkg -l | grep tensorrt

# Jetson 성능 모드 설정 (최대 성능)
sudo nvpmodel -m 0   # MAXN 모드 (AGX Orin 기준 — 모델별 다름)
sudo jetson_clocks   # CPU/GPU/메모리 클럭 최대화
```

**JetPack**은 Jetson 플랫폼의 통합 SDK입니다. CUDA, cuDNN, TensorRT, VPI가 함께 패키지로 제공됩니다. JetPack 버전과 Isaac ROS 버전의 호환성을 반드시 확인합니다.

## 단계 2: TensorRT 모델 최적화

개발 서버에서 학습한 모델을 Jetson에서 직접 TensorRT 엔진으로 변환합니다. TensorRT 엔진은 **생성된 GPU에 종속**되므로, 반드시 배포 타겟인 Jetson 위에서 변환해야 합니다.

### FP32 vs FP16 vs INT8

```text
┌───────────────────────────────────────────────────────────────────┐
│ TensorRT 정밀도 비교 (Jetson Orin 기준 참고값, 모델마다 다름)      │
├──────────────┬──────────────┬─────────────────┬───────────────────┤
│ 정밀도       │ 상대 속도    │ 메모리 사용      │ 정확도 손실       │
├──────────────┼──────────────┼─────────────────┼───────────────────┤
│ FP32         │ 1x (기준)    │ 100%            │ 없음              │
│ FP16         │ ~2x          │ 50%             │ 무시할 수준       │
│ INT8         │ ~4x          │ 25%             │ 모델/태스크 의존  │
└──────────────┴──────────────┴─────────────────┴───────────────────┘
```

```bash
# FP16 변환 (권장 시작점 — 속도와 정확도 균형)
trtexec \
  --onnx=/path/to/policy.onnx \
  --saveEngine=/path/to/policy_fp16.engine \
  --fp16 \
  --workspace=4096 \
  --verbose

# INT8 변환 (캘리브레이션 데이터 필요)
trtexec \
  --onnx=/path/to/detection_model.onnx \
  --saveEngine=/path/to/detection_int8.engine \
  --int8 \
  --calib=/path/to/calibration_data/ \
  --workspace=4096

# 변환된 엔진 성능 측정
trtexec \
  --loadEngine=/path/to/policy_fp16.engine \
  --batch=1 \
  --iterations=1000 \
  --warmUp=200
```

### DLA(Deep Learning Accelerator) 활용

Jetson Orin의 DLA는 GPU와 별개로 동작하는 추론 전용 하드웨어입니다. GPU와 DLA를 동시에 활용하면 전체 처리량을 더 높일 수 있습니다.

```bash
# DLA 코어 0에서 실행 (DLA 지원 레이어만 DLA로 오프로드)
trtexec \
  --onnx=/path/to/model.onnx \
  --saveEngine=/path/to/model_dla.engine \
  --fp16 \
  --useDLACore=0 \
  --allowGPUFallback  # DLA 미지원 레이어는 GPU 폴백
```

## 단계 3: Isaac ROS on Jetson

Isaac ROS는 Jetson을 공식 지원 플랫폼으로 제공합니다. Dev Container(Docker) 방식을 권장합니다.

```bash
# Isaac ROS Dev Container 설정 (공식 문서 참조)
# https://nvidia-isaac-ros.github.io/getting_started/index.html

# GPU 접근 권한 설정 (Jetson에서 Docker GPU 사용)
sudo usermod -aG docker $USER

# Isaac ROS 컨테이너 실행 (공식 스크립트 사용 — 버전별 명령 다름)

colcon build --symlink-install \
  --packages-select isaac_ros_object_detection isaac_ros_visual_slam

source install/setup.bash
```

## 단계 4: 실시간 추론 파이프라인 구성

Jetson 위에서 전체 인식+제어 파이프라인을 ROS 2로 구성합니다.

```text
Jetson 위 실시간 파이프라인:

카메라 (CSI/USB)
    │ sensor_msgs/Image
    ▼
Isaac ROS Image Proc (GPU 전처리)
    │ NITROS zero-copy
    ▼
Isaac ROS Object Detection (TensorRT FP16)
    │ Detection2DArray
    ▼
Isaac ROS Pose Estimation (TensorRT FP16)
    │ PoseArray
    ▼
정책/제어 노드
    │ JointState (관절 명령)
    ▼
ros2_control → 하드웨어 인터페이스
    │
    ▼
실제 로봇 관절 구동
```

```bash
# 전체 파이프라인 런치 파일 실행
ros2 launch robot_deploy full_pipeline.launch.py \
  detection_engine:=/path/to/detection_fp16.engine \
  policy_engine:=/path/to/policy_fp16.engine \
  camera_topic:=/camera/image_raw

# 파이프라인 성능 모니터링
ros2 topic hz /camera/image_raw
ros2 topic hz /detections
ros2 topic hz /joint_commands

# 엔드-투-엔드 레이턴시 측정
ros2 run rqt_graph rqt_graph
```

## 단계 5: 안전 메커니즘 구현

실제 하드웨어 통합에서 안전은 최우선 사항입니다.

```python
# 안전 감시 노드 (개념 이해용 의사코드)
class SafetyMonitorNode(Node):
    def __init__(self):
        super().__init__('safety_monitor')

        # 관절 상태 모니터링
        self.create_subscription(JointState, '/joint_states',
                                 self.monitor_joints, 1)

        # 비상 정지 발행
        self.estop_pub = self.create_publisher(Bool, '/e_stop', 10)

        # 안전 한계값 (로봇별 하드웨어 스펙 참조 필수)
        self.joint_velocity_limit = 2.0   # rad/s (예시)
        self.joint_torque_limit = 50.0    # Nm (예시)

        # 감시 루프 (100Hz)
        self.create_timer(0.01, self.safety_loop)

    def monitor_joints(self, msg: JointState):
        # 관절 속도 초과 감지
        for vel in msg.velocity:
            if abs(vel) > self.joint_velocity_limit:
                self.get_logger().error(
                    f"관절 속도 한계 초과: {vel:.2f} rad/s")
                self._trigger_estop()
                return

    def _trigger_estop(self):
        estop_msg = Bool()
        estop_msg.data = True
        self.estop_pub.publish(estop_msg)
```

```text
안전 체크리스트:

[ ] 물리적 비상 정지(e-stop) 버튼 연결 및 테스트
[ ] 소프트웨어 e-stop 토픽 구현 및 모든 컨트롤러 연결
[ ] 관절 위치/속도/토크 한계치 설정 (하드웨어 스펙 기반)
[ ] 작업 공간(workspace) 경계 제한 구현
[ ] 통신 타임아웃 감지 (ROS 2 노드 연결 끊김 시 정지)
[ ] 배터리/전원 모니터링
[ ] 로그 기록 (rosbag2로 모든 토픽 저장)
```

## 단계 6: 점진적 실제 로봇 테스트

실제 로봇 통합은 반드시 단계적으로 진행합니다.

```text
점진적 테스트 절차:

Phase 1: SIL (Software-in-the-Loop)
├── Isaac Sim에서 전체 파이프라인 동작 확인
└── Jetson이 Isaac Sim과 통신하며 명령 생성 확인

Phase 2: HIL 준비 (Hardware-in-the-Loop)
├── 로봇 전원 ON, 관절 잠금 상태에서 명령 신호 확인
├── 토크 제한을 매우 낮게 설정 (최대값의 10% 이내)
└── 각 관절을 하나씩 독립적으로 움직여 방향/크기 확인

Phase 3: 제한 조건 실행
├── 로봇 속도를 학습 속도의 25%로 제한
├── 인간 감독 하에 단순 동작(홈 포지션 이동) 테스트
└── 속도를 점진적으로 50% → 75% → 100%로 증가

Phase 4: 완전 자율 동작
├── 제어 없는 완전 자율 실행
└── 안전 구역 내에서 반복 테스트, 성공률 기록
```

## 프로젝트 완료 기준

```text
[ ] Jetson에서 정책 추론 레이턴시 < 20ms (50Hz 이상)
[ ] 인식 파이프라인 (카메라→검출) 레이턴시 < 50ms
[ ] 엔드-투-엔드 제어 루프 안정적으로 20Hz 이상 유지
[ ] 비상 정지 응답 시간 < 50ms
[ ] 연속 30분 동작 중 크래시 없음
[ ] rosbag2 세션 기록 및 사후 분석 완료
```
