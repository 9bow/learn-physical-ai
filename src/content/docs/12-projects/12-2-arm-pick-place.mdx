## 프로젝트 개요

로봇 팔 Pick-and-Place는 로보틱스 조작(manipulation)의 기본이자 산업 현장에서 가장 수요가 높은 태스크입니다. 이 프로젝트에서는 Isaac Lab 시뮬레이션 환경에서 로봇 팔이 불규칙하게 놓인 물체를 집어 지정 위치에 내려놓는 정책을 학습하고, 이를 ROS 2 + MoveIt 2 스택과 통합합니다.

## 단계 2: 보상 함수 설계

Pick-and-Place 태스크는 희소 보상(sparse reward) 문제입니다. 정책이 우연히 물체를 집을 때까지 보상을 받지 못하면 학습이 매우 느립니다. 밀도 높은 보상(dense reward)을 설계해야 합니다.

```python
# 보상 함수 구성 예시 (개념 이해용 의사코드)
class PickPlaceRewards:
    # 1. 물체에 접근 보상
    def reaching_reward(self):
        dist = distance(ee_pos, object_pos)
        return torch.exp(-dist * 5.0) * 0.5

    # 2. 파지 보상 (그리퍼가 물체를 감쌌을 때)
    def grasping_reward(self):
        is_grasped = self._check_grasp()
        return is_grasped.float() * 1.0

    # 3. 물체 들어올리기 보상 (물체가 일정 높이 이상)
    def lifting_reward(self):
        is_lifted = (object_pos[:, 2] > lift_threshold) & is_grasped
        return is_lifted.float() * 2.0

    # 4. 목표 위치 도달 보상
    def goal_reward(self):
        dist_to_goal = distance(object_pos, goal_pos)
        is_success = dist_to_goal < success_threshold
        return is_success.float() * 10.0  # 희소 성공 보상

    # 5. 동작 패널티 (부드러운 동작 유도)
    def action_penalty(self):
        return -torch.sum(actions ** 2, dim=-1) * 0.01
```

## 단계 3: 파지 성공 판정

파지(grasp) 성공 판정은 조작 환경의 핵심 로직입니다.

```python
# 파지 성공 판정 (개념 예시)
def check_grasp(robot, object_prim):
    """
    파지 성공 조건:
    1. 그리퍼 양쪽 핑거가 물체와 접촉
    2. 그리퍼 개방도가 물체 크기에 비례한 범위 내
    3. 물체가 그리퍼와 함께 이동 (속도 일치)
    """
    left_contact = robot.gripper.left_finger.in_contact_with(object_prim)
    right_contact = robot.gripper.right_finger.in_contact_with(object_prim)
    gripper_width = robot.gripper.get_width()

    return (left_contact and right_contact and
            min_width < gripper_width < max_width)
```

물리 기반 파지 성공 판정 외에도, **물체 높이 기반** 판정을 함께 사용합니다. 물체가 테이블 위 일정 높이 이상으로 올라갔으면 파지 성공으로 간주합니다.

## 단계 5: MoveIt 2 통합

순수 RL 접근 외에도 MoveIt 2를 활용한 하이브리드 접근이 있습니다. 인식 파이프라인에서 얻은 물체 포즈를 MoveIt 2에 전달하여 파지 계획을 수립하고 실행합니다.

```python
# MoveIt 2 Pick-and-Place (개념 이해용 의사코드 — 실제 API는 공식 문서 확인)
import rclpy
from moveit.planning import MoveItPy
from geometry_msgs.msg import PoseStamped

moveit = MoveItPy(node_name="pick_place_node")
arm = moveit.get_planning_component("arm")
gripper = moveit.get_planning_component("gripper")

def pick_object(object_pose: PoseStamped):
    # 1. 프리-그라스프 포즈 계산 (물체 위 10cm)
    pre_grasp = compute_pre_grasp(object_pose, offset_z=0.1)

    # 2. 프리-그라스프 위치로 이동
    arm.set_goal_state(pose_stamped_msg=pre_grasp, pose_link="panda_hand")
    arm.plan()
    moveit.execute(arm.plan().trajectory)

    # 3. 그리퍼 열기
    gripper.set_goal_state(configuration_name="open")
    moveit.execute(gripper.plan().trajectory)

    # 4. 파지 위치로 직선 이동 (Cartesian planning)
    moveit.plan_cartesian_path(waypoints=[object_pose])

    # 5. 그리퍼 닫기 (파지)
    gripper.set_goal_state(configuration_name="closed")
    moveit.execute(gripper.plan().trajectory)

def place_object(target_pose: PoseStamped):
    # 목표 위치로 이동 후 그리퍼 열기 (내려놓기)
    arm.set_goal_state(pose_stamped_msg=target_pose, pose_link="panda_hand")
    moveit.execute(arm.plan().trajectory)
    gripper.set_goal_state(configuration_name="open")
    moveit.execute(gripper.plan().trajectory)
```

## 확장 아이디어

1. **다중 물체 조작**: 여러 물체를 순서대로 집어 정렬하는 태스크
2. **다양한 형상**: 구, 원통, 불규칙 형상 등 다양한 물체에 대한 일반화
3. **외력 강건성**: 외부에서 힘이 가해지는 상황에서도 파지 유지
4. **시각 기반 정책**: 관절 상태 대신 카메라 이미지를 직접 입력으로 사용 (end-to-end vision-based manipulation)
5. **양손 조작**: 두 팔이 협력하는 bimanual manipulation
